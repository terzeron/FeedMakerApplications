벡터 검색 (Semantic Search)의 한계

✅ 강점:
  "성능 개선" ≈ "속도 향상" ≈ "최적화"
  → 의미가 비슷하면 다 찾음!

  "hallucination" ≈ "환각" ≈ "헛소리"
  → 동의어도 잘 찾음!

❌ 약점:
  "GPT-4o" vs "GPT-4"
  → 둘 다 OpenAI 모델이라 벡터가 매우 유사
  → 구분 못함!

  "2024년" vs "2023년"
  → 둘 다 '연도'라는 개념이라 유사
  → 1년 차이를 못 느낌!

  "500ms" vs "빠른 응답"
  → "500ms"는 구체적 숫자
  → "빠른"은 추상적 개념
  → 벡터가 "빠른"을 더 선호!

왜?
  벡터는 "의미의 유사성"을 학습
  → 미묘한 차이보다 큰 의미를 우선

해결책: Hybrid Search

방법 1: 벡터 검색 (FAISS)
  ✅ 의미 이해 최강
  ✅ 유연한 매칭
  ✅ 동의어 처리

방법 2: 키워드 검색 (BM25)
  ✅ 정확한 단어 매칭
  ✅ 숫자/날짜/고유명사 강함
  ✅ 철자 정확도

Hybrid = 방법 1 + 방법 2
  → 둘의 장점만 결합!

실제 예시:
┌─────────────────────────────────────┐
│ "GPT-4o API 가격" 검색              │
├─────────────────────────────────────┤

│ BM25: "GPT-4o" 정확히 매칭하고      │
│ 합치면: GPT-4o 문서만 정확히!       │

def is_correct_answer(query: str, doc: str) -> bool:
    """해당 문서가 쿼리의 정답인지 판단"""
    if "GPT-4o" in query:

# 1. 현재 시스템: 벡터 검색
class VectorSearch:
    def __init__(self, documents: List[str]):
        self.documents = documents
        self.encoder = SentenceTransformer('all-MiniLM-L6-v2')
        print("🔄 문서 임베딩 중...")
        self.doc_embeddings = self.encoder.encode(documents)
        print("✅ 임베딩 완료!")

    def search(self, query: str, top_k: int = 3) -> List[Dict]:
        query_embedding = self.encoder.encode(query)

        scores = []
        for i, doc_emb in enumerate(self.doc_embeddings):
            # 코사인 유사도
            score = np.dot(query_embedding, doc_emb) / (
                np.linalg.norm(query_embedding) * np.linalg.norm(doc_emb)
            )
            scores.append({
                'doc_idx': i,
                'document': self.documents[i],
                'score': float(score)
            })

        scores.sort(key=lambda x: x['score'], reverse=True)
        return scores[:top_k]

# 2. 제안하는 방법: 간단한 키워드 검색
def simple_keyword_search(query: str, documents: List[str], top_k: int = 3) -> List[Dict]:
    """단어 겹침 기반 간단한 키워드 검색"""
    query_words = set(query.lower().split())

    scores = []
    for i, doc in enumerate(documents):
        doc_words = set(doc.lower().split())
        overlap = len(query_words & doc_words)
        scores.append({
            'doc_idx': i,
            'document': doc,
            'score': overlap
        })

    scores.sort(key=lambda x: x['score'], reverse=True)
    return scores[:top_k]

2.1 BM25란?

📖 스토리

화요일 오전, 준혁과 레이나는 BM25 구현에 들어갔다.

"키워드 검색의 황제, BM25를 배워보자," 레이나가 말했다.

"BM25가 뭐예요?"

"Best Matching 25의 약자야," 레이나가 화이트보드에 수식을 적었다. "검색 엔진의 핵심 알고리즘이지. Google, Elasticsearch 모두 이걸 써."

BM25의 3가지 핵심 아이디어

1️⃣ TF (Term Frequency): 단어 빈도
   단어가 많이 나올수록 중요!

   예: "API"가 문서에 10번 vs 1번
       → 10번 나온 문서가 더 관련성 높음

   하지만! 무한정 높아지진 않음 (포화)
   → 10번이나 100번이나 비슷한 점수

2️⃣ IDF (Inverse Document Frequency): 역문서 빈도
   희귀한 단어일수록 중요!

   예: "the" (모든 문서에 있음) vs "GPT-4o" (1개 문서만)
       → "GPT-4o"가 훨씬 중요!

   계산: log((전체 문서수 - 단어 포함 문서수) / 단어 포함 문서수)

3️⃣ 문서 길이 정규화
   긴 문서가 불이익 받지 않게!

   예: 100자 문서와 10,000자 문서를
       공정하게 비교

준혁이 노트에 적으며 물었다. "그럼 공식은 어떻게 되요?"

레이나가 칠판에 적었다.

BM25 공식:

Score(문서, 쿼리) = Σ IDF(단어) × (TF × (k1 + 1)) / (TF + k1 × (1 - b + b × 문서길이/평균길이))

여기서:
- k1 = 1.5 (TF 포화 계수, 보통 1.2~2.0)
- b = 0.75 (길이 정규화, 보통 0.75)
- TF = 문서 내 단어 빈도
- IDF = log((N - df + 0.5) / (df + 0.5))
  - N = 전체 문서 수
  - df = 해당 단어를 포함한 문서 수

"복잡해 보이는데요," 준혁이 고개를 갸우뚱했다.

"코드로 보면 간단해," 레이나가 말했다.

2.2 DataPulse BM25 구현

📖 스토리

준혁이 DataPulse에 최적화된 BM25를 구현하기 시작했다.

import math
from collections import Counter
from typing import List, Dict
import re

class DataPulseBM25:
    """
    DataPulse 맞춤형 BM25 검색

    특징:
    - 한글 + 영문 + 숫자 모두 지원
    - "GPT-4o", "500ms" 같은 특수 토큰 보존
    - 빠른 검색 속도
    """

    def __init__(self, k1: float = 1.5, b: float = 0.75):
        """
        Args:
            k1: TF 포화 계수 (기본 1.5)
                높을수록 TF의 영향력이 커짐
            b: 길이 정규화 계수 (기본 0.75)
                1에 가까울수록 길이 영향이 커짐
        """
        self.k1 = k1
        self.b = b
        self.documents = []
        self.doc_freqs = []  # 각 문서의 단어 빈도
        self.idf = {}        # 각 단어의 IDF 값
        self.avg_doc_len = 0

    def tokenize(self, text: str) -> List[str]:
        """
        토크나이징 (한글/영문/숫자 지원)

        특별 처리:
        - "GPT-4o" → ["gpt-4o"] (하이픈 유지)
        - "2024년" → ["2024년"] (년 유지)
        - "500ms" → ["500ms"] (단위 유지)
        """
        # 소문자 변환
        text = text.lower()

        # 한글, 영문, 숫자, 하이픈만 남기고 제거
        text = re.sub(r'[^\w\s가-힣-]', ' ', text)

        # 공백으로 분리
        tokens = text.split()

        return [t for t in tokens if len(t) > 0]

    def fit(self, documents: List[str]):
        """문서 인덱싱 (사전 처리)"""
        self.documents = documents

        print(f"\n🔄 BM25 인덱싱 시작...")

        # 1. 모든 문서 토크나이징
        tokenized_docs = [self.tokenize(doc) for doc in documents]

        # 2. 각 문서의 단어 빈도 계산
        self.doc_freqs = [Counter(doc) for doc in tokenized_docs]

        # 3. 평균 문서 길이 계산
        doc_lens = [len(doc) for doc in tokenized_docs]
        self.avg_doc_len = sum(doc_lens) / len(doc_lens) if doc_lens else 0

        # 4. IDF 계산
        # 각 단어가 몇 개 문서에 출현하는지 계산
        df = {}  # Document Frequency
        for doc_freq in self.doc_freqs:
            for word in doc_freq.keys():
                df[word] = df.get(word, 0) + 1

        # IDF 공식 적용
        N = len(documents)
        for word, doc_count in df.items():
            # BM25 IDF: log((N - df + 0.5) / (df + 0.5))
            self.idf[word] = math.log((N - doc_count + 0.5) / (doc_count + 0.5) + 1.0)

        print(f"   ✅ 인덱싱 완료!")
        print(f"   📊 문서 수: {N}개")
        print(f"   📊 총 어휘: {len(self.idf)}개")
        print(f"   📊 평균 문서 길이: {self.avg_doc_len:.1f} 단어")

        # 가장 높은 IDF 단어 (희귀한 단어)
        top_idf_words = sorted(self.idf.items(), key=lambda x: x[1], reverse=True)[:5]
        print(f"   📊 희귀 단어 예시:")
        for word, idf_score in top_idf_words:
            print(f"      '{word}': IDF={idf_score:.3f}")

    def score(self, query: str, doc_idx: int) -> float:
        """쿼리-문서 간 BM25 점수 계산"""
        query_words = self.tokenize(query)
        doc_freq = self.doc_freqs[doc_idx]
        doc_len = sum(doc_freq.values())

        score = 0.0

        for word in query_words:
            if word not in doc_freq:
                continue  # 문서에 없는 단어는 점수 0

            # 1. TF (단어 빈도)
            tf = doc_freq[word]

            # 2. IDF (역문서 빈도)
            idf = self.idf.get(word, 0)

            # 3. 문서 길이 정규화
            norm = 1 - self.b + self.b * (doc_len / self.avg_doc_len)

            # 4. BM25 공식
            word_score = idf * (tf * (self.k1 + 1)) / (tf + self.k1 * norm)
            score += word_score

        return score

    def search(self, query: str, top_k: int = 5) -> List[Dict]:
        """검색 실행"""
        scores = []

        # 모든 문서에 대해 점수 계산
        for i in range(len(self.documents)):
            score = self.score(query, i)

            if score > 0:  # 점수가 0인 문서는 제외
                scores.append({
                    'doc_idx': i,
                    'document': self.documents[i],
                    'score': score
                })

        # 점수순 정렬
        scores.sort(key=lambda x: x['score'], reverse=True)

        return scores[:top_k]

    def display_results(self, query: str, employee: str = ""):
        """검색 결과 표시"""
        results = self.search(query, top_k=3)

        print(f"\n{'='*80}")
        if employee:
            print(f"👤 {employee}")
        print(f"🔑 BM25 검색: '{query}'")
        print(f"{'='*80}")

        if not results:
            print("  검색 결과 없음")
            return results

        for i, result in enumerate(results, 1):
            is_correct = is_correct_answer(query, result['document'])

            print(f"\n  {i}위 [BM25: {result['score']:.3f}] {icon}")
            print(f"  {result['document'][:70]}...")

        return results

RRF (Reciprocal Rank Fusion)

핵심 아이디어:
  점수가 아닌 **순위**를 사용!

공식:
  RRF_score(문서) = Σ 1 / (k + rank_i)

  여기서:
  - k = 60 (상수, 보통 60 사용)
  - rank_i = i번째 검색에서의 순위

예시:
  문서 0이
    - 벡터 검색에서 2위 → 1/(60+2) = 0.0161
    - BM25 검색에서 1위 → 1/(60+1) = 0.0164
    - 합계 → 0.0325

  문서 1이
    - 벡터 검색에서 1위 → 1/(60+1) = 0.0164
    - BM25 검색에서 2위 → 1/(60+2) = 0.0161
    - 합계 → 0.0325

장점:
  ✅ 점수 스케일에 독립적
  ✅ 간단하고 효과적
  ✅ 검색 엔진 업계 표준
  ✅ 두 검색이 공정하게 기여

from collections import defaultdict

class DataPulseHybridSearch:
    """
    DataPulse Hybrid Search 시스템

    벡터 검색 + BM25 검색 + RRF 융합
    """

    def __init__(
        self,
        vector_searcher,
        bm25_searcher,
        k: int = 60
    ):
        """
        Args:
            vector_searcher: 벡터 검색 엔진
            bm25_searcher: BM25 검색 엔진
            k: RRF 상수 (기본 60)
        """
        self.vector_searcher = vector_searcher
        self.bm25_searcher = bm25_searcher
        self.k = k

    def reciprocal_rank_fusion(
        self,
        vector_results: List[Dict],
        bm25_results: List[Dict],
        alpha: float = 0.5
    ) -> List[Dict]:
        """
        RRF로 두 검색 결과 융합

        Args:
            vector_results: 벡터 검색 결과
            bm25_results: BM25 검색 결과
            alpha: 벡터 가중치 (0~1)
                  0.5 = 동등 (기본)
                  0.7 = 벡터 우선 (의미 중요)
                  0.3 = BM25 우선 (정확한 매칭 중요)

        Returns:
            융합된 결과
        """

        # 문서별 RRF 점수 누적
        rrf_scores = defaultdict(float)
        doc_map = {}  # doc_idx → 문서 내용

        # 1. 벡터 검색 결과에서 RRF 점수 계산
        for rank, result in enumerate(vector_results, 1):
            doc_idx = result['doc_idx']
            doc_map[doc_idx] = result['document']

            # RRF 점수 = 1 / (k + rank)
            rrf_score = 1.0 / (self.k + rank)

            # 가중치 적용
            rrf_scores[doc_idx] += alpha * rrf_score

        # 2. BM25 검색 결과에서 RRF 점수 계산
        for rank, result in enumerate(bm25_results, 1):
            doc_idx = result['doc_idx']

            if doc_idx not in doc_map:
                doc_map[doc_idx] = result['document']

            rrf_score = 1.0 / (self.k + rank)

            # 가중치 적용 (1 - alpha)
            rrf_scores[doc_idx] += (1 - alpha) * rrf_score

        # 3. RRF 점수순으로 정렬
        ranked = sorted(
            rrf_scores.items(),
            key=lambda x: x[1],
            reverse=True
        )

        # 4. 결과 구성
        results = []
        for doc_idx, score in ranked:
            results.append({
                'doc_idx': doc_idx,
                'document': doc_map[doc_idx],
                'rrf_score': score
            })

        return results

    def search(
        self,
        query: str,
        top_k: int = 5,
        alpha: float = 0.5
    ) -> Dict:
        """
        Hybrid 검색 실행

        Returns:
            {
                'query': 쿼리,
                'vector_results': 벡터 검색 결과,
                'bm25_results': BM25 검색 결과,
                'hybrid_results': Hybrid 결과
            }
        """

        # 1. 벡터 검색 (더 많이 가져오기)
        vector_results = self.vector_searcher.search(query, top_k=top_k*2)

        # 2. BM25 검색 (더 많이 가져오기)
        bm25_results = self.bm25_searcher.search(query, top_k=top_k*2)

        # 3. RRF 융합
        hybrid_results = self.reciprocal_rank_fusion(
            vector_results,
            bm25_results,
            alpha=alpha
        )

        return {
            'query': query,
            'vector_results': vector_results[:top_k],
            'bm25_results': bm25_results[:top_k],
            'hybrid_results': hybrid_results[:top_k]
        }

    def display_comparison(self, results: Dict, employee: str = ""):
        """3가지 검색 방식 비교 표시"""
        query = results['query']

        print("\n" + "="*80)
        if employee:
            print(f"👤 {employee}")
        print(f"🔍 쿼리: '{query}'")
        print("="*80)

        # 벡터 검색

        # BM25 검색

        for i, r in enumerate(results['bm25_results'], 1):

            icon = "✅" if is_correct else "❌"

- 순위 기반 융합
- 점수 스케일 독립적
- alpha로 가중치 조절
- 업계 표준 방법
