7:15~7:30 GitHub 준비 저장소 만들기 + Antigravity 연결
7:30~7:45 회장 시연 5분 만에 홈페이지 만드는 걸 구경

Step 1: Antigravity 세팅 (7:00~7:15)

① 설치 + 로그인

  1. https://antigravity.google/download 에서 다운로드·설치
  2. 실행 → Fresh Start → 테마 선택 → Next
  3. Google 계정으로 로그인
  4. AI 사이드바(오른쪽)에 입력창이 보이면 성공 ✅

② 한글 답변 설정

  1. 우측 상단 에이전트 아이콘(사람+톱니) → Customization → Rules → + Global
  2. 아래 내용을 그대로 붙여넣고 Ctrl + S 저장:
# 기본 규칙
- 모든 답변은 한국어(한글)로 작성해.
- 코드 주석도 한국어로 작성해.
- 에러 메시지를 설명할 때 원문을 먼저 보여주고, 그 아래에 한국어 해석을 덧붙여줘.
- 설명은 프로그래밍 초보자도 이해할 수 있도록 전문 용어를 최소화해줘.
- 작업 전에 무엇을 할 건지 한 줄로 먼저 알려주고, 그 다음에 실행해줘.

③ 터미널 안전 설정

  1. 우측 상단 톱니바퀴 → Editor Settings → Antigravity Settings
  2. Terminal → Request Review 선택

⚠️ Always Proceed는 절대 선택하지 마세요! AI가 파일을 삭제할 수 있습니다.

Step 2: GitHub 저장소 만들기 (7:15~7:30)

① GitHub 로그인

https://github.com 에서 사전에 만든 계정으로 로그인

계정 없으면: Sign up → 이메일/비밀번호/사용자명 → 이메일 인증 (5분)

② 저장소 만들기

GitHub 우상단 + → New repository → 아래와 같이 설정:

Repository name:  내아이디.github.io     ← ⚠️ 본인 GitHub 아이디를 넣으세요!
                                           예: sejong-ai.github.io
Public ●                                ← 반드시 Public
☑ Add a README file                     ← 반드시 체크

→ Create repository 클릭

③ Antigravity에서 작업 폴더 열기

💡 GitHub 연결은 홈페이지를 다 만든 후에 합니다. 지금은 폴더만 열면 됩니다!

1. 바탕화면에 폴더 하나 만들기 (이름: my-homepage)
2. Antigravity → 파일 → 폴더 열기 → 그 폴더 선택

왼쪽 탐색기에 빈 폴더가 열리면 성공 ✅

💡 GitHub 연결과 push는 Step 5에서 한꺼번에 합니다. 지금은 홈페이지 만드는 데 집중하세요!

Step 3: 회장 시연 (7:30~7:45)

👀 구경만 하세요. 따라하지 마세요. 시연이 끝나면 여러분 차례입니다!

Step 4: 각자 실습 — 내 홈페이지 만들기 (7:45~8:25)

프롬프트 템플릿 — 하나를 골라서 자기 정보로 바꿔 넣으세요

💡 (여기에 입력) 부분만 바꾸면 됩니다. 카톡 단톡방에도 올려놨으니 거기서 복사하셔도 됩니다.

🅰 공무원/직장인 포트폴리오형

한국어 개인 소개 홈페이지를 만들어줘.
이 프로젝트의 루트에 index.html 하나로 만들어줘. (CSS, JS 모두 포함)

[내 정보]
- 이름: (본인 이름)
- 직책: (소속 + 직급)
- 한 줄 소개: (나를 한 문장으로)
- 경력: (주요 경력 나열)
- 관심 분야: (3~5개)
- 연락처: (이메일 또는 블로그)

[디자인 요구사항]
- 깔끔하고 신뢰감 있는 원페이지 디자인
- 다크모드/라이트모드 토글
- 반응형 (모바일 대응)
- 섹션: 히어로 → 자기소개 → 경력 타임라인 → 관심 분야 → 연락처
- 색상: 블루 계열

🅱 시민강사/교육자형

한국어 시민강사 소개 홈페이지를 만들어줘.
이 프로젝트의 루트에 index.html 하나로 만들어줘. (CSS, JS 모두 포함)

[내 정보]
- 이름: (본인 이름)
- 소개: (나를 한 문장으로)
- 전문 강의 분야: (분야 나열)
- 주요 강의 이력: (3~5개)
- 연락처: (이메일)

[디자인 요구사항]
- 따뜻하고 친근한 원페이지
- 강의 분야를 아이콘+카드로 보여주기
- 수강 후기 슬라이드 (샘플 3개)
- "강의 의뢰하기" 버튼 포함
- 반응형 + 스크롤 애니메이션
- 색상: 오렌지 + 크림 계열

🅲 취미/관심사형

한국어 개인 소개 홈페이지를 만들어줘.
이 프로젝트의 루트에 index.html 하나로 만들어줘. (CSS, JS 모두 포함)

[내 정보]
- 이름 또는 닉네임: (자유롭게)
- 한 줄 소개: (재미있게)
- 좋아하는 것들: (취미, 관심사 5~7개)
- 최근 빠져있는 것: (요즘 하는 것)
- SNS/블로그: (있으면)

[디자인 요구사항]
- 개성 있고 재미있는 원페이지
- 좋아하는 것들을 이모지+카드로
- 사진 갤러리 섹션 (플레이스홀더)
- 다크 테마 + 파티클 배경 애니메이션
- 반응형
- 색상: 보라~핑크 그라데이션

🅳 동호회/모임 소개형

한국어 동호회 소개 홈페이지를 만들어줘.
이 프로젝트의 루트에 index.html 하나로 만들어줘. (CSS, JS 모두 포함)

[동호회 정보]
- 이름: (동호회 이름)
- 슬로건: (한 줄)
- 소개: (2~3줄)
- 회원 수: (명)
- 활동 내용: (3~4개)
- 모임 일정: (요일, 시간)
- 가입 문의: (연락처)

[디자인 요구사항]
- 활기차고 전문적인 원페이지
- 성과 숫자 카운트업 애니메이션
- 활동 내용 카드 그리드
- "가입 문의하기" CTA 버튼
- 반응형
- 색상: 그린 + 화이트 계열

Step 5: GitHub Pages 배포 (8:25~8:45)

① GitHub에 연결하고 올리기

AI 사이드바에:

이 폴더를 git 저장소로 만들고,
아래 GitHub 저장소에 연결해서 모든 파일을 push해줘.
커밋 메시지는 "나만의 홈페이지 첫 배포"로 해줘.

GitHub 주소: https://github.com/내아이디/내아이디.github.io

⚠️ 내아이디를 본인 GitHub 아이디로 바꿔야 합니다!

터미널에 승인(Accept) 요청이 오면 → 승인 GitHub 로그인 팝업이 뜨면 → 로그인

② push 확인

브라우저에서 https://github.com/내아이디/내아이디.github.io 새로고침. index.html이 보이면 push 성공 ✅

⚠️ push가 안 되면 AI에게 에러 메시지를 보여주세요. 대부분 인증 문제입니다.

③ GitHub Pages 활성화 (필수!)

⚠️ 이 단계를 건너뛰면 홈페이지가 안 열립니다. 반드시 해주세요!

1. 브라우저에서 내 저장소 페이지 열기
   https://github.com/내아이디/내아이디.github.io

2. 상단 탭에서 ⚙️ Settings 클릭

3. 왼쪽 메뉴에서 Pages 클릭

4. 아래와 같이 설정:
   Source:  Deploy from a branch
   Branch:  main  /  / (root)

5. Save 클릭

④ 내 홈페이지 접속!

1~3분 기다린 후:

https://내아이디.github.io

홈페이지가 보이면 성공! 🎉

바로 안 보이면 1~3분 더 기다린 후 새로고침. GitHub Pages 배포에 시간이 걸립니다.

⑤ 단톡방에 공유!

🎉 내 홈페이지가 세상에 공개되었습니다!
👉 https://내아이디.github.io

Step 6: 마무리 (8:45~9:00)

오늘의 회고 — 단톡방에 한 줄씩 남겨주세요

질문 예시
오늘 뭘 했나요? "Antigravity로 홈페이지 만들어서 GitHub Pages에 배포"
어떤 기분이 들었나요? "이렇게 쉬운 거였어? 허탈 + 뿌듯"
새롭게 알게 된 건? "코딩 몰라도 홈페이지를 만들 수 있다"
앞으로 어떻게 활용하고 싶나요? "우리 팀 사업 소개 페이지를 만들어보고 싶다"

이후에 해볼 수 있는 것들

🔹 커스텀 도메인 연결 — 내이름.kr → GitHub Pages에 연결
🔹 콘텐츠 추가 — 블로그, 포트폴리오 갤러리
🔹 AI 챗봇 추가 — 방문자 응대 챗봇
🔹 지속 업데이트 — Antigravity에서 수정 → push → 자동 반영

⚠️ 정보화실 PC 사용한 분 — 퇴실 전 필수!

🔒 체크리스트
□ Antigravity → Google 계정 로그아웃
□ 브라우저 → GitHub 로그아웃
□ 브라우저 → 저장된 비밀번호 삭제
□ 브라우저 → 쿠키/캐시 삭제 (Ctrl+Shift+Delete)
□ Windows 자격 증명 관리자 → git 관련 항목 삭제
□ 바탕화면/다운로드 폴더 개인 파일 삭제

❓ 자주 묻는 질문

질문 답변
GitHub가 뭔가요? 코드 저장소 = 구글 드라이브. push = 업로드
홈페이지 수정하려면? Antigravity에서 수정 → AI에게 "GitHub에 다시 올려줘" → 자동 반영
URL이 영구적인가요? 네. GitHub 계정 유지하면 계속 살아있고, 무료
URL이 안 열려요! Settings → Pages 설정을 했는지 확인. 설정 후 1~3분 기다리기
내 도메인 연결 가능? 네. GitHub Pages Settings에서 커스텀 도메인 연결
코드가 영어인데요? 코드 자체는 영어가 정상. 읽을 필요 없어요
디자인이 마음에 안 들어요 "처음부터 다시 만들어줘, 이번엔 OO느낌으로"
사진 넣고 싶어요 사진 파일을 폴더에 넣고 AI에게 "이 사진 프로필에 넣어줘"

📌 오늘의 핵심: 코드를 읽을 필요 없습니다. AI에게 말로 시키세요. 마음에 안 들면 "다시 해"라고 말하면 됩니다. 이게 바이브 코딩입니다.
