현재 방식 (Bi-Encoder):

Query: [벡터 A]  Document: [벡터 B]
         ↓              ↓
      따로 인코딩됨 (상호작용 없음)
         ↓              ↓
      유사도 = 벡터 거리

문제: 단어만 비슷하면 높은 점수

"하지만 Reranking에서는," 레이나가 새로운 다이어그램을 그렸다. "질문과 문서를 함께 본다."

Reranking (Cross-Encoder):

[Query + Document]
       ↓
  함께 분석 (상호작용!)
       ↓
  진짜 관련도 점수

결과: 단어만이 아니라 의미를 이해

📚 Chapter 1: Bi-Encoder의 한계

1.1 실험: 왜 정확하지 않을까?

📖 스토리

"먼저 문제를 명확히 보자," 레이나가 말했다. "실험을 해볼까?"

준혁이 코드를 작성하기 시작했다.

from sentence_transformers import SentenceTransformer
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# Bi-Encoder 모델 로드
model = SentenceTransformer('all-MiniLM-L6-v2')

# 테스트 쿼리
query = "파이썬으로 웹 크롤링하는 방법을 알려주세요"

# 다양한 문서들
documents = [
    "파이썬 기초 문법과 자료구조를 배워봅시다",  # 단어는 비슷, 내용은 다름
    "BeautifulSoup을 이용한 웹 크롤링 완벽 가이드",  # 정답!
    "파이썬으로 데이터 분석하기 - Pandas 입문",
    "Selenium을 사용한 동적 웹 페이지 스크래핑 방법",  # 정답!
    "자바스크립트로 웹 개발 시작하기",
    "파이썬 Django 프레임워크 튜토리얼",
]

print("🔍 검색 쿼리:", query)
print("\n실험 시작...\n")

# 임베딩 생성
query_emb = model.encode(query)
doc_embs = model.encode(documents)

# 유사도 계산
similarities = cosine_similarity([query_emb], doc_embs)[0]

# 결과 정렬
results = sorted(
    zip(documents, similarities), 
    key=lambda x: x[1], 
    reverse=True
)

print("📊 Bi-Encoder 검색 결과:\n")
for i, (doc, score) in enumerate(results, 1):
    is_correct = "✅" if i <= 2 and ("크롤링" in doc or "스크래핑" in doc) else ""
    print(f"{i}위. [{score:.3f}] {is_correct}")
    print(f"    {doc}\n")

1.2 Bi-Encoder의 작동 원리

📖 스토리

"Cross-Encoder를 이해하려면, 먼저 Bi-Encoder가 정확히 뭘 하는지 알아야 해," 레이나가 설명을 시작했다.

레이나가 화이트보드에 그림을 그렸다.

Bi-Encoder (두 개의 독립적 인코더)

          Encoder 1              Encoder 2
              ↓                      ↓
Query: "웹 크롤링"  →  [0.2, 0.8, 0.1, ...]  
                            768차원 벡터

Doc: "BeautifulSoup" →  [0.3, 0.7, 0.2, ...]
                            768차원 벡터
              ↓                      ↓
            코사인 유사도 = 0.85

"Bi-Encoder는 질문과 문서를 각각 따로 인코딩해," 레이나가 설명했다. "그리고 두 벡터 사이의 거리를 계산하는 거지."

"그게 왜 문제인데요?" 준혁이 물었다.

"생각해봐. 질문이 '웹 크롤링 방법'인데, 문서가 '웹 크롤링 역사'라면?" 레이나가 물었다.

"아..." 준혁이 깨달았다. "'웹 크롤링'이라는 단어는 똑같으니까, 벡터도 비슷하게 나오겠네요."

"정확해. Bi-Encoder는 각 문장을 독립적으로 봐. 질문이 '방법'을 물었는지, 문서가 '방법'에 대한 답인지... 이런 상호작용을 전혀 보지 못하는 거야."

🎓 핵심 개념: Bi-Encoder의 한계

레이나가 화이트보드에 정리했다.

Bi-Encoder의 특징:

✅ 장점:
  - 매우 빠름 (벡터 미리 계산 가능)
  - 확장성 좋음 (100만 개도 가능)
  - 캐싱 쉬움

❌ 단점:
  - 단어 겹침에 민감
  - 의도(Intent) 파악 약함
  - 문맥 이해 부족

예시:
Query: "사과를 먹는 방법"
Doc1: "사과(Apple) 제품 사용법" → 높은 점수 (오답!)
Doc2: "과일 섭취 가이드" → 낮은 점수 (정답!)

준혁이 고개를 끄덕이며 노트에 적었다.

📚 Chapter 2: Cross-Encoder의 마법

2.1 함께 보는 힘

📖 스토리

"이제 Cross-Encoder를 보자," 레이나가 새 그림을 그렸다.

Cross-Encoder (하나의 인코더)

[CLS] Query [SEP] Document [SEP]
  ↓          ↓          ↓
        BERT 모델
  (Query와 Document 함께 분석!)
            ↓
      Attention 메커니즘
   (단어들 간 상호작용)
            ↓
      관련도 점수: 8.5

"Cross-Encoder는 질문과 문서를 하나의 입력으로 넣어," 레이나가 설명했다. "그래서 모델이 둘 사이의 관계를 직접 볼 수 있지."

"어떻게요?" 준혁이 물었다.

"BERT의 Attention 메커니즘 때문이야," 레이나가 설명했다. "질문의 '방법'이라는 단어가 문서의 '가이드', '튜토리얼' 같은 단어와 강하게 연결되는 걸 학습하지. 단순히 똑같은 단어만 찾는 게 아니야."

준혁이 이해가 되는 듯 눈을 반짝였다.

"한 번 해볼게요!" 준혁이 코드를 작성했다.

from sentence_transformers import CrossEncoder

# Cross-Encoder 모델 로드
cross_encoder = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')

print("\n🎯 Cross-Encoder Reranking 시작...\n")

# Query-Document 쌍 생성
pairs = [[query, doc] for doc in documents]

# 점수 계산
scores = cross_encoder.predict(pairs)

# 결과 정렬
reranked = sorted(
    zip(documents, scores),
    key=lambda x: x[1],
    reverse=True
)

print("📊 Cross-Encoder 결과:\n")
for i, (doc, score) in enumerate(reranked, 1):
    is_correct = "✅" if ("크롤링" in doc or "스크래핑" in doc) else ""
    print(f"{i}위. [{score:.3f}] {is_correct}")
    print(f"    {doc}\n")

2.2 속도 vs 정확도 트레이드오프

📖 스토리

레이나가 벤치마크 결과를 보여줬다.

import time

def benchmark_both(num_documents=1000):
    """두 방식 비교"""

    # 테스트 문서 생성
    test_docs = [f"문서 {i}: 내용..." for i in range(num_documents)]
    test_query = "테스트 쿼리"

    print(f"📊 벤치마크: {num_documents:,}개 문서\n")

    # Bi-Encoder
    print("1️⃣ Bi-Encoder")
    start = time.time()
    bi_embs = model.encode(test_docs, show_progress_bar=False)
    q_emb = model.encode(test_query)
    similarities = cosine_similarity([q_emb], bi_embs)[0]
    bi_time = time.time() - start
    print(f"   시간: {bi_time:.3f}초")

    # Cross-Encoder
    print("\n2️⃣ Cross-Encoder")
    start = time.time()
    pairs = [[test_query, doc] for doc in test_docs]
    scores = cross_encoder.predict(pairs, show_progress_bar=False)
    cross_time = time.time() - start
    print(f"   시간: {cross_time:.3f}초")

    # 비교
    slowdown = cross_time / bi_time
    print(f"\n⚡ Cross-Encoder가 {slowdown:.0f}배 느림")

    return bi_time, cross_time

# 실행
bi_time, cross_time = benchmark_both(num_documents=100)

from sentence_transformers import SentenceTransformer, CrossEncoder
import numpy as np
import faiss
from typing import List, Dict
import time

class TwoStageRAG:
    """Two-Stage Retrieval + Reranking 시스템"""

    def __init__(self):
        print("🚀 시스템 초기화 중...")

        # Stage 1: Bi-Encoder (빠른 검색)
        print("  📥 Bi-Encoder 로딩...")
        self.bi_encoder = SentenceTransformer('all-MiniLM-L6-v2')

        # Stage 2: Cross-Encoder (정확한 재정렬)
        print("  📥 Cross-Encoder 로딩...")
        self.cross_encoder = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')

        # FAISS 인덱스
        self.index = None
        self.documents = []

        print("✅ 초기화 완료!\n")

    def index_documents(self, documents: List[str]):
        """문서 인덱싱"""
        print(f"📚 {len(documents):,}개 문서 인덱싱 중...")

        self.documents = documents

        # 임베딩 생성
        embeddings = self.bi_encoder.encode(
            documents,
            show_progress_bar=True,
            convert_to_numpy=True
        )

        # FAISS 인덱스 생성
        dimension = embeddings.shape[1]
        self.index = faiss.IndexFlatIP(dimension)

        # 정규화 (코사인 유사도용)
        faiss.normalize_L2(embeddings)
        self.index.add(embeddings)

        print(f"✅ 인덱싱 완료!\n")

    def search(
        self,
        query: str,
        top_k_retrieval: int = 100,
        top_n_rerank: int = 5
    ) -> List[Dict]:
        """Two-Stage 검색"""

        print(f"🔍 검색: {query}\n")
        start_time = time.time()

        # ═══════════════════════════════════════
        # Stage 1: Fast Retrieval
        # ═══════════════════════════════════════
        print(f"📊 Stage 1: FAISS 검색 (Top-{top_k_retrieval})")
        stage1_start = time.time()

        # Query 임베딩
        query_embedding = self.bi_encoder.encode(
            query, 
            convert_to_numpy=True
        )
        query_embedding = query_embedding.reshape(1, -1)
        faiss.normalize_L2(query_embedding)

        # FAISS 검색
        distances, indices = self.index.search(
            query_embedding, 
            top_k_retrieval
        )

        stage1_time = time.time() - stage1_start
        print(f"  ⏱️ Stage 1 완료: {stage1_time*1000:.2f}ms")

        # 후보 추출
        candidates = []
        for i, idx in enumerate(indices[0]):
            candidates.append({
                'doc': self.documents[idx],
                'idx': int(idx),
                'bi_score': float(distances[0][i]),
                'rank_stage1': i + 1
            })

        print(f"  ✅ 후보 {len(candidates)}개 추출\n")

        # ═══════════════════════════════════════
        # Stage 2: Precise Reranking
        # ═══════════════════════════════════════
        print(f"🎯 Stage 2: Cross-Encoder Reranking (Top-{top_n_rerank})")
        stage2_start = time.time()

        # Query-Document 쌍 생성
        pairs = [[query, cand['doc']] for cand in candidates]

        # Cross-Encoder로 점수 계산
        cross_scores = self.cross_encoder.predict(pairs)

        # 점수 추가 및 재정렬
        for cand, score in zip(candidates, cross_scores):
            cand['cross_score'] = float(score)

        candidates.sort(key=lambda x: x['cross_score'], reverse=True)
        final_results = candidates[:top_n_rerank]

        stage2_time = time.time() - stage2_start
        total_time = time.time() - start_time

        print(f"  ⏱️ Stage 2 완료: {stage2_time*1000:.2f}ms")
        print(f"  ⏱️ 총 시간: {total_time*1000:.2f}ms\n")

        # 순위 변동 표시
        self._show_rank_changes(final_results)

        return final_results

    def _show_rank_changes(self, results: List[Dict]):
        """순위 변동 시각화"""
        print("📈 순위 변동:\n")

        for i, result in enumerate(results, 1):
            old_rank = result['rank_stage1']
            change = old_rank - i

            if change > 0:
                arrow = f"⬆️ +{change}"
            elif change < 0:
                arrow = f"⬇️ {change}"
            else:
                arrow = "➡️ 0"

            print(f"  {i}위 (Stage1: {old_rank}위) {arrow}")
            print(f"      Bi: {result['bi_score']:.3f} | "
                  f"Cross: {result['cross_score']:.3f}")
            print(f"      {result['doc'][:80]}...\n")

📖 스토리

🎓 핵심 개념: Precision@K

Precision@K = (상위 K개 중 정답 개수) / K

예시:
검색 결과 Top 5:
1. ✅ 정답
2. ❌ 오답
3. ✅ 정답
4. ❌ 오답
5. ✅ 정답

Precision@5 = 3/5 = 60%
Precision@3 = 2/3 = 67%
Precision@1 = 1/1 = 100%

"실전에서는 주로 Precision@5를 봐," 레이나가 설명했다. "사용자가 보통 상위 5개 결과만 보니까."

from typing import List, Set

class RAGEvaluator:
    """RAG 시스템 평가 도구"""

    def __init__(self, rag_system):
        self.rag = rag_system

    def precision_at_k(
        self,
        queries: List[str],
        ground_truth: List[Set[int]],
        k: int = 5,
        use_reranking: bool = True
    ) -> float:
        """
        Precision@K 계산

        Args:
            queries: 테스트 쿼리들
            ground_truth: 각 쿼리의 정답 문서 인덱스 Set
            k: 상위 몇 개를 볼지
            use_reranking: Reranking 사용 여부
        """
        precisions = []

        for query, truth_set in zip(queries, ground_truth):
            if use_reranking:
                # Two-Stage 검색
                results = self.rag.search(query, top_n_rerank=k)
                predicted_indices = {r['idx'] for r in results}
            else:
                # Bi-Encoder만
                query_emb = self.rag.bi_encoder.encode(query, convert_to_numpy=True)
                query_emb = query_emb.reshape(1, -1)
                faiss.normalize_L2(query_emb)

                distances, indices = self.rag.index.search(query_emb, k)
                predicted_indices = set(indices[0])

            # 정답과 비교
            relevant = len(predicted_indices & truth_set)
            precision = relevant / k
            precisions.append(precision)

        return np.mean(precisions)

    def compare_methods(
        self,
        queries: List[str],
        ground_truth: List[Set[int]]
    ):
        """Bi-Encoder vs Reranking 비교"""

        print("="*60)
        print("📊 성능 비교: Bi-Encoder vs Two-Stage Reranking")
        print("="*60 + "\n")

        k_values = [1, 3, 5, 10]

        print(f"{'K':<5} {'Bi-Encoder':<15} {'Reranking':<15} {'개선율':<10}")
        print("-" * 60)

        for k in k_values:
            # Bi-Encoder만
            p_bi = self.precision_at_k(
                queries, ground_truth, k, use_reranking=False
            )

            # Reranking 포함
            p_rerank = self.precision_at_k(
                queries, ground_truth, k, use_reranking=True
            )

            # 개선율
            improvement = ((p_rerank - p_bi) / p_bi * 100) if p_bi > 0 else 0

            print(f"{k:<5} {p_bi*100:>6.1f}%{'':<8} {p_rerank*100:>6.1f}%{'':<8} "
                  f"{'+' if improvement >= 0 else ''}{improvement:>5.1f}%")

        print("\n" + "="*60)

5.1 Cross-Encoder는 여전히 느리다

📖 스토리

🎓 핵심 개념: Late Interaction

일반 Bi-Encoder:
Query → [단일 벡터]      완전히 독립적
Doc → [단일 벡터]        상호작용 없음

Cross-Encoder:
[Query + Doc] → 함께 처리   느림!

ColBERT (Late Interaction):
Query → [토큰별 벡터들]
Doc → [토큰별 벡터들]
         ↓
    MaxSim 연산으로 상호작용
         ↓
    빠르면서도 정확!

"핵심은 Late Interaction이야," 레이나가 설명했다. "Query와 Doc을 따로 인코딩하되, 각 토큰별로 벡터를 만드는 거지. 그리고 나중에 토큰끼리 매칭하는 거야."

준혁이 고개를 갸우뚱했다. "토큰별로요?"

"예를 들어볼게," 레이나가 예시를 들었다.

Query: "파이썬 웹 크롤링"
  ↓
토큰별 벡터:
  "파이썬" → v1
  "웹"     → v2
  "크롤링" → v3

Document: "BeautifulSoup으로 웹 스크래핑"
  ↓
토큰별 벡터:
  "BeautifulSoup" → d1
  "으로"          → d2
  "웹"            → d3
  "스크래핑"      → d4

MaxSim 계산:
  v1("파이썬") vs [d1, d2, d3, d4] → 최대 유사도
  v2("웹")     vs [d1, d2, d3, d4] → 최대 유사도 (d3와 정확히 일치!)
  v3("크롤링") vs [d1, d2, d3, d4] → 최대 유사도 (d4 "스크래핑"과 유사)

최종 점수 = 각 Query 토큰의 최대 유사도 합

import torch
from transformers import AutoTokenizer, AutoModel

class SimpleColBERT:
    """간단한 ColBERT 구현"""

    def __init__(self, model_name: str = 'bert-base-uncased'):
        print("🚀 ColBERT 초기화...")

        self.tokenizer = AutoTokenizer.from_pretrained(model_name)
        self.model = AutoModel.from_pretrained(model_name)
        self.model.eval()

        print("✅ 초기화 완료!\n")

    def encode(self, text: str) -> torch.Tensor:
        """
        텍스트를 토큰별 벡터로 인코딩

        Returns:
            [num_tokens, hidden_dim] 텐서
        """
        # 토큰화
        inputs = self.tokenizer(
            text,
            return_tensors='pt',
            padding=True,
            truncation=True,
            max_length=512
        )

        # BERT 인코딩
        with torch.no_grad():
            outputs = self.model(**inputs)
            # [batch, seq_len, hidden_dim]
            embeddings = outputs.last_hidden_state[0]

        # 정규화
        embeddings = torch.nn.functional.normalize(
            embeddings, p=2, dim=1
        )

        return embeddings

    def maxsim(
        self,
        query_vectors: torch.Tensor,
        doc_vectors: torch.Tensor
    ) -> float:
        """
        MaxSim 점수 계산

        Args:
            query_vectors: [Q_tokens, dim]
            doc_vectors: [D_tokens, dim]
        """
        # [Q_tokens, D_tokens] 유사도 행렬
        sim_matrix = torch.matmul(query_vectors, doc_vectors.T)

        # 각 Query 토큰의 최대 유사도
        max_sims = sim_matrix.max(dim=1)[0]

        # 합산
        score = max_sims.sum().item()

        return score

    def rerank(
        self,
        query: str,
        documents: List[str]
    ) -> List[tuple]:
        """ColBERT로 재정렬"""

        print(f"🎯 ColBERT Reranking: {query}\n")

        # Query 인코딩
        query_vecs = self.encode(query)

        # 각 문서 점수 계산
        scores = []
        for i, doc in enumerate(documents):
            doc_vecs = self.encode(doc)
            score = self.maxsim(query_vecs, doc_vecs)
            scores.append(score)
            print(f"  문서 {i+1}: {score:.3f}")

        # 정렬
        ranked = sorted(
            zip(documents, scores),
            key=lambda x: x[1],
            reverse=True
        )

        return ranked

from typing import Dict, List
import numpy as np
from datetime import datetime

class HybridRanker:
    """여러 점수를 결합하는 Hybrid Ranker"""

    def __init__(self):
        # 가중치 (합=1)
        self.weights = {
            'semantic': 0.5,   # 의미적 유사도
            'keyword': 0.3,    # 키워드 매칭
            'recency': 0.1,    # 최신성
            'popularity': 0.1  # 인기도
        }

    def bm25_score(self, query: str, doc: str) -> float:
        """
        BM25: 키워드 기반 점수

        실제로는 rank_bm25 라이브러리 사용 권장
        여기서는 간단한 버전
        """
        query_terms = set(query.lower().split())
        doc_terms = doc.lower().split()

        # 단어 빈도
        term_freq = {}
        for term in doc_terms:
            term_freq[term] = term_freq.get(term, 0) + 1

        # BM25 간소화
        score = 0.0
        for term in query_terms:
            if term in term_freq:
                tf = term_freq[term]
                # BM25 공식 간소화
                score += (tf * 2.2) / (tf + 1.2)

        return score

    def recency_score(self, doc_metadata: Dict) -> float:
        """문서 최신성 점수"""
        created_date = doc_metadata.get('created_date')

        if not created_date:
            return 0.5  # 날짜 없으면 중간 점수

        # 날짜 차이 계산
        now = datetime.now()
        days_old = (now - created_date).days

        # 최근일수록 높은 점수 (6개월 기준)
        if days_old < 30:
            return 1.0
        elif days_old < 90:
            return 0.8
        elif days_old < 180:
            return 0.6
        elif days_old < 365:
            return 0.4
        else:
            return 0.2

    def popularity_score(self, doc_metadata: Dict) -> float:
        """문서 인기도 점수"""
        views = doc_metadata.get('views', 0)

        # 로그 스케일로 정규화
        if views == 0:
            return 0.0

        import math
        score = min(math.log10(views + 1) / 4, 1.0)
        return score

    def hybrid_rank(
        self,
        query: str,
        candidates: List[Dict]
    ) -> List[Dict]:
        """
        여러 신호를 결합하여 최종 점수 계산

        candidates 형식:
        [
            {
                'doc': "문서 내용",
                'cross_score': 8.5,
                'metadata': {
                    'created_date': datetime(...),
                    'views': 1234
                }
            },
            ...
        ]
        """
        print("🎯 Hybrid Ranking 시작...\n")

        for cand in candidates:
            # 1. 의미적 유사도 (이미 계산됨)
            semantic = cand.get('cross_score', 0)
            semantic_norm = min(semantic / 10, 1.0)  # 0-10 → 0-1

            # 2. 키워드 점수
            keyword = self.bm25_score(query, cand['doc'])
            keyword_norm = min(keyword / 10, 1.0)  # 정규화

            # 3. 최신성
            recency = self.recency_score(cand.get('metadata', {}))

            # 4. 인기도
            popularity = self.popularity_score(cand.get('metadata', {}))

            # 가중 합
            final_score = (
                self.weights['semantic'] * semantic_norm +
                self.weights['keyword'] * keyword_norm +
                self.weights['recency'] * recency +
                self.weights['popularity'] * popularity
            )

            cand['hybrid_score'] = final_score
            cand['score_breakdown'] = {
                'semantic': semantic_norm,
                'keyword': keyword_norm,
                'recency': recency,
                'popularity': popularity
            }

        # 최종 점수로 정렬
        candidates.sort(key=lambda x: x['hybrid_score'], reverse=True)

        return candidates

    def display_results(self, results: List[Dict], top_n: int = 5):
        """결과 예쁘게 출력"""
        print(f"📊 최종 Hybrid Ranking (Top-{top_n}):\n")

        for i, result in enumerate(results[:top_n], 1):
            print(f"{i}위. [최종: {result['hybrid_score']:.3f}]")
            print(f"    {result['doc'][:80]}...")

            breakdown = result['score_breakdown']
            print(f"    📈 의미: {breakdown['semantic']:.2f} | "
                  f"키워드: {breakdown['keyword']:.2f} | "
                  f"최신: {breakdown['recency']:.2f} | "
                  f"인기: {breakdown['popularity']:.2f}\n")

class CostAwareReranker:
    """비용을 고려한 Reranking 시스템"""

    def __init__(self, cross_encoder, cost_per_call=0.0001):
        self.cross_encoder = cross_encoder
        self.cost_per_call = cost_per_call  # 예시

        self.stats = {
            'total_calls': 0,
            'skipped_calls': 0,
            'total_cost': 0.0
        }

    def smart_rerank(
        self,
        query: str,
        candidates: List[Dict],
        top_n: int = 5,
        confidence_threshold: float = 0.7
    ) -> List[Dict]:
        """
        똑똑한 Reranking

        전략:
        1. Bi-Encoder 점수가 이미 높으면 Reranking 스킵
        2. 애매한 것들만 Reranking
        3. 비용 절감!
        """
        print(f"\n🎯 Smart Reranking: {query}")

        # Bi-Encoder 점수로 정렬
        candidates.sort(key=lambda x: x['bi_score'], reverse=True)

        # 확실한 상위/하위 분류
        certain_top = []
        uncertain = []
        certain_bottom = []

        for cand in candidates:
            score = cand['bi_score']

            if score > confidence_threshold:
                certain_top.append(cand)  # 확실히 좋음
            elif score < 0.3:
                certain_bottom.append(cand)  # 확실히 나쁨
            else:
                uncertain.append(cand)  # 애매함 → Reranking 필요

        print(f"  ✅ 확실히 좋음: {len(certain_top)}개 (스킵)")
        print(f"  ❓ 애매함: {len(uncertain)}개 (Reranking)")
        print(f"  ❌ 확실히 나쁨: {len(certain_bottom)}개 (제외)")

        # 애매한 것들만 Reranking
        if uncertain:
            pairs = [[query, c['doc']] for c in uncertain]
            scores = self.cross_encoder.predict(pairs, show_progress_bar=False)

            for cand, score in zip(uncertain, scores):
                cand['cross_score'] = float(score)

            # 통계 업데이트
            self.stats['total_calls'] += len(uncertain)
            self.stats['total_cost'] += len(uncertain) * self.cost_per_call
        else:
            print("  💰 Reranking 완전 스킵!")

        self.stats['skipped_calls'] += len(certain_top) + len(certain_bottom)

        # 결과 조합
        uncertain.sort(key=lambda x: x.get('cross_score', 0), reverse=True)
        final = certain_top + uncertain

        print(f"  💰 비용 절감: {len(certain_top) + len(certain_bottom)}개 스킵\n")

        return final[:top_n]

    def get_cost_report(self) -> str:
        """비용 리포트"""
        total_docs = self.stats['total_calls'] + self.stats['skipped_calls']
        saved_cost = self.stats['skipped_calls'] * self.cost_per_call

        return f"""
💰 비용 리포트:
  전체 문서: {total_docs:,}개
  Reranking 수행: {self.stats['total_calls']:,}개
  Reranking 스킵: {self.stats['skipped_calls']:,}개

  실제 비용: ${self.stats['total_cost']:.4f}
  절약 비용: ${saved_cost:.4f}
  절감률: {saved_cost/(saved_cost+self.stats['total_cost'])*100:.1f}%
