Ec2 Aws 리눅스입문 인스턴스 리눅스

EC2 서비스

EC2(Elastic Compute Cloud)는 아마존 웹 서비스(AWS)에서 제공하는 가상 서버로, 클라우드 환경에서 컴퓨팅 자원을 쉽게 생성하고 관리하는 도구입니다. 즉, AWS에서 제공하는 가상 머신이라고 생각하면 이해하기 쉽습니다. EC2는 사용량에 따라 인스턴스를 자유롭게 생성, 중지, 시작, 종료할 수 있으며, 목적에 따라 CPU, 메모리, GPU 등 최적화된 자원을 구성할 수 있습니다. EC2 서비스는 AWS의 콘솔 홈에서 [모든 서비스]  [EC2] 또는 검색 창에서 “EC2” 키워드로 검색해 찾을 수 있습니다.


EC2 서비스 찾기

EC2 인스턴스 생성하기

AWS 콘솔 홈에서 EC2 서비스를 찾아 클릭하면 그림과 같이 EC2 인스턴스를 생성할 수 있는 초기 화면이 나타납니다. EC2 인스턴스를 생성하려면 초기 화면의 [인스턴스 시작] 버튼이나 왼쪽 메뉴에서 [대시보드], [인스턴스] 메뉴를 선택해 [인스턴스 시작] 버튼을 클릭하면 생성할 수 있습니다.


EC2 인스턴스 시작 버튼

EC2 인스턴스 시작 버튼을 눌러 화면을 이동하면 인스턴스를 생성하기 위한 기본 정보를 입력하는 화면으로 이동합니다. 먼저, 인스턴스의 이름을 결정하고, 애플리케이션 및 OS 이미지를 선택합니다. OS 이미지는 AMI(Amaznon Machine Image)를 의미하며 EC2 인스턴스를 생성하기 위한 템플릿 디스크 이미지입니다. AMI에는 운영체제와 함께 애플리케이션, 설정, 데이터 등이 포함된 형태로 제공되기 때문에 목적에 맞는 이미지를 선택하면 사용하기 편리하게 제공되고 있습니다.


EC2 인스턴스 시작을 위한 정보 입력 화면

인스턴스의 이름을 “아마존 리눅스”로 설정하고, AMI는 맨 앞에 있는 [Amazon Linux]를 선택 후 화면 스크롤을 통해 아래로 이동합니다. AMI에서 [Ubuntu]를 선택하면 우분투 리눅스가 설치된 가상 환경이 만들어집니다. 인스턴스 유형은 CPU, 메모리, 네트워크 성능, 스토리지 용량 등에 따라 분류된 서버 사양을 의미하며, 추가적인 과금을 하지 않기 위해 프리 티어 사용 가능 항목을 선택하면 됩니다. 키 페어는 EC2 인스턴스에 안전하게 로그인하기 위한 인증 수단으로 별도의 비밀번호를 입력하지 않고, 미리 생성한 키 페어를 통해 로그인할 수 있습니다. 키 페어를 생성하면 공개 키(Public key)와 비밀 키(Private key)가 만들어지고 공개 키는 AWS에 저장됩니다. 비밀 키는 여러분 컴퓨터에 저장해 추후에 EC2 인스턴스에 접속할 때 사용하게 됩니다. 초기에는 키 페어가 없기 때문에 [새 키 페어 생성] 항목을 클릭해 키 페어를 만들어야 합니다. 그리고 비밀 키를 분실하거나 삭제할 경우 EC2 인스턴스에 접속할 수 없고, 유출될 경우 보안 사고가 발생할 수 있기때문에 관리에 유의하도록 합니다.


EC2 인스턴스 유형 및 키 페어

키 페어 이름은 공백 없이 영문과 숫자로 작성하면 됩니다. 키 페어 유형은 키를 저장하는 암호화 방식을 의미하며, 기본값인 RSA를 사용하도록 하겠습니다. 비밀 키 파일 형식은 두 가지가 제공되는데 .pem은 OpenSSH를 활용한 원격 접속에 사용하므로 리눅스, macOS, 윈도우 운영체제 등에서 활용 가능하며, .ppk는 윈도우 운영체제의 PuTTY 프로그램에서 주로 사용하는 포맷입니다. 따라서 확장성이 높은 .pem 파일을 사용하도록 하겠습니다. [키 페어 생성] 버튼을 클릭하면 비밀 키가 [키 페이 이름].pem 파일로 저장되는데 이 파일은 추후에 가상 머신에 접속할 때 사용합니다.


키 페어 생성을 위한 정보 입력

다음 단계는 네트워크 설정입니다. 퍼블릭 IP 자동 활당 부분이 활성화되어 있어야 외부 네트워크에서도 가상 머신에 접속할 수 있기 때문에 정상적으로 설정되어있는지 확인합니다. 보안 그룹은 인스턴스에 대한 트래픽을 제어하는 방화벽 규칙을 의미하며, 클라우드의 네트워크 환경을 구성하기 위해 접속을 허용하거나 거부할 IP 주소 및 포트를 설정하는 항목입니다. 원격으로 가상 머신에 접속하기 위해 [다음에서 SSH 트래픽 허용] 항목이 체크되어 있어야 합니다.


EC2 인스턴스의 네트워크 설정

마지막으로 스토리지 구성에서는 프리티어에서 사용할 수 있는 기본 스토리지 값을 선택하고, 화면 우측 하단의 [인스턴스 시작] 버튼을 클릭합니다.


EC2 인스턴스의 네트워크 설정

[인스턴스 시작] 버튼을 누르면 인스턴스를 시작하며, 현재 모든 인스턴스를 보기 위해 [모든 인스턴스 보기] 버튼을 클릭합니다. 만약 인스턴스가 보이지 않는다면 화면 상단 새로고침 버튼을 클릭하면 생성한 인스턴스가 나타납니다. 인스턴스를 초기화하는데 다소 시간이 소요되기 때문에 가상 머신을 사용할 준비가 된 경우 상태 검사에 표시된 문구가 [초기화]에서 [2/2개 검사 통과]로 표시될 때까지 기다려야 합니다.
