3-2. VBA 전환 프롬프트 ②

역할(Role):
당신은 VBA(Visual Basic for Applications) 프로그래밍 전문가입니다. 특히 PowerPoint 자동화 작업에 능숙합니다.

문맥(Context):
현재 PowerPoint 프레젠테이션 파일(ActivePresentation) 안에서, 슬라이드에 포함된 텍스트나 표(Table) 안에 존재하는 특정 변수(placeholder)를 미리 지정한 실제 값으로 자동 교체하려고 합니다.

요구사항(Instructions):

기본 기능

슬라이드를 하나씩 반복하면서 각 슬라이드 안에 있는 모든 Shape를 순회합니다.

Shape가 텍스트를 가지고 있으면, 미리 정의한 키워드 목록에 따라 텍스트를 일괄 치환합니다.

Shape가 표(Table) 형태라면, 표 안의 각 Cell의 텍스트도 동일한 방법으로 치환합니다.

치환할 변수 목록

{과정명}, {일시}, {장소}, {목적}, {모듈1}, {모듈1 내용}, {모듈2}, {모듈2 내용}, {모듈3}, {모듈3 내용}, {모듈4}, {모듈4 내용}, {모듈5}, {모듈5 내용}, {과정 소요시간}, {강사명}, {강사이력}, {특징1}, {특징2}, {특징3} 등을 포함합니다.

변수 목록은 VBA Dictionary(Scripting.Dictionary)를 사용하여 관리합니다.

Dictionary에 키와 대응하는 실제 값을 추가하는 코드를 포함해 주세요.

구현 방식 세부 지침

ActivePresentation.Slides를 사용해 모든 슬라이드를 대상으로 반복합니다.

각 슬라이드의 모든 Shape를 확인합니다.

HasTextFrame이 True이고 TextFrame.HasText가 True인 경우, 텍스트 내용을 확인한 후 치환합니다.

HasTable이 True인 경우, 표의 모든 Cell을 순회하며 텍스트를 확인하고 치환합니다.

치환할 때는 Replace 함수를 사용하여 변수 키를 실제 값으로 바꿔주세요.

텍스트를 치환한 후에는 해당 내용을 다시 TextRange.Text로 업데이트합니다.

최종 알림 기능

모든 치환 작업이 끝나면 MsgBox를 띄워 "모든 변수 값이 성공적으로 대체되었습니다."라는 메시지를 보여주세요.

코딩 스타일 및 가독성

가급적 주석(Comment)을 충분히 추가해서 코드의 각 단계가 어떤 역할을 하는지 쉽게 이해할 수 있도록 작성해 주세요.

중첩 루프나 조건문을 사용할 때 들여쓰기를 명확히 하여 가독성을 높여주세요.

출력 형식(Output Format):

하나의 Sub 프로시저로 완성된 VBA 매크로 코드를 작성합니다.

모듈(Module)에 바로 붙여넣기 해서 사용할 수 있도록 완전한 형태로 작성해 주세요.

추가 요청(Optional):

오류 처리를 간단하게 추가해 주셔도 좋습니다(예: 슬라이드나 Shape가 없는 경우 대비).

표 안에서도 텍스트 치환이 적용되도록 세심하게 신경 써 주세요.

기타 특이사항:

PowerPoint VBA 환경을 기준으로 작성합니다.

외부 라이브러리는 사용하지 않습니다(단, Scripting.Dictionary는 CreateObject를 통해 사용).

    MsgBox "모든 변수 값이 성공적으로 대체되었습니다.", vbInformation, "교체 완료"

End Sub
