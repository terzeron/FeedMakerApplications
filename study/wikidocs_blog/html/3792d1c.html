macOS 환경에서 EC2 인스턴스 접속

AWS의 EC2에 접속하기 위해 퍼블릭 IP 주소를 알아야 합니다. EC2의 퍼블릭 IP 주소를 확인하기 위해 실행중인 EC2 인스턴스를 클릭하면 화면 아래쪽에 세부 정보를 확인할 수 있고, 세부 정보 중 퍼블릭 IPv4 주소가 표시되는 것을 알 수 있습니다. 이 값을 복사해 사용하면 됩니다. 복사하는 방법은 IP 주소 앞 겹쳐진 사각형 아이콘을 클릭하면 클립보드에 자동 복사됩니다.


퍼블릭 IP 주소 확인

EC2에 접속하는 클라이언트 환경에 상관 없이 공통적으로 다음과 같은 명령어 형식으로 가상 머신에 원격 접속할 수 있습니다. SSH(Secure Shell)는 원격 컴퓨터에 안전하게 접속하기 위한 통신 프로토콜로 macOS에서는 ssh 명령어를 사용해 원격 접속할 수 있습니다. -i 옵션은 키 페어를 통해 접속할 때 사용하는 옵션으로 키 페어가 저장된 경로를 뒤에 입력하면 됩니다. 그리고 가상 머신에 접속하기 위해 사용자 계정과 EC2의 퍼블릭 IP 주소를 @ 기호로 구분해 작성하면 됩니다. 

ssh -i [키페어경로.pem] [사용자계정]@[EC2의 퍼블릭 IP 주소]

EC2 인스턴스를 생성할 때 별도의 사용자 계정을 생성하지 않았지만 AMI는 사용자 계정도 포함한 이미지 파일입니다. 따라서 AMI를 선택하는 순간 이미 지정된 사용자 계정명을 사용할 수 있습니다. 자주 사용하는 AMI에 대한 사용자 계정명을 정리하면 다음 표와 같습니다.


AMI에 따른 주요 사용자 계정명

macOS에서 ssh 명령어는 터미널 프로그램에서 입력할 수 있습니다. 터미널 프로그램은 설치할 필요 없이 내장된 프로그램입니다. 운영체제 우측 상단 돋보기 아이콘을 클릭하면 빠른 검색 기능인 스포트라이트(Spotlight)가 실행되고, 여기에 “terminal” 키워드로 터미널 프로그램을 빠르게 실행할 수 있습니다.


스포트라이트로 터미널 프로그램 실행

터미널 프로그램에서 ssh 명령어를 통해 클라우드 환경에 있는 EC2 가상 머신에 접속해보겠습니다. 하지만 정상적으로 원격 접속 명령어를 입력해도 그림과 같은 오류가 발생합니다. 이는 비밀 키의 접근 권한이 너무 과하게 허용되어있기 때문입니다. 따라서 비밀 키에 대한 접근 권한을 파일 소유자만 읽을 수 있게 수정한 뒤 다시 접속해야 합니다.


비밀 키의 접근 권한 문제로 원격 접속 오류 발생

접근 권한을 수정할 수 있는 chmod 명령어로 비밀 키의 권한을 사용자만 읽기 권한을 허용하고 다시 접속 시도하면 이번엔 정상적으로 접속되는 것을 볼 수 있습니다. 아마존 리눅스는 레드햇 계열 리눅스 배포판이기 때문에 로키와 동일한 명령어 체계를 갖습니다. 따라서 원격 접속 후 학습했던 리눅스 명령어를 입력하면 정상적으로 동작합니다.

윈도우 환경에서 EC2 인스턴스 접속

윈도우에서 ssh 명령어를 사용하려면 명령 프롬프트를 실행해야 합니다. 윈도우 운영체제 왼쪽 하단 검색 창에서 “명령” 키워드만 입력하면 명령 프롬프트 앱을 찾을 수 있고, 이것을 클릭해 실행합니다.


명령 프롬프트 실행

명령 프롬프트에 형식에 맞게 ssh 명령어를 입력하면 macOS 환경과 동일한 문제로 원격 EC2에 접속하지 못하는 문제가 발생합니다.


비밀 키 권한 문제로 오류 발생

이 문제를 해결하기 위해 다운로드 받은 비밀 키 파일(.pem)을 찾아 선택 후 오른쪽 마우스 버튼을 클릭합니다. 그리고 [속성] 메뉴를 선택 후 속성창이 나타나면 상단 [보안] 탭을 누릅니다.


비밀 키 파일의 속성 변경

비밀 키 파일의 모든 권한을 제거하기 위해 왼쪽 아래에 있는 [상속 사용 안함] 버튼을 클릭 후 나타나는 팝업 창에서 [이 개체에서 상속된 사용 권한을 모두 제거합니다] 항목을 클릭합니다.


비밀 키 파일의 모든 권한 제거

특정 사용자에게만 읽고 실행할 수 있는 권한을 추가하기 위해 [보안 주체 선택] 항목을 선택하고, 개체 유형은 [사용자] 선택, 선택할 개체 이름에 윈도우 운영체제에 로그인한 사용자명을 입력 후 [확인] 버튼을 클릭합니다.


비밀 키 파일의 보안 주체 선택

사용자의 기본 권한에 [읽기 및 실행], [읽기] 권한에 체크하고 나머지는 체크 해제합니다. 그리고 [확인] 버튼을 클릭하면 비밀 키 파일의 보안 속성에 사용 권한이 읽기 및 실행과 읽기에만 허용되어있는 것을 볼 수 있습니다.


사용자에게 읽기 및 실행 권한 부여

사용자에게만 읽기 및 실행 권한을 부여했기 때문에 다시 ssh 명령어로 접속 시도하면 정상적으로 EC2 인스턴스에 접속하는 것을 볼 수 있습니다. EC2 인스턴스에 접속 후 기본적인 리눅스 명령어 등을 실습해 봅시다.

EC2 인스턴스 중지 및 종료

EC2 인스턴스를 사용하지 않을 때는 중지나 종료를 통해 명시적으로 서비스를 멈추는 것이 좋습니다. 그렇지 않으면 프리티어 기간이 만료된 후 의도치 않게 과금될 수 있기 때문입니다. 인스턴스 중지는 가상 머신의 전원을 차단하는 것과 동일합니다. 따라서 인스턴스 중지만으로도 추가 과금을 방지할 수 있습니다. 하지만 EC2 인스턴스 외에 스토리지(EBS)나 다른 서비스를 함께 사용중이라면 EC2 인스턴스를 중지하더라도 서비스가 활성화되어있는 경우 과금이 될 수 있습니다. 따라서 특정 서비스를 활성화할 경우 과금 여부를 반드시 확인해야 합니다. EC2 인스턴스 종료는 가상 머신을 삭제하는 개념입니다. 따라서 더이상 해당 인스턴스를 사용하지 않을 경우 종료를 통해 데이터를 삭제합니다. 한 번 삭제한 인스턴스는 다시 복구할 수 없기 때문에 인스턴스를 종료할 때 주의하도록 합니다. EC2 인스턴스의 중지와 종료는 AWS 서비스에서 [EC2]  [인스턴스] 메뉴를 선택해 중지 또는 종료하려는 인스턴스를 클릭 후 [인스턴스 상태] 메뉴에서 결정할 수 있습니다.
