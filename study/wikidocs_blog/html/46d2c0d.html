FastAPI (2/3) : 컨트롤러 확장하기

@mattabu

Python

지난 글에서는 아주 간단한 FastAPI 프로그램을 만들어보았습니다. 앞으로 많은 API들을 추가해서 넣을텐데요. API를 추가할 때마다 app.py 파일이 중구난방으로 계속 커지겠지요? 코드를 보기가 너무 복잡해져서 유지보수하는데 어려움이 예상됩니다. 이 때문에, 기능에 따라서 잘 분류해서 파일을 분리하는 것 좋습니다. 일반적인 Webframework 에서는 보통 컨트롤러(Controller)라는 이름으로 분리합니다. FastAPI 에서는 라우터(router)라고 표현하고 있지만, 저는 좀 더 보편적인 controller 라는 이름을 선호하므로, 여기에서도 컨트롤러를 사용할게요.

기존에 작성했던 코드(app.py)에서 /items API가 이었는데, 이것을 별도의 컨트롤러로 분리를 해봅시다. C:\s\fapi 폴더 아래에 controller 폴더를 만들어 줍니다. 그리고, conroller 폴더 안에 items.py 을 만들어줍니다.

# controller/items.py

from fastapi import APIRouter
from typing import Union

router = APIRouter(
    prefix="/items",
    tags=["items"],
    responses={404: {"description": "Not found"}},
)

@router.get("/{item_id}")
def read_item(item_id: int, q: Union[str, None] = None):
    return {"item_id": item_id, "q": q}

app.py 는 아래처럼 수정해줍니다.

# app.py

from fastapi import FastAPI
from controller import items

app = FastAPI()

app.include_router(items.router)

@app.get("/")
def read_root():
    return {"Hello": "World"}

웹브라우저에서 http://localhost:8080/items/5?q=what 이라고 입력해봅니다. 기존과 같은 결과가 나왔다면 잘 된 것입니다.

이번에는 users 컨트롤러를 추가로 만들어볼까요? controller/users.py 파일을 만들어 주고,

# controller/users.py

from fastapi import APIRouter

router = APIRouter(
    prefix="/users",
    tags=["users"],
    responses={404: {"description": "Not found"}},
)

@router.get("/{user_id}")
def read_user(user_id: int):
    return {"user_id": user_id}

app.py 을 수정해줍니다.

# app.py

from fastapi import FastAPI
from controller import items, users

app = FastAPI()

app.include_router(items.router)
app.include_router(users.router)

@app.get("/")
def read_root():
    return {"Hello": "World"}

그리고, 웹브라우저에서 http://localhost:8080/users/10 이라고 입력하면, 아래와 같이 나옵니다.
