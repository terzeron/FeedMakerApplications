File Inclusion란

웹 애플리케이션에서 외부 파일을 로드하거나 실행할 때 발생하는 보안 취약점으로,
공격자가 악의적인 파일을 포함시켜 악성 코드를 실행하거나 민감한 정보를 유출시킬 수 있습니다.

이는 주로 개발자가 PHP와 같은 서버사이드스크립트 언어에 파일 경로를 안전하게 검증하지 않을 때 발생합니다.

예를들어 include() 함수만 사용하면 입력값에 대한 적절한 필터링이 없어 취약점에 노출됩니다.

File Inclusion 종류

파일의 위치에 따라 크게 2가지로 구분됩니다.

1. Local file inclusion (LFI): 서버에 있는 로컬 파일을 포함시키는 취약점
예: 서버에 있는 /etc/passwd 를 읽는 경우가 이에 해당합니다.

2. Remote file inclusion (RFI): 외부 URL에서 파일을 포함시키는 취약점
예: 악성 코드를 포함한 스크립트를 원격 서버에서 가져와 실행

DVWA 실습

실습 환경의 경우 리눅스 환경에서 DVWA 환경을 구축한 후 가장 취약한 환경인 Low 레벨 환경에서 실습을 진행하였습니다.


(참고) 해당 실습을 위해서는 세팅에서 allow_url_include enable 설정이 필요합니다.

LFI 실습

세팅 완료했으면, 다시 File Inclusion탭에 들어가 실습을 진행합시다.

file1, file2 들어가 보니, URL에 어떤 경로에 접속하여 페이지가 보이는지 유추가 가능하네요

  1. file1 들어가면, file1.php에서 가져옴

  2. file2 들어가면, file2.php에서 가져옴(대충 패턴이 보임)

그러면, etc/passwd 들어가면 그거도 가져오는지 해봅시다.
etc/passwd가 노출된것을 확인할수 있습니다.

이렇게 서버안에 있는 로컬파일을 가져오게 만드는 방법을 LFI(Local File Inclusion)이라고 합니다.

이런 식으로 ../ 를 6번 이상 작성하여, etc/passwd를 확인하는 방법도 있습니다.
이렇게 디렉토리 구조를 거슬러 올라가면서 원하는 파일에 접근하는 공격 기법을 path 트래버셜 공격이라고합니다.

(참고) 대부분의 웹 애플리케이션의 디렉토리 구조가 root까지 최대 6단계 정도의 깊이를 가지고 있어 ../를 6번 이상 작성

RFI 실습

이런식으로 google.com을 입력하면, 접속할수 있는것을 볼수있는데
이를 악용하면 외부에서 호스팅된 악성 스크립트나 파일을 웹 애플리케이션에서 실행시킬 수 있습니다.
이걸 RFI(Remote File Inclusion)라고 합니다.

(참고) PHP 웹 서버 하나 띄워서 RFI 실습하는것도 좋아보입니다.

대응방안

File Inclusion은 입력값에 대한 검증을 제대로 수행하지 않아 발생하므로,
주로 입력값 검증에 대한 조치가 대부분입니다.

1. 사용자 입력 검증
파일 이름 또는 경로를 화이트리스트로 제한하여 입력값을 검증합니다.

2. 절대 경로 사용
파일 경로를 하드코딩하고 사용자 입력을 허용하지 않음.

3. PHP 설정 강화
allow_url_fopen 및 allow_url_include를 비활성화하여, RFI를 방지합니다.

3. 파일 확장자 제한
* 허용된 파일만 포함되도록 설정.
* file inclusion 공격이 많이 발생되는 애플리케이션인 php를 기준으로php.ini 파일에서 allow_url_fopen, allow_url_include, display_errors 설정을 모두 Off 로 설정하여 RFI 공격이 불가능 하도록 설정한다.
* 파일 이름을 입력 받아 include 하는 경우입력값을 검증할 수 있는 로직(White List 방식 또는 Black List 방식)을 구현하여 반드시 필요한 파일만 include 할 수 있도록 구현한다.
