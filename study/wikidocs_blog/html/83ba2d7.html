WSL2 - Windows 개발 환경 구축

@uncleyoo
2025년 10월 9일 12:58 오후
Wsl2 Windows

저는 최근 Visual Studio Code 기반으로 만들어진 AI Agent를 지원하는 IDE(Cursor, Windsurf, Visual Studio Code)를 사용하고 있습니다. 물론 Android Studio도 사용하고 있습니다만 점차 사용빈도를 줄여가고 있습니다. 주된 이유 중 하나는 웹개발을 하고 있기 때문일지도 모릅니다. 사족은 이만 줄이면서 최근 저의 작업환경이 잠시 Windows로 넘어왔기 때문에 개발환경을 설정해야 했고, 이 과정에서 있었던 일들을 글로써 공유하고자 합니다.

나의 개발 환경

  • Windows 11 Home edition
  • RAM 16 GB
  • 내장 그래픽 사용, 외장 그래픽카드 없음.

WSL2, 그거 참 계륵이다.

Docker가 세상에 나오면서 가상화라는게 예전처럼 느리다 라고 느끼는 사람들은 이제 없을 것입니다. AWS부터 수도 없이 많은 Cloud 서비스가 가상화를 통해 충분한 성능을 제공하고 있으니까요. 그런데 이상하게 저는 싫었습니다. 메모리를 점유하고 CPU의 자원을 할애해야 한다고 합니다. Windows를 깔았는데 Linux를 또 깔으라니요?

이 마음을 극복하는게 시작이었습니다.

WSL 설치를 시작합시다.

PowerShell을 '관리자 권한'으로 실행 하고 WSL을 설치합니다. 너무나 간단하게 명령어 하나로 설치가 끝납니다. 예전에는 레지스트리 마법사처럼 업데이트 설정에서 선택하는 번거로움이 사라졌습니다. 

wsl --install -d Ubuntu

이 작업이 끝난 뒤에는 Microsoft Store에서 WSL Ubuntu를 설치합니다.

Docker와 매우 유사한 이 작업은 다음의 플로우로 이해하세요.

  1. Windows에서 WSL 기능을 활성화
  2. WSL로 구동할 Ubuntu Image를 다운로드 받고 WSL에 등록, 이제 Windows 시작시 자동으로 구동됩니다.

Visual Studio Code에 연결하기

왼쪽 하단에 있는 '><' 버튼을 클릭하세요.

기존에 SSH를 통해 원격 워크스페이스를 관리하는 방법을 그대로 사용하고 있기 때문에 WSL의 연결과정이랄게 새로울게 없습니다. 로컬에서 구동되는 컨테이너에 접속하기 때문에 성능적 제한도 없습니다. 말그대로 쾌적한 Linux 환경에서의 개발을 경험 할 수 있는 겁니다. (Linux에서 Wine을 설치하는 것보다 100% 만족스럽습니다.)

WSL을 꼭 사용해야 하는가?

WSL을 사용하지 않아도 괜찮습니다. Windows에서는 안정적으로 구동되는 개발툴이 워낙 많습니다. 오히려 macOS나 Linux에서 구동되는지를 확인하는게 더 빠르겠죠. 그 정도로 호환성이 높습니다. 저처럼 Node의 특정 라이브러리를 사용해야 할 일이 아니라면 Path 문제도 사실상 없다고 봐야죠.

Android Studio와 같은 IntelliJ 계열은 WSL을 지원하는 플러그인이 있지만 Visual Studio Code와 같이 직관적이지 못합니다. 스마트폰 기기나 에뮬레이터를 활용하기 위해서는 결국 WSL에도 설정을 함께 해줘야 하는데 이게 또 여간 귀찮은게 아닙니다. 저 같은 경우에는 결국 Windows와 WSL에 각각 환경을 구축해야 했을 정도니까요.

그럼에도 불구하고 WSL을 사용해야 하는 이유는?

결국 구동 방식은 Docker나 Virtual Machine과 유사합니다. 철저하게 분리된 Sandbox에서 내가 원하는대로 다양한 솔루션들을 구동시키고 연결 할 수 있습니다. 그리고 삭제나 관리도 간편하죠. WSL에 연결되는 Docker Desktop은 엄청난 시너지를 제공합니다.

프론트 엔드의 경우 엔터프라이즈급 서비스를 제공하려면 버전에 대해서 엄격한 관리가 중요합니다. 이런 점에 동일한 개발환경을 구축 할 수 있다는 장점이 있습니다.

마무리하며

리눅스를 사용하려면 몇가지 포기해야 할게 있습니다. Windows, macOS도 각각의 플랫폼을 사용하기 위해서 일부를 포기해야 합니다. 다시 말하면 각각의 OS에서 추구하는 것들을 한곳에서 사용하기 위한 노력의 하나입니다. 저처럼 거부감이 초기에 있으셨던 분들도 꾸욱 참아보시고 다시 환경을 구축해보고 사용해보시면 어떨까요?
