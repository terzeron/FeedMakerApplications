[LLM 기본 및 자연어 처리]

  1. LLM(Large Language Model)의 정의와 현재 주된 역할로 가장 적절한 것은?
    ① 이미지 내의 객체를 식별하고 경계 상자를 그리는 역할
    ② 수치 데이터를 분석하여 미래의 주가 지수를 산출하는 역할
    ③ 언어 패턴을 학습하여 자연스러운 언어를 생성하는 역할
    ④ 로봇의 움직임을 제어하기 위한 물리적 피드백 계산
    ⑤ 네트워크 트래픽을 분석하여 보안 위협을 차단하는 역할

  2. 현재 대부분의 최신 LLM을 구성하는 핵심적인 신경망 구조는 무엇인가?
    ① CNN (Convolutional Neural Network)
    ② RNN (Recurrent Neural Network)
    ③ LSTM (Long Short-Term Memory)
    ④ Transformer 구조
    ⑤ Decision Tree (의사결정나무)

  3. 기계 학습의 컨셉 중 '개와 고양이 사진을 입력하여 각각의 레이블을 판별'하는 방식은 무엇인가?
    ① 회귀 (Regression)
    ② 분류 (Classification)
    ③ 군집화 (Clustering)
    ④ 차원 축소 (Dimensionality Reduction)
    ⑤ 생성 (Generation)

  4. '한 달간의 주가 정보'를 입력하여 '오늘의 종가'를 맞추는 학습의 주된 목적은?
    ① 새로운 주식 종목의 이름 생성
    ② 과거 데이터를 기반으로 한 내일의 값 예측
    ③ 주식 시장의 비정상 거래 패턴 탐지
    ④ 기업 로고 이미지의 해상도 개선
    ⑤ 뉴스 기사의 핵심 키워드 추출

  5. 데이터 분포를 이해하기 위해 데이터들 사이를 지나는 최적의 선(회귀선)을 긋는 모델링의 목표는?
    ① 데이터의 개수를 최대한 늘리는 것
    ② 모델의 파라미터를 0으로 만드는 것
    ③ 실제 데이터와 예측 선 사이의 오차를 최소화하는 것
    ④ 모든 데이터를 선 위에 완벽하게 일치시키는 것
    ⑤ 데이터의 색상과 모양을 변경하는 것

  6. '예측(Prediction)'과 '생성(Generation)'의 차이점에 대한 설명으로 옳은 것은?
    ① 예측은 새로운 데이터를 만드는 것이고, 생성은 기존 값을 맞추는 것이다.
    ② 생성은 기존 데이터 분포를 무시하고 무작위로 결과물을 낸다.
    ③ 예측은 기존 분포를 이해하여 값을 추론하는 것이고, 생성은 그 분포를 기반으로 새로운 데이터를 만드는 것이다.
    ④ 생성 모델은 오차를 계산할 필요가 없다는 점에서 예측 모델과 다르다.
    ⑤ 두 개념은 기술적으로 완전히 동일하며 혼용해서 사용해도 무방하다.

  7. LLM이 선형 회귀 모델보다 압도적으로 많은 파라미터(수억~수천억 개)를 사용하는 근본적인 이유는?
    ① 컴퓨터의 메모리 성능이 남기 때문에
    ② 단순한 연산을 더 빠르게 수행하기 위해
    ③ 언어의 복잡한 패턴과 맥락, 유창성을 정교하게 표현하기 위해
    ④ 데이터의 저장 용량을 압축하기 위해
    ⑤ 사람이 직접 파라미터를 입력하기 편하게 하기 위해

  8. Qwen 3 모델 사례를 볼 때, LLM 학습에 사용되는 텍스트 데이터의 규모(Token/용량)는 대략 어느 정도인가?
    ① 약 100MB (책 한 권 분량)
    ② 약 1GB (뉴스 기사 수천 개 분량)
    ③ 약 100GB (일반적인 하드 드라이브 용량)
    ④ 수십 TB (테라바이트) 규모의 방대한 데이터
    ⑤ 데이터의 양은 모델 성능과 아무런 상관이 없음

  9. 생성 AI의 동작 원리를 가장 잘 요약한 문장은?
    ① 규칙을 외워서 정답을 골라내는 과정
    ② 데이터의 패턴을 이해하고 이를 바탕으로 새로운 데이터를 만드는 과정
    ③ 입력된 문장을 다른 언어로 사전식 치환하는 과정
    ④ 단순히 인터넷의 정보를 검색하여 링크를 제공하는 과정
    ⑤ 잘못된 데이터를 삭제하여 깨끗한 데이터셋을 만드는 과정

  10. 트랜스포머(Transformer) 구조로부터 파생되어 현재 LLM 열풍의 시작점이 된 모델의 명칭은?
    ① ResNet
    ② BERT
    ③ GPT (Generative Pre-trained Transformer)
    ④ AlphaGo
    ⑤ YOLO

  11. 자연어 처리에서 '임베딩(Embedding)'이 반드시 필요한 이유는 무엇인가?
    ① 텍스트를 이미지로 변환하기 위해
    ② 인간이 읽기 편하도록 글꼴을 바꾸기 위해
    ③ 컴퓨터가 이해할 수 있는 숫자(벡터) 형태로 변환하기 위해
    ④ 문장의 길이를 강제로 줄이기 위해
    ⑤ 오타를 자동으로 수정하기 위해

  12. 기계 학습 모델 내부에서 연산이 이루어지는 과정에 포함되지 않는 것은? (교재 4페이지 기준)
    ① 가중치와의 곱 연산
    ② 결과값의 합 연산
    ③ 활성화 함수(Activation Function) 적용
    ④ 물리적인 문서 출력
    ⑤ 오차를 줄이기 위한 가중치 업데이트

  13. LLM의 '유창성'은 어디에서 오는가?
    ① 사람이 직접 작성한 수만 개의 템플릿
    ② 방대한 데이터 속에 숨겨진 언어적 통계 패턴 학습
    ③ 특정 단어의 빈도수만 계산하는 단순 알고리즘
    ④ 인터넷 백과사전의 문장을 그대로 복사하는 능력
    ⑤ CPU의 클럭 속도 향상

  14. 교재 목차상 '1. 자연어 처리와 임베딩' 다음에 이어지는 핵심 구조와 진행 방향으로 언급되지 않은 것은?
    ① Transformer 구조와 원리 소개
    ② GPT의 시작과 발전 과정 탐구
    ③ 최신 LLM 발전 과정 탐구
    ④ 객체 지향 프로그래밍 문법 강의
    ⑤ 데이터 분포와 패턴의 이해

  15. 선형 모델($y=ax+b$)과 비교했을 때, LLM 모델의 '가중치(Weight)'에 해당하는 개념은?
    ① 데이터의 개수
    ② 입력되는 문장의 길이
    ③ 모델 내부에 저장된 수많은 파라미터
    ④ 사용자의 질문(Prompt) 길이
    ⑤ 결과물이 출력되는 속도

  16. 다음 중 생성 모델(Generative Model)의 예시로 가장 적절한 것은?
    ① 스팸 메일인지 아닌지 분류하는 모델
    ② 사진 속 인물이 누구인지 맞추는 모델
    ③ 주어진 주제에 대해 새로운 에세이를 작성하는 모델
    ④ 내일의 기온이 몇 도인지 수치로 맞추는 모델
    ⑤ 엑셀 시트에서 중복된 데이터를 찾아 삭제하는 모델

  17. 교재에서 설명하는 '데이터 분포'의 의미는?
    ① 데이터를 외부 서버로 전송하는 방식
    ② 데이터가 공간상에서 형성하는 통계적인 형태나 패턴
    ③ 데이터를 가나다순으로 정렬하는 것
    ④ 데이터의 보안 등급을 나누는 것
    ⑤ 데이터를 하드디스크의 여러 구역에 나누어 저장하는 것

  18. LLM이 학습하는 약 37TB~100TB 규모의 데이터는 실제 어느 정도의 정보량인가?
    ① 스마트폰 사진 10장 분량
    ② 개인용 PC의 문서 폴더 1개 분량
    ③ 전 세계 도서관의 상당 부분 혹은 인터넷상의 방대한 지식 집합
    ④ 백과사전 1권 분량
    ⑤ 한 사람이 평생 읽을 수 있는 책의 양보다 적음

  19. LLM 학습 과정에서 '패턴 학습'이란 결국 무엇을 의미하는가?
    ① 단어를 무조건 가나다순으로 배열하는 법을 배우는 것
    ② 문맥상 다음에 올 단어의 확률 분포를 익히는 것
    ③ 프로그래밍 코드를 실행하여 오류를 찾는 것
    ④ 이미지의 픽셀 색상을 반전시키는 법을 배우는 것
    ⑤ 하드웨어의 전력 소모를 줄이는 법을 익히는 것

  20. 'Transformer' 모델이 기존 RNN 모델과 차별화되는 가장 큰 특징은?
    ① 데이터를 한 번에 하나씩만 처리한다.
    ② 텍스트를 숫자로 바꾸지 않고 처리한다.
    ③ 문장 내 단어 간의 관계를 병렬적으로 한꺼번에 처리할 수 있다.
    ④ 오직 이미지 처리 전용으로 설계되었다.
    ⑤ 학습 데이터가 전혀 필요 없는 구조이다.
