Claude, 이제 내 컴퓨터를 직접 조작한다: Computer Use와 Dispatch가 바꿀 업무 자동화의 미래

@jaehong
2026년 3월 25일 8:23 오전
Claude

2026년 3월 23일, Anthropic이 Claude Cowork과 Claude Code에 컴퓨터 직접 제어 기능을 정식 탑재했다. 마우스를 움직이고, 클릭하고, 화면을 탐색하며 작업을 수행하는 이른바 'Computer Use'가 리서치 프리뷰로 공개된 것이다. 여기에 지난주 출시된 Dispatch까지 결합하면, 출근길 지하철에서 스마트폰으로 Claude에게 업무를 지시하고 사무실 데스크톱에서 완성된 결과물을 확인하는 워크플로가 현실이 된다. API 연동도, 별도 설정도 필요 없다. 그냥 화면 위에서 사람처럼 일한다.

도구가 없으면 직접 화면을 쓴다

Claude의 컴퓨터 제어 방식은 단순한 매크로 자동화와 근본적으로 다르다. 핵심 설계 원칙은 "가장 정밀한 도구부터 시도한다"는 것이다. Slack이나 Google Calendar처럼 전용 커넥터가 있으면 먼저 그걸 쓴다. 커넥터가 없는 애플리케이션이라면 그때 브라우저, 마우스, 키보드를 직접 제어하며 작업을 수행한다. 파일을 열고, 개발 도구를 실행하고, 웹 브라우저에서 정보를 탐색하는 일을 사람이 하는 것과 동일한 방식으로 처리한다.

이 접근법이 의미 있는 이유는 확장성에 있다. 기존 RPA(Robotic Process Automation)나 자동화 도구들은 대상 애플리케이션마다 별도의 통합 작업이 필요했다. Zapier든 n8n이든, 새로운 서비스를 연결하려면 API 키를 발급받고 워크플로를 설계해야 한다. Claude의 Computer Use는 이 병목을 우회한다. GUI가 있는 모든 애플리케이션이 잠재적 자동화 대상이 되는 셈이다.

물론 트레이드오프는 명확하다. Anthropic도 인정하듯, 화면 기반 조작은 직접 API 연동보다 느리다. 복잡한 작업은 재시도가 필요할 수 있고, 코드 작성이나 텍스트 처리에 비하면 아직 초기 단계다. 하지만 "모든 앱에 API가 있는 건 아니다"라는 현실을 생각하면, 이 기능의 실용적 가치는 분명하다.

Dispatch: 스마트폰이 리모컨이 된다

Computer Use 단독으로도 흥미롭지만, 진짜 시나리오는 Dispatch와의 조합에서 나온다. Dispatch는 스마트폰에서 데스크톱의 Claude와 하나의 연속된 대화를 유지하는 기능이다. 모바일에서 작업을 지시하면 Claude가 데스크톱에서 실행하고, 나중에 데스크톱을 열었을 때 완성된 결과를 확인할 수 있다.

Anthropic이 제시하는 활용 사례를 보면 방향성이 뚜렷하다. "매일 아침 이메일을 자동으로 정리해서 브리핑 만들기", "매주 특정 지표를 뽑아서 정리하기", "IDE에서 코드 수정하고 테스트 돌린 뒤 PR 올리기" 같은 것들이다. 이건 단발성 질의응답이 아니라 지속적인 업무 위임이다.

개발자 관점에서 특히 주목할 부분은 Claude Code와의 통합이다. 기존에도 Claude Code는 터미널에서 코드를 작성하고 파일을 편집하는 강력한 도구였다. 여기에 Computer Use가 더해지면, IDE의 GUI 기능(디버거 설정, 빌드 도구 조작, 브라우저 테스트 확인 등)까지 Claude가 직접 다룰 수 있게 된다. 코드 작성부터 테스트, PR 생성까지 전체 개발 루프를 하나의 에이전트가 처리하는 그림이 구체화되고 있다.

안전장치 설계: 신중한 출발

Anthropic은 이 기능의 보안 리스크를 상당히 의식하고 있다. 프롬프트 인젝션(악의적 입력으로 AI 행동을 조작하는 공격)을 자동으로 탐지하는 시스템을 모델 활성화 단계에서 작동시킨다고 밝혔다. 사용자가 언제든 Claude를 중단할 수 있고, 새로운 애플리케이션에 접근할 때는 반드시 명시적 허가를 요청한다. 일부 앱은 기본적으로 접근이 차단되어 있다.

이 접근은 현실적이다. 컴퓨터를 직접 제어하는 AI가 민감한 데이터에 무분별하게 접근하거나, 악의적으로 조작되는 시나리오는 당연히 우려 대상이다. Anthropic이 "신뢰할 수 있는 앱부터 시작하고 민감한 데이터는 피하라"고 권고하는 것도 같은 맥락이다. 기능을 먼저 내놓되, 위험을 투명하게 공개하는 전략은 Claude Cowork 초기 출시 때와 동일한 패턴이다.

현재 macOS에서만 지원되고, 데스크톱 앱이 실행 중이어야 한다는 제약도 있다. Pro와 Max 구독자만 사용 가능하다는 점에서, Anthropic이 통제된 환경에서 피드백을 수집하려는 의도가 읽힌다.

업계 맥락에서 본 의미

Computer Use 자체는 Anthropic이 2024년부터 꾸준히 발전시켜 온 기능이다. 2024년 10월 처음 API로 공개했을 때는 연구 데모 수준이었지만, 이제 일반 사용자 제품에 탑재될 만큼 안정화된 것이다. 이 흐름은 AI 에이전트 시장 전체의 방향과 맞닿아 있다. OpenAI의 Operator, Google의 Project Mariner 등 주요 플레이어들이 모두 "AI가 컴퓨터를 직접 쓰는" 기능을 경쟁적으로 내놓고 있다.

차별점은 통합 경험에 있다. 대부분의 경쟁 제품이 별도의 에이전트 환경을 만드는 반면, Anthropic은 기존 Claude 대화 인터페이스 안에서 Computer Use, Dispatch, Cowork, Code를 하나의 연속된 경험으로 엮고 있다. 사용자가 새로운 도구를 배울 필요 없이, 기존 대화 흐름 그대로 업무를 위임할 수 있다는 점에서 진입 장벽이 낮다.

실무적으로 이 기능이 당장 모든 업무를 대체하지는 않는다. 복잡한 UI 조작은 여전히 실수가 발생할 수 있고, 속도도 사람의 직접 조작보다 느릴 수 있다. 하지만 반복적이고 정형화된 작업, 특히 "매일 같은 사이트 들어가서 같은 데이터 뽑는" 종류의 일에서는 즉각적인 효용이 있다. 그리고 이런 종류의 업무가 실제 근무 시간의 상당 부분을 차지한다는 점에서, 생산성 임팩트는 결코 작지 않다.

AI 에이전트가 "대화 상대"에서 "업무 대행자"로 진화하는 전환점이 빠르게 다가오고 있다. Anthropic은 이번 릴리스로 그 전환의 구체적인 형태를 보여줬다.
