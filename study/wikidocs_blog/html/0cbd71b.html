맥 Zsh 환경에서 Claude CLI 단축어(alias) 설정하기 — 처음부터 실행까지 전체 과정

환경
- macOS
- /bin/zsh
- oh-my-zsh + powerlevel10k 사용 중
- Claude CLI가 이미 설치되어 있다고 가정

1. 현재 쉘 확인

먼저 현재 사용 중인 쉘이 zsh인지 확인합니다.

echo $SHELL
````

출력 예시:

```bash
/bin/zsh

zsh라면 .zshrc 파일을 수정하면 됩니다.

bash라면 아래 명령어를 참고하면됩니다. 내용은 동일입니다.

# 명령어
nano ~/.bashrc

## 내부 코드 수정 맨 밑에 추가
# Claude shortcut
alias ccc='claude --dangerously-skip-permissions'

# 적용 후 명령어
source ~/.bashrc

2. .zshrc 파일 열기

zsh 설정 파일을 엽니다.

nano ~/.zshrc

  • nano는 터미널 기반 편집기입니다.

  • 파일 맨 아래로 이동해서 alias를 추가합니다.

3. Claude CLI alias 추가

파일 맨 아래에 다음 내용을 추가합니다.

# ---------------------
# Custom Aliases
# ---------------------
alias ccc='claude --dangerously-skip-permissions'

의미 설명

  • ccc → 내가 입력할 짧은 명령어

  • claude --dangerously-skip-permissions → 실제 실행될 긴 명령어

  • 즉 앞으로 터미널에서 ccc만 입력하면 Claude CLI가 해당 옵션과 함께 실행됩니다.

4. 저장하고 종료하기 (nano 기준)

nano에서:

  • Ctrl + O → 저장 (WriteOut)

  • Enter → 파일명 확인 후 저장

  • Ctrl + X → 종료

5. 설정 적용하기

파일을 수정했으면 현재 세션에 반영해야 합니다.

source ~/.zshrc

이 명령은 새로운 터미널을 열지 않고도 설정을 즉시 반영합니다.

6. alias가 정상 등록되었는지 확인

alias | grep ccc

정상 출력 예:

alias ccc='claude --dangerously-skip-permissions'

참고

  • 출력이 없으면 아직 등록되지 않은 상태입니다.

  • powerlevel10k 사용 시 grep이 매칭을 못 찾으면 exit code 1로 표시될 수 있습니다.
    → 에러라기보다 “검색 결과 없음”입니다.

7. claude CLI 경로 확인 (선택 사항)

alias는 등록됐지만 claude 자체가 PATH에 있어야 실행됩니다.

which claude

출력 예:

/opt/homebrew/bin/claude

경로가 출력되면 정상입니다.

8. 실제 실행해보기

이제 단축어를 실행합니다.

ccc

또는 옵션 확인:

ccc --help

정상이라면 Claude CLI가 --dangerously-skip-permissions 옵션과 함께 실행됩니다.

9. 보안 주의 사항

--dangerously-skip-permissions 옵션은:

  • 파일 접근 제한을 우회할 수 있음

  • 실수로 중요한 파일을 수정할 위험 존재

자동화 실험용 또는 로컬 개발 환경에서만 사용하는 것을 권장합니다.

10. 전체 흐름 요약

echo $SHELL
nano ~/.zshrc
# alias 추가
source ~/.zshrc
alias | grep ccc
which claude
ccc

이 과정을 통해:

  • zsh 환경에서

  • Claude CLI를

  • ccc라는 단축 명령으로

  • 영구적으로 사용할 수 있게 됩니다.

결론

alias는 단순한 기능이지만,
AI CLI 워크플로우, 자동화 실험, 로컬 LLM 환경 구성에서
생산성을 크게 높여주는 핵심 도구입니다.

작은 단축어 하나가
작업 흐름 전체를 바꿉니다.
