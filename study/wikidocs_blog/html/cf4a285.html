우분투에서 MySQL 설치 및 설정: 데이터베이스 배포를 위한 단계별 가이드

#Ubuntu 서버에서 MySQL 설치하기

Ubuntu 서버에 MySQL을 설정하는 것은 생각보다 간단합니다. 적절한 리소스와 서버 환경이 있다면 더욱 쉽습니다. 저는 최근에 Vultr 서버를 이용해 MySQL을 웹 프로젝트를 위해 설정했으며, 여기에서 단계별로 설명하겠습니다. 이 가이드는 같은 작업을 준비 중이신 분들에게 도움이 될 것입니다.

왜 최적의 서버를 선택해야 할까요?

저는 Vultr를 선택한 이유는 서버 배포가 간단하며, 데이터베이스 호스팅에 신뢰성이 높기 때문입니다. 데이터베이스를 활용하는 애플리케이션을 작업 중이라면 Vultr 서버를 이용해 최적화된 클라우드 호스팅 환경을 경험할 수 있습니다.

설정 중 참고한 Vultr 공식 가이드는 다음과 같습니다:

  • MySQL 관리형 데이터베이스 가이드 - MySQL 데이터베이스를 손쉽게 관리하는 방법
  • SSH를 사용하여 VPS에 액세스 - 안전하게 VPS에 SSH로 접속하는 방법
  • 우분투 20.04에 MySQL 설치 - MySQL 설치 가이드

준비 사항

설치를 시작하기 전에 다음 준비 사항이 필요합니다:

  1. Vultr 서버에서 Ubuntu 설치 - Vultr 사이트를 통해 간단히 서버를 설정할 수 있습니다.
  2. SSH를 통해 서버 접근 가능해야 함
  3. 터미널 명령어에 대한 기본적인 이해

단계 1: 서버 업데이트

먼저 서버를 최신 상태로 유지하는 것이 중요합니다. 다음 명령어를 실행했습니다: 

sudo apt update && sudo apt upgrade -y

이 명령어를 실행하면 서버를 최신 버전으로 업데이트할 수 있습니다.

단계 2: MySQL 설치

서버가 준비되었으므로 MySQL을 설치합니다: 

sudo apt install mysql-server -y

설치는 간단하며 빠르게 완료됩니다.

단계 3: MySQL 서비스 시작

설치가 완료되었다면, MySQL 서비스를 시작합니다: 

sudo systemctl start mysql
sudo systemctl enable mysql

이 명령어는 MySQL 서비스를 시작하며, 서버 부팅 시 자동으로 재시작되도록 설정합니다.

단계 4: MySQL 보안 설정

이 단계는 데이터베이스 보안을 위해 반드시 필요합니다. 다음 명령어를 실행합니다: 

sudo mysql_secure_installation

이 과정을 통해 MySQL 루트 비밀번호를 설정하고, 테스트 데이터베이스를 제거하며, 원격 접근을 제한합니다.
MySQL 보안 설정 가이드

단계 5: MySQL 로그인

MySQL에 로그인합니다: 

sudo mysql -u root -p

이 명령어를 실행하면 비밀번호를 입력하라는 메시지가 뜹니다.

단계 6: 데이터베이스 및 사용자 생성

이제 애플리케이션에서 사용할 데이터베이스와 사용자를 설정합니다: 

CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';
CREATE DATABASE mydb;
GRANT ALL PRIVILEGES ON mydb.* TO 'username'@'localhost';
FLUSH PRIVILEGES;
EXIT;
  • 'username', 'password', 'mydb'는 원하는 값으로 변경합니다.

단계 7: 데이터베이스 테스트

생성한 계정으로 MySQL에 접속하여 데이터베이스 설정이 잘 작동하는지 확인합니다: 

mysql -u username -p

로그인에 성공하면 모든 설정이 정상입니다.

자주 발생하는 문제와 해결 방법

1. MySQL 서비스가 시작되지 않는 경우

  • 상태 확인:
    bash sudo systemctl status mysql
  • 로그 확인:
    bash sudo journalctl -xe | grep mysql

2. 권한 오류

  • 명령어 실행 시 sudo 권한이 있는지 확인합니다.

왜 Vultr가 MySQL에 최적인 선택인가?

저는 Vultr를 1년 넘게 사용 중이며, MySQL 데이터베이스 호스팅에 매우 만족하고 있습니다.

  1. 신뢰할 수 있는 클라우드 인프라
    Vultr 서버를 이용하면 데이터베이스 호스팅이 원활합니다.

  2. 간편한 서버 설정
    MySQL 및 Ubuntu 설정이 최적화된 이미지를 통해 손쉽게 배포 가능

  3. 확장 가능한 성능
    작은 리소스에서 고성능 데이터베이스까지 유연하게 선택할 수 있습니다.

  4. 자동 백업 기능
    Vultr의 자동 백업 기능을 통해 MySQL 데이터 손실 위험을 방지할 수 있습니다.

마무리

Vultr를 통해 Ubuntu 서버에서 MySQL을 설치하는 과정은 매우 간단하며, 안정적이면서도 최적화된 성능 덕분에 프로젝트 작업이 수월했습니다.
