Shamiko를 이용한 루팅 우회

1. Shamiko란?

Magisk의 Zygisk 기반 모듈로, 루팅 탐지를 우회하는 도구.
LSPosed 팀에서 만들었으며 MagiskHide가 deprecated된 이후 대안으로 많이 사용됨.

2. 동작 원리

앱이 루팅 탐지 시도
        ↓
Zygisk가 앱 프로세스에 개입
        ↓
Shamiko가 Magisk 흔적 숨김
        ↓
앱은 루팅 안 된 것처럼 인식

핵심은 Denylist에 등록된 앱에 대해 Magisk 관련 파일, 프로세스, 마운트 정보를 숨겨주는 것.

3. 설치 방법

선수 조건: Zygisk 활성화

Magisk 앱 → 설정 → Zygisk → ON → 재부팅

Shamiko 다운로드 및 설치

https://github.com/LSPosed/LSPosed.github.io/releases

→ shamiko-vX.X.X-release.zip 다운로드

Magisk → 모듈 → 저장소에서 설치 → shamiko zip 선택 → 재부팅

4. Denylist 설정 (핵심)

Shamiko는 Magisk의 Denylist에 등록된 앱에만 루팅 우회를 적용함.

4-1. Denylist 활성화

Magisk → 설정 → Zygisk Denylist 활성화 ON

4-2. 루팅 탐지 앱 등록

Magisk → 설정 → Configure DenyList
→ 탐지 우회할 앱 선택 (체크)
→ 앱 이름 클릭하면 하위 프로세스도 선택 가능 (모두 체크 권장)

Magisk 자체 앱은 Denylist에 넣으면 안 됨

5. 동작 모드

모드 설명
Whitelist 모드 (기본) Denylist에 등록된 앱에만 숨김 적용
Blacklist 모드 모든 앱에 숨김 적용, Denylist는 예외 처리

Blacklist 모드 전환 (필요 시)

adb shell su -c "touch /data/adb/shamiko/whitelist"
adb reboot

6. 루팅 앱 자체 숨기기

Shamiko 외에 Magisk 앱 자체도 숨겨야 탐지 우회가 더 완벽해짐.

Magisk → 설정 → Magisk 앱 숨기기
→ 임의의 앱 이름으로 변경 (패키지명도 변경됨)

7. 동작 확인

루팅 탐지 우회 확인용 테스트 앱:

앱 설명
RootBeer 기본 루팅 탐지 테스트
YASNAC Play Integrity 체크
Native Detector Native 레벨 탐지 테스트

8. Play Integrity 실패 시 추가 모듈

Shamiko만으로는 Play Integrity (강화된 무결성) 우회가 안 될 수 있음.

모듈 역할
PlayIntegrityFix Play Integrity Basic/Device 우회
TrickyStore 강화된 무결성 우회 보조

설치 방법은 Shamiko와 동일 (Magisk 모듈로 설치).

9. 한계

탐지 방식 Shamiko 우회 여부
파일 기반 탐지 (/su, /magisk) ✅ 우회 가능
프로세스 기반 탐지 ✅ 우회 가능
Play Integrity Basic ✅ 우회 가능
Play Integrity Strong ❌ 추가 모듈 필요
Frida 탐지 ❌ 별도 우회 필요

요약 순서

1. Magisk → Zygisk ON → 재부팅
2. Shamiko 모듈 설치 → 재부팅
3. Magisk → Denylist 활성화
4. Denylist에 탐지 앱 등록 (하위 프로세스 포함)
5. Magisk 앱 숨기기 (선택)
6. 재부팅 후 테스트 앱으로 확인
