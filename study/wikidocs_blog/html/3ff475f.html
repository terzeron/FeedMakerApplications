🤖 4. 자동조치 Runbook (핵심 시나리오)

🚨 시나리오 1: CPU 과부하

[이벤트]
CPU > 90%

    ↓

[AI Agent 판단]
- 트래픽 증가 vs 오류 분석

    ↓

[조치]
1. Pod scale-out
2. 로그 분석
3. 티켓 생성

    ↓

[결과]
자동 복구 + 기록

🚨 시나리오 2: 서비스 장애

[이벤트]
서비스 응답 없음

    ↓

[AI Agent]
- Health check
- 최근 배포 확인

    ↓

[조치]
1. Pod 재시작
2. 롤백
3. 알림

🚨 시나리오 3: 사용자 문의 자동화

사용자: "로그인 안됨"

    ↓

AI Agent:
- 원인 분석 (계정 잠김)
- 해결 방법 안내
- 필요시 티켓 생성

☸️ 5. Kubernetes 연동 (핵심 코드 구조)

from kubernetes import client, config

def restart_pod(namespace, pod_name):
config.load_incluster_config()
v1 = client.CoreV1Api()
v1.delete_namespaced_pod(pod_name, namespace)
return "Pod restarted"
