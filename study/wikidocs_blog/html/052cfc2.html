Hand Gesture Recognition (Video) — NuGet 패키지 프로젝트 #2

@gellston
2026년 1월 3일 11:27 오전

비교해보자

아카이브를 뒤져보면서, 3D convolution을 설계에 비교적 적극적으로 사용하는 모델들만(그리고 제 기준에서 구현 난이도가 너무 높지 않은 것들만) 대충 추려서 비교해봤습니다.

…그런데 결론이 조금 씁쓸하네요.

쩝… 추려지고 추려진 녀석이, 하필 회사에서 이미 써봤던 모델이라니…

  • 논문: https://arxiv.org/pdf/1904.02422

이 논문의 재미있는 점은, 기존에 유명한 2D 컨볼루션 모델들을 3D 컨볼루션 구조로 변환해서 Video Classification 성능 비교를 시도했다는 점입니다.
게다가 논문 한 편 안에서 여러 모델을 구현해 비교하다 보니, 이 논문 안에서도 “어떤 모델을 선택할지” 고민해야 합니다.

사내 프로젝트에서 사용했던 모델은 아래 구조였습니다.

사내에서는 적당한 성능 + 적당한 속도라는 이유로 MobileNetV2의 3D 버전을 가져다 썼었는데요.

음… 그런데 토이 프로젝트에서까지 똑같은 걸 하면 재미가 없죠.
이 프로젝트의 취지에도 잘 안 맞고요. 😅

그래서 이번에는:

이 논문에서 이미 “컨버전(변환)”된 모델을 그대로 쓰지 않고,
기존 2D 모델을 기반으로 3D 컨볼루션 구조로 “재구성(리어레인지)”하는 시도를 해보려고 합니다.

어떤 2D 컨볼루션 모델을 재구성할 것인가?

앞에서 본 Video Classification 논문은 마지막 수정이 2021년 즈음인데, 그 이후에도 Convolution 계열에서 성능을 유지하면서도 훨씬 가벼운 모델들이 계속 나왔습니다.

제가 글을 쓰면서 가장 먼저 떠올린 건 GhostNet이었어요.
예전에 한번 구현해본 기억이 있어서 레포를 뒤져보니… 다행히도 레이어 모듈이 있더군요.

  • 레포 링크: https://github.com/gellston/DeepLearningStudy/blob/main/torch/util/helper.py

제가 지금까지 공부하면서 정리해둔 모델 설계용 레이어 모듈(일종의 식재료들)이 위 링크에 모여 있습니다.

오, Ghost Module V1이 있네요.

기억을 더듬어가며 이 2D Ghost Module(V1)을 3D convolution 기반으로 확장해서 Video Classification을 시도해보면,
속도와 성능을 둘 다 잡을 가능성이 있지 않을까 생각합니다.

참고로… 제가 일에 매몰돼 있는 동안, GhostNet은 벌써 V3까지 나왔더군요. 😵‍💫
Transformer가 판치는 세상인데도, 여전히 간단한 아이디어와 Convolution만으로 경량성과 성능을 파는 연구가 이어진다는 게 인상적입니다.
