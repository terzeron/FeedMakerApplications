1. Comparing Collections

@doldoli_coding_diary
2025년 5월 2일 7:22 오후
  1. ordered - 데이터 구조 내의 요소들이 특정한 순서를 가지고 있으며, 그 순서가 유지됨.
  2. 예측 가능성 : for 문 루프 등 사용시 정해진 순서대로 요소에 접근가능
  3. 인덱싱/슬라이싱 가능
  4. immutable vs mutable : 객체가 생성된 후에 그 값을 변경할 수 있나?
    1. immutable type
    2. 객체가 생성된 후에 그 값을 직접 변경할 수 없는 데이터 타입
    3. 불변 객체를 수정하려고 시도하면, 파이썬은 새로운 객체를 생성하여 변경된 값을 할당하고, 원래 객체는 그대로 둔다.
    4. 예측 가능
    5. 해시 가능 - 딕셔너리 key, set의 element로 사용될 수 있는 조건
    6. int, float, bool, str, tuple
    7. mutable type
    8. 객체가 생성된 후에 그 값을 직접 변경할 수 있는 데이터 타입
    9. 값을 변경하면 원래 객체 자체가 수정되기에 객체의 메모리 주소는 그대로 유지된다.
    10. 예측 어려움
    11. 해시 불가 - 객체의 내용이 언제든 변할 수 있기에 일관된 해시 값 계산 X. -> 딕셔너리 key 나 set의 element 값으로 직접 사용 X
    12. list, dict, set, 사용자 정의 class 객체(달라질 수 있음)
    13. 왜 중요할까?
    14. 함수 인자 전달 : 함수에 mutable 객체를 전달하면 함수 내부에서 그 객체를 수정했을 때 함수 외부 원본 객체에도 영향을 미친다.
    15. 딕셔너리 키/set 원소 : 딕셔너리 키와 Set의 원소는 반드시 immutable 하고 hashable해야 함.
    16. mutable 객체는 내용이 같더라도 매번 새로운 객체를 생성하며, 일부 immutable 객체에 대해 캐싱(인터닝)이 가능함.
    17. 참고.
    18. 해시값 : 객체의 데이터를 바탕으로 계산된, 객체를 대표하는 고정 길이의 정수 값(객체의 지문)
    19. 해시 가능 : 객체의 수명 동안 값이 변하지 않아 일관된 해시값을 가질 수 있는 성질
    20. 딕셔너리 key, 집합의 element는 반드시 해시 가능해야하는데, 이를 통해 빠른 데이터 검색 및 중복 값 확인이 가능해진다.
    21. 해시한다 : 어떤 객체에 해시함수(알고리즘)를 적용하여 객체 고유의 해시값을 계산하는 과정
