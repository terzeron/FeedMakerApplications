이 문제는 주어진 MenOfPassion 알고리즘의 수행 시간을 분석하는 문제입니다. 입력 크기 n에 따라 몇 번 실행되는지, 그리고 그 실행 횟수를 n에 대한 다항식으로 표현했을 때 최고차항의 차수가 얼마인지를 알아내는 것이 목표입니다.

알고리즘 분석

먼저 주어진 알고리즘을 살펴보겠습니다.

MenOfPassion(A[], n) {
    i = ⌊n / 2⌋;
    return A[i]; # 코드1
}
  1. 입력으로 배열 A와 그 크기 n을 받습니다.
  2. n을 2로 나눈 몫을 구해 변수 i에 저장합니다. (여기서 ⌊ ⌋ 기호는 나머지 버림을 의미합니다.)
  3. 배열 A의 i번째 요소를 반환합니다. 이 부분이 코드1입니다.

핵심은 입력 n의 크기와 관계없이 코드1는 항상 한 번만 실행된다는것입니다.

정답 찾기

1. 코드1 수행 횟수

알고리즘을 보면 return A[i]; 라는 코드1은 n값에 상관없이 항상 1번만 실행됩니다. 예를 들어 n이 1이든, 100이든, 500,000이든, i값을 한 번 계산하고 A[i]를 한 번 반환할 뿐입니다.

  • 따라서 첫 번째 줄의 답은 항상 1입니다.

2. 최고차항의 차수

수행 횟수인 '1'을 n에 대한 다항식으로 표현해 보겠습니다.

이 다항식에서 최고차항은 n^0이며, 이 항의 차수는 0입니다. 시간 복잡도(Big-O) 표기법으로는 O(1) 이라고 표현하며, 이는 입력 크기에 영향을 받지 않는 '상수 시간' 복잡도를 의미합니다.

따라서 두 번째 줄의 답은 항상 0입니다.

예제 입력 풀이 (n=1)

  • 입력: n = 1
  • 수행 횟수: i = ⌊1 / 2⌋ = 0이 계산되고, return A[0]가 1번 실행됩니다.
  • 최고차항 차수: 수행 횟수는 1이고, 이는 n0에 해당하므로 차수는 0입니다.

  • 출력:

    1 0

결론적으로, 이 문제는 입력 n의 값에 관계없이 항상 동일한 답을 출력하게 됩니다.

코드

print(1)
print(0)

(참고) Big-O notation(빅오 표기법)

Summary

  • Big-O는 알고리즘이 얼마나 확장 가능한지(Scalable)를 보여주는 지표입니다.
  • 데이터가 작을 때는 어떤 알고리즘을 써도 큰 차이가 없지만, 대용량 데이터를 다룰수록 효율적인 알고리즘(O(1), O(log n), O(n))을 선택하는 것이 매우 중요해집니다.
  • 알고리즘을 짤 때 "내 코드가 대략 어떤 Big-O에 속할까?"를 고민하는 습관은 좋은 개발자로 성장하는 데 큰 도움이 됩니다.

Big-O 표기법이란

시간 복잡도를 나타내는 Big-O(빅오) 표기법은 입력 데이터의 크기(n)가 커질 때, 알고리즘의 실행 시간이 얼마나 오래 걸리는지를 나타내는 방법입니다.

쉽게 말해, 알고리즘의 성능과 효율성을 분석하는 척도라고 할 수 있습니다. Big-O는 실제 걸리는 시간(예: 0.02초)을 측정하는 것이 아니라, 데이터 크기 증가에 따른 실행 시간의 증가율(Growth Rate)을 표현합니다.

왜 Big-O를 사용할까요? 🤔

어떤 알고리즘이 "빠르다"고 말할 때, 기준이 애매할 수 있습니다. 컴퓨터의 성능이나 프로그래밍 언어에 따라 실제 실행 시간은 달라지기 때문입니다. Big-O 표기법은 이런 환경적인 요소를 제외하고, 데이터가 아무리 커져도 이 알고리즘의 성능이 어느 정도로 유지되는지를 객관적으로 표현해 줍니다.

가장 중요한 것은 최악의 경우(Worst-case)를 기준으로 한다는 점입니다. 운 좋게 빨리 끝나는 경우가 아니라, "아무리 운이 없어도 이 정도 시간 복잡도를 넘지는 않는다"는 것을 보장해 줍니다.

흔히 사용되는 Big-O 표기법

실행 시간 증가율이 빠른(효율적인) 순서대로 몇 가지 주요 Big-O 표기법을 알아보겠습니다.

1. O(1) - 상수 시간 (Constant Time)

  • 가장 빠르고 이상적인 성능입니다.
  • 입력 데이터 n의 크기와 상관없이 항상 일정한 시간이 걸립니다.
  • 예시: 배열의 특정 인덱스에 있는 요소 접근하기
def get_element(arr, index):
    return arr[index] # 데이터가 100만 개라도 한 번의 연산으로 끝남

2. O(log n) - 로그 시간 (Logarithmic Time)

  • 매우 효율적인 성능입니다.
  • 데이터 n이 두 배로 늘어나도, 실행 시간은 한 단계만 늘어나는 수준입니다. 검색 범위를 계속 절반씩 줄여나가는 알고리즘이 여기에 해당합니다.
  • 예시: 정렬된 리스트에서 이진 탐색 (Binary Search)

3. O(n) - 선형 시간 (Linear Time)

  • 가장 흔하게 볼 수 있는 준수한 성능입니다.
  • 데이터 n의 크기에 정비례하여 실행 시간이 늘어납니다. n이 2배가 되면 시간도 2배가 됩니다.
  • 예시: 리스트의 모든 요소를 한 번씩 확인하는 반복문
def find_value(arr, value):
    for item in arr: # arr의 길이(n)만큼 반복
        if item == value:
            return True
    return False

4. O(n log n) - 선형 로그 시간 (Linearithmic Time)

  • O(n)보다는 느리지만 O(n²)보다는 훨씬 빠릅니다. 효율적인 정렬 알고리즘들이 대부분 여기에 속합니다.
  • 예시: 병합 정렬(Merge Sort), 퀵 정렬(Quick Sort)

5. O(n²) - 이차 시간 (Quadratic Time)

  • 데이터가 조금만 커져도 성능이 급격히 나빠집니다. 되도록 피하는 것이 좋습니다.
  • n이 2배가 되면, 실행 시간은 4배(22)가 됩니다. 보통 이중 반복문을 사용할 때 나타납니다.
  • 예시: 리스트의 모든 요소 쌍을 비교하는 경우
def print_pairs(arr):
    for item_a in arr: # n번 반복
        for item_b in arr: # 바깥쪽 반복 한 번당 n번 반복
            print(item_a, item_b) # 총 n * n = n² 번 실행

6. O(2ⁿ) - 지수 시간 (Exponential Time)

  • 가장 느리고 비효율적인 유형입니다. n이 조금만 커져도 사실상 사용하기 불가능한 수준의 시간이 걸립니다.
  • 예시: 피보나치 수를 계산하는 단순한 재귀 함수
