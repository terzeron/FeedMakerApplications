SharePoint 목록을 다른 사이트로 마이그레이션하는 방법

@techtips
2026년 3월 20일 9:45 오후

SharePoint 목록을 다른 사이트로 마이그레이션하는 것은 콘텐츠 재구성, 팀 구조 변경 또는 테넌트 이전 시 자주 필요한 작업입니다. 그러나 올바르게 수행하지 않으면 메타데이터, 권한 또는 목록 구조가 손실될 수 있습니다.

이 가이드에서는 SharePoint 목록을 다른 사이트로 마이그레이션하는 검증된 여러 방법과 함께 실제 경험 기반 인사이트, 제한 사항 및 모범 사례를 안내합니다.

왜 SharePoint 목록을 마이그레이션해야 할까요?

다음과 같은 경우에 목록 마이그레이션을 추천합니다:

부서 또는 팀 간 데이터 이동

SharePoint 사이트 구조 재구성

SharePoint tenant to tenant migration

백업 및 복원 시나리오

테스트 또는 스테이징 환경 구축

마이그레이션 전에 확인할 사항

제 경험상, 아래 사항을 확인하지 않으면 문제가 발생할 수 있습니다:

원본 및 대상 사이트가 모두 존재하는지 확인

권한 확인 (사이트 소유자 또는 관리자 권한 필요)

목록 크기 및 항목 수 확인

조회 열, 워크플로, Power Automate 흐름 확인

백업 수행 (항상 권장)

방법 1: “Save List as Template” 사용

구조 + 데이터(선택)를 함께 마이그레이션할 때 가장 간단한 방법입니다.

단계:

원본 SharePoint 목록으로 이동

설정(⚙️) → List Settings 클릭

Save list as template 클릭

다음 정보 입력:

파일 이름

템플릿 이름

Include Content 체크 (데이터 포함 시)

OK 클릭

대상 사이트로 가져오기:

대상 SharePoint 사이트로 이동

Site Settings → List Templates 클릭

템플릿(.stp 파일) 업로드

해당 템플릿으로 새 목록 생성

장점:

간단하고 빠름

코딩 불필요

단점:

일부 테넌트에서는 최신 SharePoint에서 지원되지 않음

목록 크기 제한 (일반적으로 50MB 이하)

방법 2: Excel 내보내기/가져오기

복잡한 관계가 없는 단순 목록에 적합합니다.

단계:

원본 목록 열기

Export to Excel 클릭

파일 저장

대상 사이트 이동

New → List → From Excel 클릭

파일 업로드 후 열 매핑

장점:

초보자에게 쉬움

최신 SharePoint에서 사용 가능

단점:

메타데이터 손실 (작성자, 수정자 등)

워크플로 및 권한 유지 불가

방법 3: Power Automate 사용 (증분 마이그레이션에 최적)

자동화 또는 지속적인 동기화가 필요할 때 추천합니다.

단계:

Power Automate 이동

새 Flow 생성

트리거 선택:

“When an item is created” 또는

“Manually trigger a flow”

Get items (원본 목록) 추가

Create item (대상 목록) 추가

모든 열 매핑

Flow 저장 및 실행

장점:

자동화 지원

증분 마이그레이션 가능

단점:

수동 열 매핑 필요

대용량 목록에서는 느림

방법 4: SharePoint PnP PowerShell 사용 (고급 방법)

완전한 제어가 필요한 경우 추천합니다.

설치:
Install-Module PnP.PowerShell
연결:
Connect-PnPOnline -Url "https://your-site-url" -Interactive
내보내기:
Get-PnPListItem -List "Source List" | Export-Csv -Path "C:\list.csv"
가져오기:
Connect-PnPOnline -Url "https://destination-site-url" -Interactive

Import-Csv "C:\list.csv" | ForEach-Object {
Add-PnPListItem -List "Destination List" -Values @{
Title = $_.Title
}
}
장점:

매우 유연함

대규모 마이그레이션에 적합

단점:

스크립트 지식 필요

수동 매핑 필요

방법 5: 전문 도구 사용

대규모 또는 비즈니스 중요 SharePoint 목록 마이그레이션을 수행할 때는 SharePoint Online Migration Tool과 같은 전문 솔루션을 사용하는 것을 선호합니다. 수동 방법은 특히 복잡한 환경에서 메타데이터, 권한 및 버전 기록을 제대로 유지하지 못하는 경우가 많습니다. 이 도구는 SharePoint 목록, 문서 라이브러리 및 사이트 콘텐츠를 데이터 손실 없이 사이트 또는 테넌트 간에 마이그레이션하도록 설계되었습니다.

또한 대량 및 엔터프라이즈 수준의 마이그레이션을 효율적으로 처리할 수 있어 매우 신뢰할 수 있습니다. 고급 API와 자동화를 활용하여 데이터 구조와 무결성을 유지하면서 안전하고 정확한 데이터 전송을 보장합니다. 제 경험상 이러한 도구는 마이그레이션 시간을 크게 줄이고 수동 작업에서 발생할 수 있는 위험을 최소화합니다.

주요 기능

SharePoint 목록, 문서 라이브러리, 사이트 및 폴더 전체 마이그레이션

생성 날짜, 수정 날짜, 작성자 등의 메타데이터 유지

버전 기록 유지

증분(델타) 마이그레이션 지원

여러 사이트 및 목록의 대량 마이그레이션 지원

사용자 및 권한 매핑 기능 제공

필터 기반 선택적 마이그레이션 지원

사전 분석 기능으로 문제 사전 탐지

엔터프라이즈 환경에서 대용량 데이터 처리 가능

상세 보고서 및 로그 생성

예약 및 자동화 지원

내가 항상 따르는 모범 사례

소규모 데이터로 먼저 테스트

업무 시간 외에 마이그레이션 수행

마이그레이션 후 데이터 검증

워크플로 및 자동화 재구성

필요 시 버전 기록 유지

일반적인 문제 및 해결 방법

  1. 열 누락 문제
    → 대상 목록의 스키마 확인

  2. 조회 필드 오류
    → 조회 관계를 수동으로 다시 생성

  3. 권한 손실
    → 마이그레이션 후 권한 재설정

  4. 데이터 마이그레이션 오류
    → 열 유형 및 형식 확인

어떤 방법을 선택해야 할까요?

사용 사례별 추천:

소규모 목록 → Excel 방법

구조 + 데이터 → 템플릿 방법

자동화 필요 → Power Automate

대규모/복잡 → PowerShell 또는 전문 도구

마무리

SharePoint 목록 마이그레이션은 단순한 데이터 복사가 아니라 구조, 사용성 및 데이터 무결성을 유지하는 것이 핵심입니다.

제 경험상 적절한 방법 선택은 다음 요소에 따라 달라집니다:

데이터 복잡성

마이그레이션 규모

기술 수준

중요한 마이그레이션이라면 반드시 충분히 테스트하고 자동화 또는 전문 도구 사용을 고려하는 것이 좋습니다.

FAQs

Q1. SharePoint 목록을 다른 사이트로 복사하려면 어떻게 해야 하나요?
템플릿, Excel, Power Automate 또는 PowerShell을 사용할 수 있습니다.

Q2. 메타데이터를 포함하여 마이그레이션할 수 있나요?
네, PowerShell 또는 전문 도구를 사용하면 가능합니다.

Q3. SharePoint 목록을 직접 이동하는 방법이 있나요?
아니요, 기본 이동 기능은 없으며 마이그레이션 방법을 사용해야 합니다.
