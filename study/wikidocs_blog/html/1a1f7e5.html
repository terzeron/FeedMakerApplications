0. 도구별 역할 개요

이 가이드에서 설치하는 각 도구의 역할과 상호관계를 이해하면 개발 환경 구축이 더 쉬워집니다.

🔧 주요 도구 역할

Node.js

  • 역할: JavaScript 실행 환경 및 패키지 관리자(npm) 제공
  • 왜 필요한가: clasp와 gas-fakes가 Node.js 기반으로 개발되어 npm을 통해 설치됨
  • 사용 예시: npm install -g @google/clasp

clasp (Command Line Apps Script Projects)

  • 역할: Google Apps Script의 CLI(Command Line Interface) 도구
  • 주요 기능:
  • 로컬에서 Apps Script 코드 작성 및 편집
  • 로컬 코드를 Google Apps Script 클라우드 환경에 푸시/동기화
  • TypeScript 문법 지원 (.ts 파일을 .gs로 자동 변환)
  • Git과 유사한 버전 관리 (clasp의 버전 기능)
  • 중요 제한사항:
  • TypeScript로 작성 후 clasp가 변환한 Apps Script 코드는 클린업(정리)됨
  • 변환된 .gs 파일을 직접 수정하면 다음 푸시 시 로컬 변경사항이 덮어씀
  • 결론: 항상 로컬 .ts 파일만 수정하고, 클라우드 .gs 파일은 직접 수정하지 말 것
  • 사용 예시:
    cmd clasp push # 로컬 → 클라우드 업로드 clasp pull # 클라우드 → 로컬 다운로드 clasp open # 웹 에디터 열기

gas-fakes (Google Apps Script Fakes)

  • 역할: 로컬 환경에서 Apps Script 코드를 실행하고 테스트하는 샌드박스 도구
  • 주요 기능:
  • Google Apps Script API(Drive, Sheets, Docs 등)를 로컬에서 에뮬레이션
  • 실제 Google 서비스와 유사한 동작을 로컬에서 테스트
  • 안전한 테스트 환경 제공 (실제 데이터 건드리지 않음)
  • 사용 예시:
    cmd gas-fakes run src/Code.js # 로컬에서 스크립트 실행 gas-fakes test # 테스트 실행

Visual Studio Code (VS Code)

  • 역할: 코드 편집기 (IDE)
  • 주요 기능:
  • JavaScript/TypeScript 문법 하이라이팅
  • IntelliSense (자동완성)
  • 디버깅 도구
  • 확장 프로그램 생태계
  • 왜 사용하는가: Google Apps Script 웹 에디터보다 강력한 편집 기능 제공

opencode

  • 역할: AI 기반 개발 도구 및 통합 개발 환경 관리
  • 주요 기능:
  • AI 어시스턴트를 통한 코드 작성/리뷰 지원
  • 다양한 개발 도구와의 통합 관리 (MCP 프로토콜 지원)
  • 사용 방법 1 (기본): 직접 명령어 입력 - 별도 설정 없이 즉시 사용, 개발자가 직접 제어
  • 사용 방법 2 (MCP): clasp/gas-fakes를 MCP 서버로 등록 - 초기 설정 필요하나 opencode 내에서 명령 통합 관리 가능
  • 참고: 직접 명령어 방식은 간단하고 예측 가능함. MCP 방식은 반복 작업 자동화에 유리하나 초기 설정과 AI 명령어 해석에 주의 필요

🔄 도구 간 관계도

그림 1: 로컬 개발 환경과 Google Cloud 간의 도구 관계도

그림 2: TypeScript 개발부터 클라우드 배포까지의 개발 워크플로우

⚠️ 중요: opencode와의 관계

두 가지 사용 방식:

방식 1: 직접 명령어 실행 (권장)
- opencode는 AI 개발 어시스턴트 도구입니다
- 별도 설정 없이 개발자가 명령어를 직접 입력하고 AI가 응답하는 구조입니다
- opencode에서 clasp push나 gas-fakes run 같은 명령어를 직접 입력하여 사용합니다

방식 2: MCP(Model Context Protocol) 서버로 설정
- clasp와 gas-fakes를 MCP 서버로 등록할 수 있습니다
- AI가 직접 도구를 호출하여 명령어를 실행할 수 있습니다
- 고급 사용자용이며, 일반 사용자는 직접 명령어 입력 방식을 권장합니다
- MCP 설정 방법은 5.3절을 참고하세요

5.3 opencode 사용 방법

방법 1: 직접 명령어 실행 (기본)

별도 설정 없이 opencode에서 직접 명령어를 입력하는 방식:

# opencode 실행
opencode

# 또는 특정 디렉토리에서 실행
cd C:\Projects\gas-projects
opencode

opencode에서 clasp/gas-fakes 명령어 사용:

# opencode 프롬프트에서 직접 명령어 실행
> clasp push
> gas-fakes run src/Code.js
> clasp open

방법 2: MCP(Model Context Protocol) 서버로 설정 (통합 방식)

clasp와 gas-fakes를 MCP 서버로 등록하여 opencode가 직접 도구를 호출할 수 있게 설정:

설정 방법:
1. opencode 설정 파일 (~/.opencode/config.json 또는 프로젝트 루트 .opencode/config.json) 생성/수정:

{
  "mcpServers": {
    "clasp": {
      "command": "clasp",
      "args": [],
      "env": {}
    },
    "gas-fakes": {
      "command": "gas-fakes",
      "args": [],
      "env": {}
    }
  }
}
  1. opencode 재시작 후 MCP 도구 사용 가능

MCP 방식의 특징:

측면 직접 명령어 방식 MCP 서버 방식
초기 설정 불필요 MCP 서버 등록 필요
명령 일관성 수동 입력 opencode 내 통합 관리
AI 자동화 제한적 자연어 명령 가능 ("코드 푸시해줘")
제어권 개발자가 직접 제어 AI가 명령어 생성/실행
예측 가능성 높음 (명확함) 낮음 (AI 해석 의존)
디버깅 쉬움 복잡할 수 있음

MCP 방식이 유리한 경우:
- ✅ 반복적인 clasp/gas-fakes 명령어 조합을 자주 사용하는 경우
- ✅ AI에게 "로컬 테스트하고 문제 없으면 배포해줘" 같은 통합 작업을 맡기고 싶은 경우
- ✅ opencode 인터페이스 내에서 모든 개발 작업을 통합하여 관리하고 싶은 경우

주의사항:
- MCP 서버 등록은 추가 초기 설정이 필요함
- AI가 명령어를 잘못 해석하거나 예상치 못한 동작을 할 수 있음
- 중요한 작업(프로덕션 배포 등)은 반드시 확인 후 실행 권장

추천 사용법:
- 초보자/단순 작업: 방법 1 (직접 명령어 실행) - 간단하고 명확함
- 반복 작업 자동화: 방법 2 (MCP 서버) - 일관된 워크플로우 구축 가능

5.4 Gemini API 설정 (AI 기능 활성화)

opencode의 AI 기능을 사용하려면 Google Gemini API key가 필요합니다.

1. Gemini API Key 생성

  1. Google AI Studio 접속
  2. Google 계정으로 로그인
  3. "Create API key" 버튼 클릭
  4. API key 이름 입력 (예: "opencode-gas-dev")
  5. 생성된 API key 복사 (⚠️ 이 창을 닫으면 다시 볼 수 없으므로 반드시 복사필수)

2. opencode에 API Key 설정

Windows에서 환경 변수로 설정 (권장):

# 시스템 환경 변수 설정 (관리자 권한 PowerShell)
[Environment]::SetEnvironmentVariable("GEMINI_API_KEY", "your-api-key-here", "User")

# 현재 터미널에만 설정 (임시)
setx GEMINI_API_KEY "your-api-key-here"

또는 opencode 설정 파일에 직접 설정:

~/.opencode/config.json 파일 수정:

{
  "llm": {
    "provider": "google",
    "model": "gemini-2.0-flash-exp",
    "apiKey": "your-api-key-here"
  }
}

3. 설정 확인

opencode

- 공개 저장소(GitHub 등)에 API key를 올리지 마세요
- .gitignore에 config.json 추가 권장
- 필요시 API key를 정기적으로 재생성하세요

5.5 oh-my-opencode 설치 (선택사항, 권장)

oh-my-opencode는 opencode의 기능을 확장하는 플러그인으로, 특히 멀티 에이전트 기능이 매우 강력합니다.

🌟 주요 장점:

기능 설명
멀티 에이전트 시스템 Sisyphus(메인), Prometheus(기획), Oracle(아키텍처/디버깅), Librarian(문서검색) 등 전문 에이전트들이 협업
백그라운드 에이전트 여러 작업을 병렬로 처리하여 개발 속도 향상
LSP & AST 도구 리팩토링, 코드 진단, AST 기반 검색 등 고급 코드 분석
Claude Code 호환 기존 Claude Code 사용자라면 설정 그대로 사용 가능
미리 설정된 MCP gas-fakes, clasp 등의 MCP 도구가 미리 설정됨

설치 방법:

# npm으로 전역 설치
npm install -g oh-my-opencode

# 또는 bun 사용 시
bunx oh-my-opencode install

opencode에 플러그인 등록:

~/.config/opencode/opencode.json 파일 수정:

{
  "plugin": [
    "oh-my-opencode"
  ],
  "llm": {
    "provider": "google",
    "model": "gemini-2.0-flash-exp",
    "apiKey": "your-api-key-here"
  }
}

사용 예시:

# oh-my-opencode 설치 후 opencode 실행
opencode

# 에이전트 호출 예시
> @oracle 이 코드의 아키텍처를 검토해줘
> @librarian gas-fakes 사용법을 알려줘
> @prometheus 이 기능 구현 계획을 세워줘

# 백그라운드 에이전트 실행
> @sisyphus 이 파일들을 동시에 분석해줘

주요 에이전트:

에이전트 역할 사용 예시

6.1 새 프로젝트 생성 예시

전체 도구를 통합하여 사용하는 예시입니다.

단계 1: 프로젝트 디렉토리 생성

mkdir C:\Projects\gas-projects
cd C:\Projects\gas-projects

단계 2: clasp로 프로젝트 생성

# 새 Apps Script 프로젝트 생성
clasp create --title "내 첫 GAS 프로젝트" --type standalone

# 또는 기존 프로젝트 클론
# clasp clone <SCRIPT_ID>

단계 3: 프로젝트 구조 설정

my-gas-project/
├── .clasp.json          # clasp 설정
├── appsscript.json      # Apps Script 매니페스트
├── src/                 # 소스 코드 디렉토리
│   ├── Code.js
│   └── utils.js
├── tests/               # 테스트 파일
│   └── test.js
├── gasfakes.json        # gas-fakes 설정
└── package.json         # Node.js 의존성

단계 4: gas-fakes 설정 파일 생성

gasfakes.json 파일 생성:

{
  "projectId": "your-google-cloud-project-id",
  "sandbox": true,
  "cacheDir": "./.gas-fakes-cache",
  "testDir": "./tests"
}

단계 5: package.json 생성

npm init -y

package.json 수정:

{
  "name": "my-gas-project",
  "version": "1.0.0",
  "scripts": {
    "test": "gas-fakes run tests/test.js",
    "push": "clasp push",
    "watch": "clasp push --watch",
    "open": "clasp open"
  },
  "devDependencies": {
    "@types/google-apps-script": "^1.0.83"
  }
}

단계 6: TypeScript 타입 설치 (선택)

npm install --save-dev @types/google-apps-script

7. 문제 해결

7.1 일반적인 문제

Q: clasp login이 브라우저를 열지 않습니다.

A: 수동으로 URL을 복사하여 브라우저에 붙여넣으세요. 인증 후 표시되는 코드를 터미널에 입력하세요.

Q: "No credentials found" 오류

A: clasp 인증이 필요합니다. clasp login을 다시 실행하세요.

Q: "Google Apps Script API has not been used in project" 오류

A: Apps Script API 설정 페이지에서 API를 활성화하세요.

Q: gas-fakes에서 "Authentication required" 오류

A: gas-fakes auth를 실행하여 인증을 완료하세요.

Q: 한글 경로 문제

A: 프로젝트 경로에 한글이 있으면 문제가 발생할 수 있습니다. 영문 경로 사용을 권장합니다.

# 권장
C:\Projects\gas-project\

# 비권장
C:\사용자\문서\gas프로젝트\

7.2 PATH 환경 변수 문제

Node.js를 설치 후에도 명령어를 찾을 수 없는 경우:

PATH 확인:

echo %PATH%

PATH 수동 추가 (필요시):
1. "시스템 환경 변수 편집" 검색 및 실행
2. "환경 변수" 버튼 클릭
3. "사용자 변수" 또는 "시스템 변수"에서 Path 선택
4. "편집" 클릭
5. 새 항목 추가:
- C:\Program Files\nodejs\ (Node.js)
- C:\Users\<사용자명>\AppData\Roaming\npm\ (npm 전역 패키지)
6. 확인 클릭 후 새로운 명령 프롬프트 창 열기

7.3 Windows 방화벽/보안 문제

PowerShell 실행 정책 문제:

# 관리자 권한 PowerShell에서 실행
Set-ExecutionPolicy RemoteSigned -CurrentUser

Windows Defender SmartScreen:
- "추가 정보" 클릭 → "실행" 클릭

clasp open                     # 웹 에디터 열기
clasp logs                     # 스택드라이버 로그 보기
clasp version "설명"           # 버전 생성
clasp deploy                   # 배포

# gas-fakes 주요 명령어
gas-fakes auth                 # 인증
gas-fakes run <file.js>        # 스크립트 실행
gas-fakes test                 # 테스트 실행
gas-fakes sandbox              # 샌드박스 모드 실행

9. 정리

이제 다음 도구들이 설치되어 Google Apps Script 개발이 가능합니다:

도구 역할 사용 시점
✅ Node.js JavaScript 실행 환경 모든 도구의 기반 (필수)
✅ clasp Apps Script CLI 로컬 ↔ 클라우드 코드 동기화
✅ gas-fakes 로컬 테스트 샌드박스 코드 로컬 테스트 시
✅ VS Code 코드 편집기 코드 작성 및 수정 시
✅ opencode AI 개발 어시스턴트 개발 도중 질문하거나 도움이 필요할 때
⭐ oh-my-opencode opencode 플러그인 (멀티 에이전트) 복잡한 작업, 리팩토링, 코드 분석 시

opencode 사용 방식 선택:

방식 특징 추천 대상
직접 명령어 별도 설정 없이 즉시 사용, 개발자가 직접 제어 초보자, 단순 작업
MCP 서버 초기 설정 필요하나 opencode 내 명령 통합, 반복 작업 자동화 반복 작업 많은 사용자
+ oh-my-opencode 멀티 에이전트 시스템, 고급 코드 분석 복잡한 리팩토링, 대규모 프로젝트

도구별 상호작용:
1. VS Code에서 코드 작성 (.ts 또는 .js)
2. gas-fakes로 로컬에서 테스트
3. clasp push로 클라우드에 배포
4. 필요시 opencode에서 AI에게 질문하거나 명령어 실행
5. (선택) oh-my-opencode 멀티 에이전트로 복잡한 작업 처리 (@oracle, @librarian 등)
