MCP 기반 에이전트 연동: n8n x OpenCode로 구축하는 고성능 1인 에이전시 자동화

@wilson_lim
2026년 3월 15일 7:00 오후
에이전트 Mcp 워크플로우 Llm Opencode 자동화 N8n

왜 지금, MCP + n8n + OpenCode인가

고성능 1인 에이전시 시대에 핵심은 “작게 시작해 빠르게 반복”입니다. n8n은 시각적 워크플로우 자동화로 운영의 속도를, OpenCode는 LLM 기반 코드 생성을 통해 기능 확장을 제공합니다. 여기에 MCP(Model Context Protocol)를 결합하면 에이전트가 표준화된 방식으로 도구와 리소스에 접근하며, 복잡한 비즈니스 프로세스를 비용 효율적으로 자동화할 수 있습니다.

  • n8n: 트리거, 분기, 재시도, 스케줄링, 에러 핸들링 등 운영 자동화의 골격
  • OpenCode: LLM으로 코드/스크립트/테스트/문서 생성 및 리팩터링을 담당
  • MCP: 모델이 사용할 수 있는 도구·리소스·프롬프트를 명세화하여 상호 운용성을 보장

핵심 아이디어: n8n이 오케스트레이션을, OpenCode가 기능 생성을, MCP가 연결 규약을 맡아 엔드투엔드 자동화 파이프라인을 구성합니다.

MCP 빠른 개념 정리

MCP는 LLM이 외부 세계와 상호작용하는 표준 인터페이스를 제공합니다.
- Tools: 모델이 호출할 수 있는 함수(예: code_generate, run_tests, fetch_resource)
- Resources: 모델이 읽고 쓸 수 있는 데이터/아티팩트(예: repo://, s3://, db://)
- Prompts: 일관된 작업 컨텍스트를 제공하는 템플릿(시스템/태스크/롤 가이드)
- Sessions: 호출 범위와 권한을 캡슐화(스코프·쿼터·타임아웃)

이 구조 덕분에 LLM이 “무엇을 어떻게 쓸 수 있는지”를 명확히 이해하고, 실행 결과를 재현 가능하게 관리할 수 있습니다.

아키텍처 개요

  • 입력 채널: Webhook, Slack, 이메일, 폼 제출 등 → n8n 트리거
  • 컨텍스트 수집: n8n이 CRM/ERP/DB/API에서 데이터 수집 및 전처리
  • MCP 세션: n8n이 OpenCode MCP 서버에 세션 생성(권한·리소스 스코프 지정)
  • 코드 생성/검증: OpenCode가 LLM으로 코드 생성 → 테스트 런 → 아티팩트 저장
  • 실행/배포: n8n이 생성된 코드 실행(로컬 함수, 서버리스, 컨테이너) 또는 릴리스 파이프라인 트리거
  • 관측/로깅: 실행 로그, 토큰 사용량, 퀄리티 메트릭을 저장 및 대시보드화

OpenCode + n8n 시너지 포인트

  • 비용 효율: 작은 모델로 플래닝, 큰 모델은 코드 생성·리팩토링에만 온디맨드 사용
  • 민첩성: 새 요구사항을 프롬프트/워크플로우 수정으로 단시간 반영
  • 표준화: MCP 스키마로 도구·리소스 계약을 명시, 구성 변경에도 안정성 유지
  • 가시성: n8n의 재시도/에러브랜치 + OpenCode 테스트 결과로 디버깅이 용이

레퍼런스 패턴: 리드 스코어링 + CRM 업데이트 에이전트

1) Webhook 트리거로 신규 리드 유입
2) n8n이 회사 웹/소셜/뉴스를 크롤링(HTTP Request, Function)
3) MCP 세션 생성: 리드 프로필과 크롤링 데이터 전달, 사용 가능한 툴 스코프 지정
4) OpenCode가 스코어링 코드 생성 → 단위 테스트 자동 생성 → 샌드박스 실행
5) 스코어 결과/근거를 아티팩트로 저장(repo:// 또는 s3://)
6) n8n이 CRM 업데이트 후 Slack에 요약 보고

빠른 시작: 로컬 개발 구성 예시

MCP 툴/리소스 스키마 설계 예시

{
  "tools": [
    {
      "name": "code_generate",
      "description": "Generate production-ready code with docstring and tests",
      "input_schema": {
        "type": "object",
        "properties": {
          "language": {"type": "string"},
          "spec": {"type": "string"},
          "constraints": {"type": "array", "items": {"type": "string"}}
        },
        "required": ["language", "spec"]
      }
    },
    {
      "name": "run_tests",
      "description": "Execute unit tests in sandbox and return results",
      "input_schema": {"type": "object", "properties": {"path": {"type": "string"}}}
    },
    {
      "name": "artifact_write",
      "description": "Write files to repo or object store",
      "input_schema": {
        "type": "object",
        "properties": {"uri": {"type": "string"}, "content": {"type": "string"}}
      }
    }
  ],
  "resources": [
    {"uri": "repo://agent/lead_scoring", "writable": true},
    {"uri": "s3://artifacts/lead_scoring", "writable": true}
  ]
}

n8n 워크플로우 뼈대

  • Trigger: Webhook
  • Data Prep: Function(리드 정규화), HTTP Request(외부 데이터 수집)
  • MCP Session: HTTP Request → POST /sessions (scope: code_generate, run_tests, artifact_write)
  • Tool Calls: HTTP Request → POST /tools/code_generate, POST /tools/run_tests
  • Branching: IF(테스트 통과 여부)
  • Deploy: GitHub 노드(PR 생성) 또는 AWS Lambda 노드(배포)
  • CRM Update: HubSpot/Salesforce 노드
  • Notify: Slack 노드(요약 리포트)

OpenCode 프롬프트 전략

  • 시스템 프롬프트: 코딩 표준, 보안 가이드, 성능 제약(시간/메모리), 로깅 포맷 명시
  • 태스크 프롬프트: 입력 스키마, 산출물 경로, 테스트 기준(KPI) 구체화
  • 컨텍스트 주입: 최근 실패 로그, 실제 샘플 데이터 3~5건, 에러 케이스 우선
  • 출력 계약: “코드 + 테스트 + 실행 가이드”를 단일 아티팩트로 반환

예시 태스크 스펙:

목표: 리드 프로필(JSON)과 외부 신호를 입력으로 0~100 리드 스코어 산출.
요구사항:
- 가중치 기반 + 규칙 기반 혼합
- 누락 필드에 대한 강건성
- explainability: 기여도 상위 3개 피처와 이유 제공
산출물:
- src/score.py, tests/test_score.py, README.md
성능 기준:
- 샘플 벤치마크 데이터셋에서 MAE <= 5

비용 최적화 체크리스트

  • 모델 구분: LLM-S(플래닝, 요약), LLM-L(코드 생성·리팩토링 전용)
  • 캐시/요약: 장문 컨텍스트는 요약·스냅샷·키 밸류 캐시 활용
  • 함수 호출 우선: 텍스트 생성보다 구조화 호출로 토큰 절감
  • 배치 처리: n8n SplitInBatches로 API 콜 묶기
  • 재사용: 생성 코드/프롬프트 버전 관리로 중복 생성을 최소화

보안·거버넌스

  • 스코프 토큰: 세션 단위로 도구/리소스 권한 최소화
  • 샌드박스: 네트워크/파일시스템 격리, 타임/메모리 리밋 설정
  • 감사 로그: who/what/when/why를 이벤트 스트림으로 기록(OpenTelemetry)
  • 비밀 관리: n8n 크리덴셜 + KMS 연계, 프롬프트 내 비밀값 노출 금지

운영과 관측

  • SLO: 성공률, 평균 지연, 재시도율, 테스트 통과율, 토큰/콜당 비용
  • 얼럿: 실패 패턴(타임아웃/429/테스트 실패)별 Slack/PagerDuty 라우팅
  • 회귀 방지: 골든셋으로 매 릴리스 전 자동 평가(n8n Cron → MCP run_tests)

확장 아이디어

  • 멀티에이전트: 리서처(정보수집) → 플래너(스펙) → 코더(구현) → QA(테스트)
  • 하이브리드 RAG: n8n이 벡터스토어 업데이트, OpenCode는 질의 최적화 코드 생성
  • 옴니채널: 이메일/슬랙/노션/티켓 시스템을 하나의 MCP 세션으로 관통

실전 도입 로드맵(1주)

  • Day 1–2: 핵심 프로세스 1개 선택, n8n 초안 작성
  • Day 3–4: MCP 스키마/프롬프트 정의, OpenCode로 v0 코드 생성
  • Day 5: 테스트/관측/비용 가드레일 추가, 스테이징 릴리스
  • Day 6–7: 실제 데이터로 튜닝, 실패 케이스 학습 사이클 구축

마무리

MCP의 표준화, n8n의 오케스트레이션, OpenCode의 코드 생성이 결합되면 1인 에이전시도 엔터프라이즈급 자동화를 구현할 수 있습니다. 작은 범위에서 시작해 프롬프트·툴·워크플로우를 버전 관리하며 반복하면, 비용은 낮추고 속도와 품질은 높일 수 있습니다. 지금 바로 “하나의 핵심 시나리오”로 파일럿을 만들어 보세요.
