Mfa 계정생성 아마존 Iam Aws 클라우드 리눅스입문 리눅스

AWS 계정 생성

아마존 클라우드 서비스를 사용하기 위해 계정을 먼저 생성해야 합니다. 웹 브라우저에서 웹 사이트 주소(https://aws.amazon.com/ko/free)를 입력해 [무료 계정 생성] 버튼을 클릭하면 계정 생성 화면으로 이동합니다. 프리 티어(Free Tier)는 신규 사용자에게 일정 기간 동안 무료로 AWS 서비스를 체험할 수 있는 혜택을 제공합니다.


AWS 계정 생성을 위한 사이트

계정 생성을 위한 첫 화면에서 이메일 주소와 계정명을 입력하고 [Verify email address] 버튼을 클릭하면 입력한 이메일로 본인 확인용 검증 코드를 전송합니다. 메일이 수신되지 않는다면 스팸 메일함에 도착한 것은 아닌지 확인합니다. 6자리 검증 코드를 입력하고 [Verify] 버튼을 클릭하면 다음 페이지로 넘어갑니다.


본인 확인을 위한 이메일 검증

루트 사용자의 비밀번호를 설정하는 단계로 넘어오면 대소문자, 숫자, 특수문자를 조합한 비밀번호를 조합해 입력합니다. 비밀번호는 실수로 잘못 입력하는 것을 예방하기 위해 두 번 입력합니다.


루트 사용자의 비밀번호 설정

다음은 AWS 사용 목적과 연락처를 입력하는 화면입니다. 사용 목적은 기업용 또는 개인용 중 하나를 선택하고, 국가와 주소, 우편번호를 작성합니다. 대한민국 국가 코드는 82번으로 입력하면 됩니다.


AWS 사용 목적과 연락처 정보 입력

AWS 프리 티어는 12개월 간 무료로 사용할 수 있지만, 기본 사용 기간이 지나거나 기본 서비스가 아닌 초과 사용량에 대해서는 과금이 이뤄지기 때문에 카드 정보를 필수로 입력해야 회원 가입을 할 수 있습니다. 카드 정보와 영수증이 전달될 이메일 주소를 입력하고 [Verify and continue] 버튼을 클릭해 회원 가입을 계속 진행합니다.


결제를 위한 카드 정보 입력

본인 확인을 위한 핸드폰 번호 검증 절차를 진행합니다. 문자 메시지 또는 음성 전화를 통해 보인 확인이 가능합니다. 문자 메시지를 선택하고, 핸드폰 번호를 입력 후 [Send SMS] 버튼을 클릭하면 핸드폰으로 네 자리의 검증 숫자가 전달되며, 해당 번호를 입력해 다음 단계로 넘어갑니다.


본인 확인을 위한 핸드폰 번호 인증

마지막 단계는 사용할 플랜을 결정하는 것입니다. 무료 서비스인 프리 티어를 선택하기 위해 맨 왼쪽에 있는 기본 플랜을 선택 후 [Complete sign up] 버튼을 클릭하면 회원 가입이 완료됩니다.


사용할 플랜 결정

성공적으로 회원 가입이 되면 다음과 같이 축하 메시지가 나타나며, [Go to the AWS Management Console] 버튼을 클릭하면 AWS 관리 콘솔 화면으로 이동합니다.


AWS 관리 콘솔로 이동

AWS 콘솔 홈의 언어 및 리전 설정

회원 가입 후 가장 먼저 확인하는 화면은 AWS 관리 콘솔로 AWS의 모든 서비스를 확인하고, 언어 및 리전에 대한 설정을 수행할 수 있습니다. 먼저 언어를 변경하기 위해 화면 우측 상단의 톱니바퀴 모양의 아이콘을 클릭후 [Language] -> [한국어]로 선택해 변경합니다. 웹 브라우저의 기본 언어가 한국어라면 별도로 언어를 변경할 필요는 없습니다.


언어 설정을 한국어로 변경

AWS에서 리전(Region)은 데이터 센터가 위치한 물리적인 지역을 의미합니다. AWS는 전 세계 여러 지역에 리전을 두고 있으며, 리전에 따라 응답 속도, 데이터의 법적 충족 요건, 가격 등이 달라질 수 있습니다. 리전을 서울로 변경하기 위해 화면 우측 상단의 지역명을 클릭하고, 서울(ap-northeast-2) 리전을 선택합니다.


서울 리전으로 변경

루트 사용자에 대한 MFA 추가

MFA(Multi-Factor Authentication, 다중 요소 인증)는 사용자의 신원을 확인하기 위해 두 가지 이상의 서로 다른 인증을 요구하는 방법입니다. 일반적으로 사용자를 인증하기 위한 방법은 크게 세 가지로 구분할 수 있습니다. 첫 번째는 지식 기반으로 사용자가 기억할 수 있는 비밀번호, PIN 번호, 질문에 대한 답 등이 있습니다. 두 번째는 소지 기반으로 스마트폰, 보안 토큰, 1회용 비밀번호인 OTP 생성기 등이 있습니다. 세 번째는 생체 기반으로 지문, 얼굴, 홍채 인식 등이 있습니다. AWS에서 최초 생성한 계정은 루트 계정이므로 MFA를 추가하는 것은 중요합니다. AWS 계정에 MFA를 추가하기 위해 콘솔 홈 왼쪽 삼선 버튼을 클릭해 모든 서비스에서 IAM 서비스를 찾거나 검색 창에서 “IAM” 키워드로 해당 서비스를 찾아 클릭합니다.


MFA 추가를 위한 IAM 서비스

IAM(Identity and Access Management)은 조직 내 사용자들이 누구인지 확인하고, 어떤 자원에 접근할 수 있는지 제어 및 관리하는 서비스입니다. IAM 대시보드로 이동하면 보안 권장 사항으로 루트 사용자에 대한 MFA 추가 항목이 표시됩니다. 화면 중앙에 [MFA 추가] 버튼을 클릭합니다.


루트 계정에 MFA 추가

MFA 추가는 두 단계로 진행되며, 첫 단계는 MFA 디바이스 이름을 정하고 두 번째 인증 방법을 선택합니다. 디바이스명은 알파벳과 숫자, 일부 특수문자를 사용할 수 있습니다. 디바이스 옵션으로 인증 관리자 앱을 선택합니다. 인증 관리자 앱은 스마트폰 앱을 통해 인증을 수행할 수 있습니다.


루트 계정에 MFA 추가

디바이스 이름과 옵션을 선택하고 [다음] 버튼을 누르면 두 번째 단계인 디바이스 설정화면으로 이동합니다. 이 단계를 진행하기 위해 스마트폰에 앱을 먼저 설치해야 합니다. 인증을 위한 여러 앱이 있지만 예시는 구글 Authenticator 앱을 사용한 것입니다. 앱 설치 및 실행 후 [+] 버튼을 누르면 카메라로 QR 코드 스캔할 수 있는 메뉴가 나타납니다. 이 메뉴를 선택 후 MFA 추가 2단계에서 [QR 코드 표시] 영역을 클릭하면 QR 코드가 표시되고, 카메라로 QR 코드를 인식하면 모바일 단말에 6자리 인증 코드가 나타납니다. 인증 코드를 MFA 코드 1에 입력 후 30초 뒤에 6자리 코드가 변경되면 MFA 코드 2에 변경된 코드를 입력하고 [MFA 추가] 버튼을 클릭하면 MFA가 추가됩니다.


MFA 추가를 위한 디바이스 설정

MFA가 추가되었는지 확인하기 위해 로그아웃 후 다시 로그인해 봅시다. 화면 우측 상단에 계정명을 클릭 후 [로그아웃] 버튼을 클릭합니다.


AWS 계정 로그아웃

로그아웃 후 [다시 로그인] 버튼 또는 [콘솔에 로그인] 버튼을 클릭합니다. 그러면 로그인할 수 있는 페이지로 이동하는데 우리가 생성한 계정은 루트 계정이므로 [Sign in using root user email] 버튼을 클릭합니다. IAM 계정은 루트 계정이 아닌 별도의 계정을 의미합니다. 추후에 더욱 안전하게 사용하기 위해 루트 사용자가 아닌 IAM 계정을 별도로 추가해 권한을 부여하고 사용하는 것을 권장합니다.


루트 사용자로 로그인하기 선택

루트 사용자 이메일 주소와 비밀번호를 입력하면 추가적으로 MFA 코드를 입력하는 화면이 나타나며, 구글 Authenticator 앱을 실행해 6자리 코드를 입력하면 됩니다. 코드는 주기적으로 변경되기 때문에 변경되기 전 빠르게 입력해야 합니다.


루트 사용자로 MFA 인증

루트 사용자가 아닌 IAM 사용자를 추가하려면 IAM 서비스에서 왼쪽 메뉴 중 [사용자]를 선택하고, 우측 상단의 [사용자 생성]을 추가하면 됩니다. IAM 사용자는 루트 계정을 대신 사용할 일반 관리자 계정을 생성하는 것이며, IAM 계정도 MFA를 추가해 사용해야 합니다. 그리고, 로그인 과정에서는 루트 사용자가 아닌 IAM 사용자 항목을 선택하고 로그인 시도하면 됩니다. 루트 사용자로도 AWS 서비스를 사용할 수 있지만 실제 환경에서는 IAM 계정을 별도로 생성하고 관리하는 것이 더 안전한 방법입니다.
