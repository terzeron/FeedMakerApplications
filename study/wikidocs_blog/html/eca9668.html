새 에이전트 추가 방법

OpenClaw에서 새 에이전트를 추가하려면 openclaw agents add 명령을 사용합니다. 이 명령은 격리된 작업 공간, 인증, 라우팅을 가진 새 에이전트를 생성합니다 register.agent.ts:106-138 .

기본 사용법
openclaw agents add [이름]

대화형 마법사가 실행되어 작업 공간, 모델, 바인딩 등을 설정합니다 agents.md:26-32 .

주요 옵션

  • --workspace <dir>: 에이전트 작업 공간 디렉터리 (필수: 비대화형 모드)
  • --model <id>: 이 에이전트에 사용할 모델 ID
  • --agent-dir <dir>: 에이전트 상태 디렉터리
  • --bind <channel[:accountId]>: 채널 바인딩 (반복 가능)
  • --non-interactive: 프롬프트 비활성화
  • --json: JSON 출력 register.agent.ts:109-114
예제

  1. 대화형으로 에이전트 추가

    openclaw agents add work

    마법사가 작업 공간(~/.openclaw/workspace-work)과 기본 설정을 안내합니다 wizard.md:173-187 .

  2. 비대화형으로 에이전트 추가

    openclaw agents add work \ --workspace ~/.openclaw/workspace-work \ --model openai/gpt-5.2 \ --bind whatsapp:biz \ --non-interactive \ --json

    작업 공간, 모델, WhatsApp 비즈니스 계정 바인딩을 한 번에 설정합니다 

확인

에이전트 목록과 바인딩을 확인하려면:

openclaw agents list --bindings

Agent에 LLM모델을 추가하기

CLI로 모델 설정하기

1) 전역 기본 모델 설정

openclaw models set <provider/model>

예: openclaw models set anthropic/claude-opus-4-6 models.md:20-24

2) 특정 에이전트에만 모델 설정

openclaw agents set-config --agent <에이전트ID> --model <provider/model>

이 명령은 applyAgentConfig를 통해 해당 에이전트의 model 필드를 업데이트합니다 agents.config.ts:132-153

설정 파일 직접 수정

~/.openclaw/openclaw.json에서 해당 에이전트 항목에 model을 추가/수정:

{  
  agents: {  
    list: [  
      {  
        id: "work",  
        model: "anthropic/claude-opus-4-6",  // 이 줄 추가/수정  
        workspace: "~/.openclaw/workspace-work"  
      }  
    ]  
  }  
}

Agent 삭제하기

openclaw agents delete <에이전트ID> 명령으로 에이전트를 삭제할 수 있습니다. 기본적으로 워크스페이스와 상태 디렉터리를 휴지통으로 이동시킵니다 register.agent.ts:190-206 .

명령어 옵션
  • --force: 확인 프롬프트 건너뛰기
  • --json: JSON 요약 출력 
예시
openclaw agents delete work  
openclaw agents delete work --force --json
