OpenClaw 설치 및 설정 가이드

마지막 업데이트: 2026-02-07

이 문서는 Mac mini에서 OpenClaw를 설치하고 설정하는 과정을 기록한 가이드입니다.

📋 목차

  1. 사전 준비
  2. Claude Code 설치
  3. OpenClaw 설치
  4. 초기 설정
  5. 연동 서비스 설정

1. 사전 준비

필수 도구 확인

# Git 확인
git --version
# 없으면: xcode-select --install

# Node.js 확인 (v18+ 권장)
node --version
npm --version
# 없으면: brew install node 또는 nvm 사용

# Homebrew 확인
brew --version
# 없으면: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

왜 필요한가?

도구 용도
Git OpenClaw 업데이트, 버전 관리
Node.js OpenClaw 런타임 (Node.js 기반)
npm 패키지 관리, OpenClaw 설치
Homebrew macOS 패키지 관리자

2. Claude Code 설치

Mac mini에서 설치

# Claude Code 설치 (Anthropic 공식)
# 방법 1: npm
npm install -g @anthropic-ai/claude-code

# 방법 2: 공식 설치 스크립트
curl -fsSL https://claude.ai/code/install.sh | bash

PATH 설정 (중요!)

설치 후 claude 명령어가 안 되면 PATH 추가 필요:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

왜 필요한가?
- Claude Code가 ~/.local/bin/에 설치됨
- 기본 zsh PATH에 포함 안 되어 있을 수 있음
- .zshrc에 추가해서 영구 적용

인증

# Anthropic API 인증
claude auth

# 또는 환경변수로 설정
export ANTHROPIC_API_KEY="sk-ant-..."

3. OpenClaw 설치

원클릭 설치

curl -fsSL https://openclaw.ai/install.sh | bash

설치 스크립트가 하는 일:
1. Node.js 버전 확인
2. npm으로 openclaw 글로벌 설치
3. 초기 설정 디렉토리 생성 (~/.openclaw/)
4. 설정 마법사 실행

openclaw --version
openclaw status

4. 초기 설정

설정 마법사

설치 후 첫 실행 시 설정 마법사가 나타남:

openclaw onboard

모델 선택

초기 설정 (비용 효율):

Model: anthropic/claude-haiku-4-5-20251001
모델 용도 비용
haiku-4.5 일상 대화, 간단한 작업 저렴
sonnet-4 균형 잡힌 성능 중간
opus-4.5 복잡한 추론, 코딩 비쌈

나중에 변경:

# 설정 파일 직접 수정
nano ~/.openclaw/openclaw.json

# 또는 세션 내에서
/model opus4.5

주요 설정 파일

~/.openclaw/
├── openclaw.json          # 메인 설정
├── workspace/
│   ├── AGENTS.md          # 에이전트 행동 지침
│   ├── SOUL.md            # 페르소나
│   ├── USER.md            # 사용자 정보
│   └── MEMORY.md          # 장기 기억

5. 연동 서비스 설정

Telegram 연동

  1. @BotFather에서 봇 생성
  2. 토큰 받기
  3. 설정에 추가:
// ~/.openclaw/openclaw.json
{
  "plugins": {
    "telegram": {
      "token": "YOUR_BOT_TOKEN",
      "allowedUsers": [YOUR_USER_ID]
    }
  }
}

추후에 구글캘린더 등 적용하려면 ? Google Calendar (gog)

# gog 설치
brew install steipete/tap/gogcli

키체인(Keyring) 방식 변경

만약 계속해서 refresh token missing 에러가 난다면, 맥의 시스템 키체인과 gog 사이의 통신 문제일 수 있습니다. 이 경우 파일 저장 방식으로 변경해 보세요.

Bash
gog auth keyring file

이후 다시 2단계를 실행하여 인증

(파일 방식으로 바꾸면 비밀번호를 물어볼 수 있는데, 이때 설정한 비밀번호는 잘 기억해 두셔야 합니다.)

OAuth 설정

gog auth credentials ~/.config/gog/client_secret.json
gog auth add your@gmail.com --services calendar

방법 1: 환경 변수에 비밀번호 저장 (가장 추천)
현재 사용 중인 file 백엔드를 그대로 쓰면서, 터미널 세션이 열릴 때마다 비밀번호를 자동으로 공급하는 방식입니다.

셸 설정 파일 열기 (사용 중인 셸에 따라 ~/.zshrc 또는 ~/.bash_profile):

Bash
nano ~/.zshrc
파일 맨 아래에 다음 내용 추가:

Bash
export GOG_KEYRING_PASSWORD='설정하신비밀번호'
저장 후 적용: Ctrl + O (저장), Enter, Ctrl + X (나가기)를 누른 후 아래 명령 실행:

Bash
source ~/.zshrc
이제 gog 명령어를 칠 때 더 이상 비밀번호를 묻지 않습니다.

-> 처음에는 오픈클로가 정신못차리는데.
"~/.zshrc 에 저장해놨으니까 터미널을 다시 열어서 써봐" 등으로 말하면 정신차리고 잘 받아옵니다.

Notion 연동

  1. https://www.notion.so/my-integrations 에서 Integration 생성
  2. 토큰 복사
  3. 연동할 DB에 Integration 공유 권한 부여
  4. 스크립트에서 사용:
NOTION_TOKEN = "ntn_..."
DATABASE_ID = "..."

🚀 시작하기

# Gateway 시작
openclaw gateway start

# 상태 확인
openclaw status

# Telegram으로 대화 시작!

🔧 트러블슈팅

command not found: openclaw

# PATH 확인
echo $PATH

# npm 글로벌 경로 확인
npm config get prefix

# PATH에 추가
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

gog: keyring password prompt

Headless 환경에서 keyring 접근 불가:

export GOG_KEYRING_PASSWORD="your-password"

세션 리셋

# 강제 새 세션
openclaw gateway restart
