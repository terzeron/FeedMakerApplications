Exa Deep: AI가 검색하는 시대, 검색 엔진도 에이전트가 된다

@jaehong
2026년 3월 24일 5:39 오전
Ai

복잡한 질문에 단일 검색 한 번으로 답을 얻는 시대는 끝나가고 있다. 쿼리를 분해하고, 병렬로 검색하고, 결과를 추론으로 합성하는 "에이전트형 검색"이 새로운 기본값이 되고 있다. AI 검색 엔진 Exa가 2026년 3월 공개한 Exa Deep은 이 흐름의 최전선에 서 있다. 기존 버전 대비 더 빠르고, 20% 저렴하며, 필드 단위 근거(grounding)가 달린 구조화된 출력을 지원한다. Perplexity Sonar Reasoning Pro와 Parallel Task API를 상대로 세 가지 벤치마크에서 속도와 정확도 모두 앞섰다는 결과를 내놓았다.

에이전트형 검색이란 무엇인가

전통적인 검색 API는 키워드를 받아 관련 문서 목록을 돌려주는 단순 구조다. 개발자가 복잡한 질의를 처리하려면 쿼리 분해, 병렬 호출, 결과 병합, 중복 제거, 인용 추적까지 직접 오케스트레이션 레이어를 구축해야 했다. Exa Deep은 이 전체 파이프라인을 하나의 API 호출로 대체한다.

내부적으로는 최근 출시된 Exa Instant 검색과 LLM 추론을 결합해 동작한다. 사용자의 쿼리 의도를 파악한 뒤, 최적화된 쿼리 확장(query expansion)으로 여러 검색 에이전트를 병렬 생성하고, 각 결과를 추론 과정을 거쳐 합성한 뒤 인용과 함께 반환한다. 복잡한 리서치 에이전트를 만들 때 검색 부분을 통째로 위임할 수 있다는 점이 핵심 가치다.

벤치마크: 속도와 정확도 동시 우위

Exa는 세 가지 벤치마크에서 Perplexity의 Sonar Reasoning Pro, Parallel Task API(Core, Base)와 비교 평가를 진행했다. 사용된 벤치마크는 HLE-Search, Deep Search QA, FRAMES로, 모두 단순 검색으로는 답하기 어려운 복합 질의를 다룬다. 결과는 Exa Deep이 모든 벤치마크에서 "더 빠르고 더 정확한" 위치(그래프에서 좌상단)를 차지했다고 발표했다.

다만 이 벤치마크는 Exa가 자체적으로 수행한 것이므로, 제3자 독립 검증이 뒤따르기 전까지는 참고 수준으로 봐야 한다. 그럼에도 Perplexity의 최상위 모델을 비교 대상으로 잡았다는 점은 자신감의 표현이고, 실제 사용 사례에서의 피드백이 이를 뒷받침할지 지켜볼 필요가 있다.

구조화된 출력과 필드 단위 근거

Exa Deep이 기존 딥 검색 도구들과 차별화되는 지점은 구조화된 출력(Structured Outputs)이다. API 요청 시 outputSchema를 정의하면, 자연어 답변이 아닌 JSON 구조로 결과를 받을 수 있다. 여기서 중요한 건 단순히 JSON을 뱉는 게 아니라, 각 필드마다 인용 출처와 신뢰도를 함께 제공하는 "필드 단위 근거(field-level grounding)"를 지원한다는 점이다.

예를 들어 "항공우주 기업 Top 리스트와 CEO 이름"을 요청하면, 각 CEO 이름 필드에 해당 정보의 출처 URL, 제목, 신뢰도(high/medium/low)가 달려 나온다. 이건 단순 편의 기능이 아니다. AI 에이전트가 검색 결과를 다음 단계 의사결정에 활용할 때, 어떤 정보를 신뢰할 수 있는지 프로그래밍적으로 판단할 수 있게 해준다. 환각(hallucination) 문제에 대한 구조적 방어선인 셈이다.

실제 사용 패턴과 가격

현재 Exa Deep의 주요 사용 패턴은 네 가지로 나뉜다. 금융 리서치(SEC 공시, 분기 보고서 등 다수 문서 합성), 과학 문헌 조사(전체 쿼리의 약 20% 차지), 뉴스 모니터링(다수 출처에 걸친 이슈 추적), 그리고 리서치 에이전트의 검색 기반 레이어로의 활용이다.

가격은 두 티어로 나뉜다. 일반 deep 모드는 4~12초 응답에 1,000건당 12달러, 추론 강화 deep-reasoning 모드는 12~50초에 1,000건당 15달러다. 일반 deep의 경우 이전 버전 대비 20% 가격 인하가 적용됐다. 건당 약 1.2~1.5센트 수준으로, 고품질 검색 결과에 인용과 구조화까지 포함된 점을 감안하면 직접 오케스트레이션을 구축하는 비용 대비 경쟁력이 있다.

더 큰 그림: 검색의 소비자가 인간에서 AI로 바뀌고 있다

Exa의 비전은 명확하다. "곧 AI가 인간보다 더 많이 웹을 검색하게 될 것이며, 이 AI들에게는 단순히 결과를 가져오는 게 아니라 추론할 수 있는 검색이 필요하다." 이건 과장이 아니라 이미 진행 중인 현실이다.

OpenAI, Anthropic, Google 모두 에이전트 프레임워크를 강화하고 있고, 이 에이전트들의 가장 빈번한 외부 행동이 바로 웹 검색이다. 기존 검색 API(Google Custom Search, Bing API 등)는 인간이 결과 페이지를 눈으로 훑는 것을 전제로 설계됐지만, AI 에이전트에게는 구조화되고, 근거가 명시되며, 프로그래밍적으로 소비 가능한 형태가 훨씬 효율적이다. Exa Deep은 정확히 이 "AI-native 검색 인프라" 포지션을 노리고 있다.

경쟁 구도도 흥미롭다. Perplexity는 소비자 대면 AI 검색에서 강세를 보이고 있고, Tavily는 에이전트용 검색 API로 빠르게 성장 중이며, Google은 Gemini에 검색 기능을 내장하는 방향으로 가고 있다. Exa의 차별점은 검색 자체를 에이전트화하면서도 API-first 접근을 유지한다는 점이다. 자체 소비자 제품보다는 다른 AI 에이전트의 "눈과 귀" 역할에 집중하겠다는 전략이다.

다만 이 시장의 본질적 리스크도 있다. OpenAI나 Anthropic 같은 대형 모델 제공자가 자체 검색 기능을 충분히 고도화하면, 별도 검색 API의 필요성이 줄어들 수 있다. Exa가 장기적으로 살아남으려면, 범용 모델이 쉽게 복제하기 어려운 수준의 검색 품질과 특화된 데이터 접근성을 유지해야 할 것이다.
