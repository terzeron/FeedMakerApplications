장고로 Test Driven Development(TDD) 하기

@angie
2024년 11월 18일 3:59 오전

Test Driven Development(TDD)는 요구사항을 명확히 하고 안정적인 코드를 작성할 수 있는 매우 효과적인 방법입니다. 장고에서는 TDD를 쉽게 할 수 있도록 다양한 테스트 기능을 지원합니다. 여기서는 뷰 테스트를 예시로 기본적인 장고의 TDD 과정을 설명해 보겠습니다.

1. 테스트 계획 및 작성

  • TDD의 첫 단계는 테스트 계획입니다. 요구사항을 정리하고, 개발해야 할 기능을 구체화한 후 테스트 코드를 작성합니다.
  • 장고에서는 TestCase 클래스를 상속받아 사용자 정의 테스트를 작성할 수 있습니다.

아래는 도서 데이터를 가져오는 API를 테스트하는 예제입니다.

# api/tests.py
from django.test import TestCase
from rest_framework.test import APIClient
from rest_framework import status
from .models import Book

class BookViewTest(TestCase):
    def setUp(self):
        self.client = APIClient()  # API 클라이언트를 불러옵니다.
        book_data = {
            "title": "test_title",
            "introduction": "this is introduction",
            "author": "test_author",
        }
        self.book = Book.objects.create(**book_data)  # 임의의 책 데이터를 생성합니다.

    def test_get_book_list(self):
        response = self.client.get('/api/book/')  # GET 요청으로 데이터 리스트를 가져옵니다.
        self.assertEqual(response.status_code, status.HTTP_200_OK)  # 응답 코드가 200인지 확인합니다.

2. 테스트 실행

작성한 테스트 코드를 실행해 테스트가 실패하는지 확인합니다. TDD 원칙에 따르면, 처음에는 테스트가 실패하는 것이 정상입니다.

python manage.py test

3. 기능 구현

테스트를 통과하기 위해 필요한 기능을 구현합니다. 여기서는 Book 모델에 대한 시리얼라이저, 뷰, URL을 작성했습니다.

# api/serializers.py
from rest_framework import serializers
from .models import Book

class BookSerializer(serializers.ModelSerializer):
    class Meta:
        model = Book
        fields = ["id", "title", "introduction", "author"]

# api/views.py
from rest_framework import viewsets
from .serializers import BookSerializer
from .models import Book

class BookViewSet(viewsets.ModelViewSet):
    queryset = Book.objects.all()
    serializer_class = BookSerializer

# api/urls.py
from django.urls import path
from rest_framework import routers
from .views import BookViewSet

router = routers.SimpleRouter()
router.register("book", BookViewSet)

urlpatterns = router.urls

# mybook/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api/', include('api.urls')),
]

4. 테스트 재실행

기능을 구현한 후, 다시 테스트를 실행해 모든 테스트가 성공적으로 통과하는지 확인합니다.

python manage.py test

5. 리팩토링

기능이 올바르게 동작하는 것을 확인한 후, 코드의 중복을 제거하거나 더 나은 구조로 개선하는 리팩토링 과정을 진행합니다. 리팩토링을 할 때마다 테스트를 실행해 코드가 여전히 정상적으로 동작하는지 확인해야 합니다.

6. 반복

다음 기능을 개발할 때는 다시 테스트 작성 단계로 돌아가서 TDD 과정을 반복합니다. 이를 통해 점진적으로 소프트웨어를 안정적으로 확장해 나갈 수 있습니다.
