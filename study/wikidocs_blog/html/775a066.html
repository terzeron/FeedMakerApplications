Avalonia UI 2026 — 3년이 지난 지금, 어디까지 왔을까?

@lukewire
2026년 3월 24일 5:48 오전
닷넷 크로스플랫폼 아발로니아 Avaloniaui

3년 전, Avalonia를 처음 소개하는 글을 짧게 쓴 적이 있다. 당시엔 "이런 프레임워크가 있다" 정도의 맛보기 수준이었는데, 그로부터 3년이 지난 지금 제대로 한번 정리해보려 한다.

Avalonia란?

Avalonia UI는 .NET 기반의 크로스플랫폼 데스크톱 UI 프레임워크다. XAML 문법을 사용하며, Windows · Linux · macOS에서 동일한 코드베이스로 동작한다. 모바일(Android, iOS)과 WebAssembly도 지원 범위에 포함되어 있다.

WPF를 써본 개발자라면 친숙하게 접근할 수 있다. 레이아웃 구조, 바인딩 방식, MVVM 패턴 모두 WPF와 매우 유사하다. 다만 "WPF의 복사본"이 아니라 독자적인 렌더링 엔진(Skia / Direct2D) 위에 새로 만들어진 프레임워크다.

2023년 당시의 기억

3년 전에 가장 인상적이었던 부분은 WPF처럼 XAML을 쓰면서도 Linux와 macOS까지 지원한다는 것이었다. 그것도 MIT 라이선스 무료로. .NET MAUI가 모바일 중심이라면, Avalonia는 데스크톱에 진심이었다.

그러나 당시엔 명확한 한계도 있었다.

  • VS, VSCode, Rider에서 XAML 미리보기가 불안정했음
  • 개발 도구(Previewer) 완성도가 낮았음
  • WPF 대비 생태계 빈약
  • 기업 도입 레퍼런스 부족
  • Prism 포팅 버전의 라이선스 불명확

요약하면 프레임워크 자체의 가능성은 충분했지만, 개발 경험(DX)이 따라오지 못했다.

3년간의 흐름

2023 — MIT 라이선스 전체 무료. VS/VSCode/Rider XAML 미리보기 불안정, 생태계 초기 단계.

2024 — Avalonia 11 정식 버전 안착. Prism.Avalonia v8 계열 등장. 커뮤니티 성장.

2025 — Avalonia Accelerate 발표. 개발 도구에 처음으로 라이선스 정책 도입. Prism.Avalonia v9 등장.

2026 — VSCode Extension 완전 재작성 및 Accelerate 편입. 월간 구독 옵션 추가. WebView 오픈소스화 예정(Avalonia 12).

Avalonia Accelerate란?

Avalonia 팀은 2025년, Avalonia Accelerate라는 개발 도구 패키지를 출시했다. 여기서 중요한 전제가 하나 있다.

Avalonia 프레임워크 자체는 여전히 MIT 라이선스 오픈소스다.
Accelerate는 프레임워크가 아니라, 그 위에서 개발 경험을 높여주는 "프로 툴링"이다.
즉, Accelerate의 라이선스 조건은 Avalonia로 개발할 때가 아니라, Accelerate의 도구와 컴포넌트를 사용할 때만 적용된다.

Accelerate는 크게 개발 도구(Tools)와 UI 컴포넌트(Components)로 나뉜다.

개발 도구 (Tools)

Visual Studio / VSCode Extension

기존 FOSS 버전을 버리고 완전히 밑바닥부터 재작성했다. 2026년 2월에는 VSCode Extension도 동일한 XAML 파서 엔진을 공유하도록 재작성됐다. 덕분에 VS에 추가되는 기능이 VSCode에도 동시에 반영되는 구조가 됐다. 3년 전 가장 아쉬웠던 XAML 도구의 불안정함을 정면으로 해결한 셈이다.

주요 개선 사항:
- IntelliSense 대폭 강화 — x:DataType Quick Info, Go To Definition 지원
- XAML 미리보기 안정성 개선, DPI 처리 및 Zoom to Fit 추가
- 에러 진단 개선 및 자동 xmlns 네임스페이스 임포트
- 이벤트 핸들러 자동 생성

참고로 VSCode Extension은 이제 Accelerate 구독이 필요하며, 더 이상 오픈소스가 아니다. 2023년 당시엔 VSCode 플러그인이 무료 MIT였다는 점을 감안하면 체감상 달라진 부분이다.

Dev Tools

앱 실행 중 UI 구조를 실시간으로 디버깅·인스펙션하는 독립 애플리케이션이다. 비주얼 트리 검사, 바인딩 디버깅, 레이아웃 확인 등을 지원한다. 기존 FOSS 버전의 업데이트가 아닌 완전히 새로 만들어진 별도 앱이다.

Parcel

크로스플랫폼 앱 패키징을 위한 전용 도구다. GUI와 CLI 두 가지 방식을 모두 지원하며, Windows · macOS · Linux 배포 패키지를 단일 환경에서 만들 수 있다. 코드 서명과 notarization도 처리해준다. 크로스플랫폼 앱 배포가 항상 까다로운 작업이었다는 점에서 실용적인 도구다.

UI 컴포넌트 (Components)

Accelerate에는 도구 외에도 유료 UI 컴포넌트들이 포함되어 있다.

  • WebView — Chromium을 번들링하지 않고 플랫폼 네이티브 웹 렌더링 엔진을 사용하는 웹 뷰 컴포넌트
  • Media Player — Windows · macOS · Linux 네이티브 미디어 재생 컴포넌트
  • Markdown Viewer — 마크다운 렌더링 컨트롤
  • Virtual Keyboard — 키오스크·터치 환경을 위한 온스크린 키보드 (80개 이상 언어 레이아웃 지원)
  • TreeDataGrid — 기존 FOSS 버전 대비 개선된 트리 데이터 그리드

Accelerate 라이선스 구조

Accelerate의 도구와 컴포넌트를 사용하려면 라이선스가 필요하다.

티어 대상 비용
Community 개인 개발자, 소규모 조직(연매출 €100만 미만), 교육기관 무료
Business 250인 이상 또는 연매출 €100만 초과 기업 유료 (Named User)
Startup Program 창업 5년 미만, 매출/투자 €1000만 미만 기업 Business 50% 할인

사용자의 약 67%가 Community 라이선스 조건에 해당한다고 한다.

12개월 구독 시 영구 라이선스 구조도 있다. 구독 취소 후에도 이미 빌드된 앱은 그대로 동작한다. 런타임 라이선스 체크가 없으므로 최종 사용자에게는 영향이 없다.

Community 라이선스에는 익명 사용 텔레메트리 수집이 포함된다. 소스코드나 개인정보는 수집하지 않으며 JetBrains Rider와 유사한 방식이라고 명시하고 있다.

구버전 MIT 라이선스 도구들(구 VS Extension, 구 Dev Tools)은 여전히 GitHub에서 사용 가능하다. 단, 신규 기능 개발은 Accelerate 버전에만 집중될 예정이다.

Prism.Avalonia의 현재 상황

WPF 생태계에서 Prism은 사실상 표준 아키텍처 프레임워크다. MVVM, 의존성 주입, 모듈화, EventAggregator, 리전 네비게이션까지 대규모 엔터프라이즈 앱에 필요한 구조를 제공한다.

Avalonia용 Prism은 커뮤니티 포크인 Prism.Avalonia로 시작해 현재 v9.0까지 왔다. Avalonia 11.x를 지원하며 MIT 라이선스로 배포된다.

한 가지 주의할 부분이 있다. Prism Library 본체(WPF, MAUI 등을 위한 공식 버전)가 v9.0부터 이중 라이선스 체계로 전환됐다. 개인·소규모 팀은 Community License로 무료지만, 일정 규모 이상 기업은 Commercial License가 필요하다.

Prism.Avalonia는 커뮤니티 포크이므로 현재는 별개의 MIT 라이선스를 유지하고 있다. 다만 공식 Prism 팀과의 통합 논의가 있었던 만큼, 앞으로 통합이 이루어진다면 라이선스 구조가 변동될 여지는 있다. 기업 프로젝트라면 패키지별 라이선스를 주기적으로 확인하는 것이 좋다.

Accelerate 도입, 기업 입장에서 바라보면

솔직히, 아직은 조심스럽다는 생각이 든다.

Community 티어 조건에 해당하는 팀이라면 Accelerate 도구들을 사실상 무료로 쓸 수 있다. 문제는 엔터프라이즈 규모다. 연매출 €100만이 넘거나 250인 이상이면 Business 라이선스가 필요하다. 도구에 비용을 지불하는 것 자체가 문제가 아니라, 라이선스 정책이 앞으로 어떻게 변할지에 대한 불확실성이 부담으로 남는다.

WPF 기반 레거시가 많은 기업이라면 굳이 옮길 유인도 적다. WPF는 Windows 내부 도구에는 여전히 충분하고, 익숙한 아키텍처가 있다면 더욱 그렇다.

반면 처음부터 멀티 플랫폼이 필요한 신규 프로젝트라면 Avalonia는 진지하게 검토할 만한 선택지다. .NET MAUI와 달리 데스크톱에 무게중심이 있고, Skia 렌더링 기반으로 플랫폼 일관성도 높은 편이다.

결론

3년 전의 질문과 달라졌다. 그때는 "쓸 수 있을 만한가?"였다면, 지금은 "어떤 상황에서 선택하는 게 맞는가?"가 더 적절한 질문이다.

  • 신규 크로스플랫폼 데스크톱 앱 → Avalonia는 진지한 후보다
  • WPF 레거시 마이그레이션 → 조심스럽게 접근. 1:1 변환은 없으며 아키텍처 레이어도 재검토 필요
  • 개인·소규모 팀 → Community 라이선스로 Accelerate 도구도 무료. 3년 전과는 개발 경험이 다르다
  • 엔터프라이즈 기업 → Accelerate Business 라이선스 비용과 향후 정책 변동 리스크를 함께 고려해야 한다
  • Prism 아키텍처를 그대로 가져오고 싶다면 → Prism.Avalonia는 옵션이 되지만, 라이선스 상태를 주기적으로 확인하는 것이 좋다

프레임워크 자체는 계속 MIT다. Accelerate는 그 위에서 개발 경험을 높여주는 도구이고, 이 도구를 쓰려는 경우에만 라이선스 조건이 생긴다. 3년 전보다 Avalonia는 분명히 더 나은 선택지가 됐다. 도구가 성숙했고, 라이선스도 방향을 잡았다. 이제 남은 건 각자의 프로젝트와 팀 상황에 맞는 판단이다.

2026년 3월 기준. Avalonia Accelerate 라이선스 세부 내용은 공식 문서 참고.
