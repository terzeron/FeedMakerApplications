[macOS] launchd/launchctl 완벽 가이드 - 슬립모드와 정전에도 끄떡없는 스케줄링

0. 들어가며

기존에는 Windows 작업 스케줄러로 자동화 작업들을 관리해왔다. 최근 Windows 10 기술지원 종료와 개발환경이 macOS에 더 친화적으로 변하면서 전환하게 됐고, 기존 스크립트들을 마이그레이션해야 했다. macOS의 launchd와 launchctl을 사용해봤지만 GUI도 없고 설정도 복잡한데다 제대로 동작하지 않는 경우가 많았다. 이 글은 슬립 모드, 화면 잠금, 정전 등의 상황에서도 안정적으로 실행되도록 launchd를 제대로 설정하는 방법을 정리한 것이다. 물론 Claude에 물어봐서 해결했다. 좋은 세상이다.

1. LaunchDaemon plist 파일 작성

매일 동일한 시간 실행과 슬립/잠금과 무관하게 실행하려면 LaunchDaemon을 사용해야 합니다:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.yourname.dailyscript</string>

    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>/path/to/your/script.sh</string>
    </array>

    <!-- 매일 오전 9시에 실행 -->
    <key>StartCalendarInterval</key>
    <dict>
        <key>Hour</key>
        <integer>9</integer>
        <key>Minute</key>
        <integer>0</integer>
    </dict>

    <!-- 부팅 시 자동 실행 -->
    <key>RunAtLoad</key>
    <true/>

    <!-- 종료되면 자동 재시작 (선택사항) -->
    <key>KeepAlive</key>
    <false/>

    <!-- 표준 출력/에러 로그 -->
    <key>StandardOutPath</key>
    <string>/tmp/dailyscript.log</string>
    <key>StandardErrorPath</key>
    <string>/tmp/dailyscript.err</string>
</dict>
</plist>

2. 파일 배치 및 권한 설정

# LaunchDaemon 디렉토리에 배치 (시스템 레벨)
sudo cp com.yourname.dailyscript.plist /Library/LaunchDaemons/

# 권한 설정
sudo chown root:wheel /Library/LaunchDaemons/com.yourname.dailyscript.plist
sudo chmod 644 /Library/LaunchDaemons/com.yourname.dailyscript.plist

# 스크립트 실행 권한
sudo chmod +x /path/to/your/script.sh

3. 로드 및 실행

# 로드
sudo launchctl load /Library/LaunchDaemons/com.yourname.dailyscript.plist

# 즉시 실행 (테스트용)
sudo launchctl start com.yourname.dailyscript

# 상태 확인
sudo launchctl list | grep yourname

# 로그 확인
tail -f /tmp/dailyscript.log

4. 슬립모드 대응

macOS는 기본적으로 슬립 중에는 예약 작업을 실행하지 않습니다. 이를 해결하려면:

# 전원 어댑터 연결 시 자동 깨우기 활성화
sudo pmset -c sleep 0
sudo pmset -c disksleep 0

# 또는 스케줄 깨우기 설정
sudo pmset repeat wakeorpoweron MTWRFSU 08:55:00

plist에 추가할 옵션:

<key>LaunchOnlyOnce</key>
<false/>

5. 정전 후 자동 재부팅

# 전원 복구 시 자동 부팅 설정
sudo systemsetup -setrestartpowerfailure on

# 확인
sudo systemsetup -getrestartpowerfailure

6. 트러블슈팅

동작하지 않는 경우 체크 사항:

# 1. plist 문법 오류 확인
plutil -lint /Library/LaunchDaemons/com.yourname.dailyscript.plist

# 2. 시스템 로그 확인
log show --predicate 'subsystem == "com.apple.launchd"' --last 1h

# 3. 언로드 후 재로드
sudo launchctl unload /Library/LaunchDaemons/com.yourname.dailyscript.plist
sudo launchctl load /Library/LaunchDaemons/com.yourname.dailyscript.plist

# 4. 직접 스크립트 실행해서 에러 확인
sudo /bin/bash /path/to/your/script.sh

7. 주의사항

  • LaunchAgent vs LaunchDaemon: 사용자 로그인 없이 실행하려면 반드시 LaunchDaemon 사용
  • PATH 환경변수: 스크립트에서 전체 경로 사용 (/usr/bin/python3 대신 상대 경로 피하기)
  • 권한: 파일 접근 권한 문제가 가장 흔한 원인
  • macOS 보안: Catalina 이후 Full Disk Access 권한 필요할 수 있음
