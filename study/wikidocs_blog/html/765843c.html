[SV] Class 의 Polymorphism (다형성)

1. SystemVerilog 에서의 Polymorphism

  • Object Oriented Programming(OOP) 에서 Polymorphism(다형성) 은 하나의 인터페이스로 여러 가지 형태의 동작을 구현할 수 있는 특성을 이야기 함
  • SystemVerilog 에서의 polymorphism 은 superclass type의 variable 을 사용하여 subclass 의 object 를 reference 할 때, superclass 의 variable 을 사용하여 subclass 의 method 를 access 할 수 있도록 함으로써 구현 된다
  • 이렇게 하기 위해서는 class 에 정의된 method 를 virtual 로 선언해야 함
  • virtual 로 선언되지 않은 method 는 subclass object 를 superclass type variable로 referencing 할 때, superclass 에서 선언된 method 를 access 하게 됨
  • 그래서 SystemVerilog 의 Polymorphism 은 base class 로 만들어 놓은 infra 를 extend 된 class 들이 extended 된 class 의 기능을 가지고 이용하도록 하는 효과를 제공함
  • 이 특징은 재사용, 확장성 측면에서 매우 유용한 benefit 을 제공함

2. SV Class polymorphism 이해

  • Base class 와 extend 된 class 가 아래와 같이 있다고 가정한다
  • method_a() 는 non-virtual 이고, method_b() 는 virtual 로 선언되어 있다

  • 그리고 method_a() 와 method_b() 모두 extend 된 class 에서 재정의 하였다

    [ BasePacket 과 NewPacket 의 정의 ]

  • 각각 BasePacket 과 NewPacket 의 type handle 로 생성된 object 를 reference 한다면 아래와 같이 access 된다

    [ Class 들의 생성과 Access ]

  • 만약, BasePacket type 의 variable, 즉 handle 로 NewPacket 의 object 를 referencing 한다면, method 의 type 에 따라서 아래의 그림과 같이 handle 로 reference 해서 access 할때 virtual 과 non-virtual 은 다르게 access 된다

  • Non-virtual type 인 method_a()는 BP.method_a() 로 access 하면, BasePacket 에서 정의된 method_a() 가 access 되고
  • Virtual type 인 method_b() 는 BP.method_b() 로 access 하면, NewPacket 에서 정의된 method_b() 가 access 된다
  • Virtual 로 정의된 method는 이렇게 superclass 에서 정의된 method 가 subclass의 method로 override 되어지고, 이는 polymorphism 의 핵심 특징으로 사용된다

3. SV Class Polymorphism 응용

  • Polymorphism 은 다음과 같이 응용된다

  • 아래의 그림과 같이 BasePacket 에서 ErrPacket, NewPacket, 그리고 NewPacket 에서 NewErrPacket 이 extend 되고, 각 class에서 virtual 인 method_a() 가 재정의 되었다고 가정하자

    [ Class의 확장과 method override ]

  • 첫번째 경우로, BasePacket type 으로 BP_Array 가 5개의 array variable 로 선언된 경우, BP_Array[n] 은 BasePacket type 의 handle 이다

  • 이때 BasePacket, NewPacket, ErrPacket, NewErrPacket type 으로 handle 을 선언하고, 각각 object 를 생성하여 assign 하고, BP_Array[0] ~ BP_Array[3] 까지 assign 한다. BP_Array[n] element 은 base class 이므로 extended class 들 object 를 assign 하는 것은 문제 없는 assign 이다
  • 그리고, 그리고, BP_Array[n].method_a() 를 호출하면 각각 element 에 연결되어 있는 object 의 class type 에서 정의된 method_a() 들이 access 된다

  • 이는 BP_Array[n] 이라는 하나의 type 의 인터페이스로 여러 type 형태의 동작을 구현할 수 있게 해 준다

    [ BasePacket type array에 응용 ]

  • 두번째 경우로, BasePacket type 을 argument 로 하는 trans_pkt() 라는 task 가 구현되어 있다고 가정하고, task 내에서는 argument 인 PKT의 method_a() 를 호출해서 사용하는 경우를 생각해 보자

  • BasePacket, NewPacket, ErrPacket, NewErrPacket type 의 handle 을 선언하고, 각각의 type별로 object 를 생성하여 assign 되었다고 하자
  • 그리고 각 type의 handle 을 tran_pkt() 에 argument 로 전달할 때, tran_pkt() 내에서 실행되는 PKT.method_a() 는 전달되는 class 의 type 에 따라서 각 type에서 정의된 method_a() 가 실행되게 된다

  • 즉, trans_pkt( BasePacket PKT ) 라는 하나의 인터페이스를 사용하여 여러 형태의 동작 기능이 구현될 수 있도록 해 주는 것이다

    [ BasePacket type argument의 trans_pkt() 활용 ]

  • 이와 같이 SystemVerilog 의 Class 에서 polymorphism 은 virtual method 를 사용하여 구현되며, 이는 재사용성, 확장성 활용을 극대화 시켜주며, UVM 등의 methodology 를 제공할 수 있는 특징이 된다
