Weights & Biases(W&B)는 머신러닝 실험을 추적하고 관리할 수 있는 플랫폼
wandb 맞음 (파이썬에서 그렇게 부름)

특히 파라미터 튜닝, 모델 아키텍처 변경, 데이터 전처리 방식 변경 등 다양한 실험 과정에서 손쉽게 로그를 남기고 성능을 비교할 수 있다.

W&B는 GitHub, Colab, PyTorch, TensorFlow와 같은 인기 있는 툴과의 연동을 지원하여 실험 관리에 필요한 통합적인 환경을 제공

1. Experiment Tracking

모델 학습 과정에서 다양한 실험 정보를 기록하고 추적하는 기능
모델의 학습 로그(train loss,val accuracy 등)을 실시간으로 저장하고 시각화 함
이를 통해서 실험 중 성능을 확인하고 비교 가능, 학습이 제대로 되는지도 확인 가능

1-1. run

w&b에서 Run은 기본 실험 단위, 하나의 모델 학습 세션이다.
각 Run은 모델이 특정 하이퍼파라미터,설정을 통해 학습,평가되는 한번의 실험을 의미한다.
실험중 발생하는 다양한 데이터를 기록하고 시각화 비교 가능

run 주요기능
- 로그 기록 및 시각화
- 하이퍼파라미터 및 메타데이터 기록
- 모델 체크포인트 및 파일 관리
저장 관리가 쉬우며, 이전 상태로 모델을 복원하는것도 가능하다. 또한 배포도 가능함
- 체계적인 실험 관리
여러 실험을 관리하는데 효율적이다. 각 Run별로 각각 저장되기에

run 구현하기

  1. Run 생성하기 
wandb.init()
  1. 파라미터 설정 - dictionary 형태로 작성 
config = { "epochs" : 50, 
"num_layer" : 2, 
"depth_1" : 32, 
"depth_2" : 64, 
"fc_size" : 512, 
"lr" : 0.001 }
  1. 파라미터 설정 입력하기 
wandb.config=config 
wandb.init(config=config)

생성한 run에 config로 입력해도 되고, 최초 생성시 입력을 해도 된다.
이후에 수정도 가능하다
4. 사용시 불러오는법

config=wandb.config
config.epochs

이제 학습코드를 동작시킨다
각 run에는 프로젝트 명이 필요하며, 입력하지 않으면 uncategorized로 분류됨

wandb.init(project = "프로젝트명", config = config) 
# 모델 학습 과정에서 로그 기록 
for epoch in range(5): 
    # 학습 코드는 생략 
    # W&B에 기록 
    wandb.log({"train_loss": train_loss, "val_loss": val_loss}) 
# Run 종료 
wandb.finish()

log의 경우 매 iter 마다 실행시키면 step에 맞춰서 대시보드에 기록하고,그래프도 그려준다.
또한 데이터가 아닌 표,이미지 등 미디어로 저장도 가능

1-2 Sweep

Sweep은 w&b에서 제공하는 하이퍼파라미터 최적화기능
하이퍼파라미터 공간을 지정하고, 다양한 값의 조합을 시도하며 최적의 하이퍼파라미터를 찾는다
옛날에 배웠던 grid search 개념인듯

실험에 중요한 하이퍼파라미터를 설정하고 sweep을 실행하면
실험을 자동으로 관리하고 결과를 추적한다

sweep agent

하이퍼 파라미터 튜닝을 위해선 2가지가 필요하다
1. Sweep configure
조정할 하이퍼 파라미터와 각 파라미터의 범위를 설정
grid,random에 더불어 bayes 방식도 지원
bayes는 주어진 조합 내에서도 더 효과적인 조합을 찾아간다.
2. Train function
각 파라미터의 조합이 결정되면 이를 반복수행할 함수
각 run이 시작할때 반복할것과 아닌것을 구분하여 작성해야함
예를 들면 데이터로더를 불러와 배치단위로 자르고 하는 준비단계는 스킵
모델은 새로 불러와야 파라미터가 초기화됨으로 포함

sweep agent는 이 2가지를 받아서 동작하는 객체다.
반복 종료 조건 or 임의로 중지할때 까지 반복을 수행

sweep config

예시 

sweep_config = {
    "method": "bayes",
    "metric": {
        "name": "val_loss",
        "goal": "minimize"
    },
    "parameters": {
        "epoches": {
            "value": 50
        },
        "num_layer": {
            "values": [1, 2, 3]
        },
        "depth_1": {
            "values": [16, 32, 64]
        },
        "depth_2": {
            "values": [32, 64, 128]
        },
        "fc_size": {
            "values": [256, 512, 1024]
        },
        "lr": {
            "distribution": "log_uniform",
            "min": -6,
            "max": -2
        }
    },
    "early_terminate": {
        "type": "hyperband",
        "max_iter": 27,
        "s": 2
    }
}
