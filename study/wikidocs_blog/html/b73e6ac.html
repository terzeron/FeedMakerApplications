iptable 작성방법

@seedwiki
2025년 1월 16일 2:08 오후

한 개의 IP 차단

iptables -A INPUT -s xxx.xxx.xxx.xxx -j DROP

IP 대역 차단

iptables -A INPUT -t filter -s xxx.xxx.xxx.xxx/yy -j DROP

IP 차단 해제

iptables -D INPUT -s xxx.xxx.xxx.xxx -j DROP

IP 차단 목록 보기

iptables -L

목록 전체 삭제

iptables -F INPUT

난 항상 사용을 해보고 의미를 찾는 이상 한 방식으로 일을 하고 있는데 그래서 어려운가 보다....
공부를 하고 사용하면 그러려니 사용하겠는데 모른체로 사용하고는 두려움에 떨다가 공부를 하다니...나쁜 버릇이다.

iptables 의 개념 : 방화벽 설정 도구. 리눅스의 사용자공간 응용프로그램이다. 운영체제단에서 방화벽을 운영할 때 사용한다.
로그도 남겨야지...그래야 잘 막앗는지 알지.

iptables -A INPUT -j LOG // 이건 아마도 전체 로그 추가 하는 방법이고

iptables -l INPUT -p tcp -s 192.168.1.10 --dport 22 -j LOG --log-prefix "192.168.1.10 To SSH " eg) 192.168.1.10 pc에서 ssh포트 22번 의 접근을 기록하는 설정
대역의 로그를 기록하기 위해서는
iptables -A INPUT -s 192.168.1.0/24 -j LOG

iptables -A INPUT -s 192.168.1.0/24 -j LOG --log-level 4 이걸 쓰면 로그의 레벨별로 기록 할 수 있다.

로그를 기록했으니 어디서 보냐??

ubuntu 같으면 tail -f /var/log/kern.log 라는디...kern 이 안생기고 messages 만 있더라구
centos /fedora /var/log/messages

로그 설정 파일은
/etc/rsyslog.d/iptables.conf 또는 syslog.conf

설정하면 재시작해야지 service rsyslog restart

netfilter 라는 프로젝트에서 개발됨.

tcp 80 port연결 요청(syn)에 대하여 동일 출발지 ip로 동시 연결개수가 5개 초과시 이를 차단한다.
iptables -A INPUT -p TCP --dport 80 --syn -m connlimit --connlimit-above 5 -j DROP

이런식으로 보호할 수 있다고한다. .... TCP SYN Flooding 공격에 대한 제어 방법 이라고 하네 .

또는 아이피 테이블 설정 1초에 10회 이상의 요청에 대해 차단하는 방법

iptables -A Input -p tcp --dport 80 -m limit --limit 10/s -j Drop
