@Veri_Fun

검증 Class 시스템베릴로그 Systemverilog Verification 회로

1. Class 설명

  • SystemVerilog 에서의 Class 는 하나의 type 으로서 data 와 그 data 들을 처리하는 subroutine 들을 가진다.
  • Class 의 data 들은 property 들이라고 부르고
  • Class 의 subroutine 들은 method 들이라고 부른다.
  • 이 둘을 합쳐 member 들이라고 칭함.
  • 이들 member 들을 사용하여 어떠한 객체(Object) 의 내용과 기능을 정의하게 된다.
  • Class 가 생성되어 있는 형태를 Object 라고 이야기 함
  • Object 는 Simulation run time 중에 dynamic 하게 생성되고 소멸됨
  • Object 는 object handle ( object 를 가리키는 포인터 ) 로 instance 되며, 이 handle 은 port 들 처럼 argument 로서 전달될 수 있음 => 매우 중요
  • Object handle 의 assign 이나, copy 는 object 내용의 copy가 아니라, object를 가리키는 point 의 copy 를 의미함

2. Class 의 정의

  • Class 의 정의는 다음과 같이 class ~ endclass 로 이루어 짐 
class <class name>;
  ...
  function new();
  endfunction
  ...
endclass
  • 실제 Packet 이라는 class 를 정의하면 아래와 같이 할 수 있음
class Packet ;
  //class property 들, 필요한 data의 정의
  bit [3:0] command;
  bit [40:0] address;
  bit [4:0] initiator_id;
  integer time_requested;
  integer time_issued;
  integer status;
  typedef enum {ERR_OVERFLOW = 10, ERR_UNDERFLOW = 1123} PCKT_TYPE;
  const integer buffer_size = 100;
  const integer header_size;

  // new() 생성자, initialization 에 사용
  function new();
    command = 4'd0;
    address = 41'b0;
    initiator_id = 5'bx;
    header_size = 10;
  endfunction

  // 필요한 method 들
  // public 하게 access 하는 entry point 역할을 함
  task clean();
    command = 0; address = 0; initiator_id = 5'bx;
  endtask

  task issue_request( int delay );
  // send request to bus 의 내용을 기술할 수 있음
  endtask

  function integer current_status();
    current_status = status;   // function name은 return type의 variable로 사용됨
  endfunction
endclass

3. Class 의 instance 와 생성

  • SystemVerilog 에서의 class 는 하나의 data type 이며, class 의 instance 를 하나의 object(객체) 라고 칭함
  • Object 의 사용은 아래와 같이 이루어 짐
  • 해당 class type 의 variable 선언 ( handle 을 선언 함 )
  • new() 생성자를 사용하여 해당 class 의 내용을 갖는 object 를 생성함 
Packet p;  // class Packet type 의 variable(handle) p 를 선언함

p = new();  // Class Packet type 의 object를 생성하고 p 에 연결 함


class 의 instance 와 생성

  • Packet type 의 class object 를 p 라는 handle 로서 가리키게 됨
  • 생성자 new() 없이 선언만 되어 있으면 handle 에는 "null" 값을 갖게 됨
  • Compile 시 또는 Simulation 시 "null object handle" 이 나오면 이 상태를 나타내는 것과 관련됨

Class 의 생성자 new()

  • SystemVerilog의 class 에서 new() 는 class 생성자 라고 정의함
  • new() 는 특별한 function 으로 return type 은 정의되지 않으나, object creation 시에 <handle> = new() 형태로 call이 됨
  • new() 는 class 생성할 때 object 생성된 이후 자동으로 call이 되고,
  • 이 new() 내부에 생성시 특별한 동작을 기술하여 사용할 수 있음 ( ex. property 들의 초기화 등 )
class C;
  int c1 = 1;
  int c2 = 1;
  int c3 = 1;

  function new(int a);
    c2 = 2;   // c2 는 최종 이 값으로 초기화됨
    c3 = a;   // 생성시 new(a) 로 argument 준 값으로 초기화 됨
  endfunction
endclass

4. Class 의 object handle assignment

  • Class 의 object handle 은 variable 들이며, 이들의 assign 은 object 의 copy 를 의미하지 않고, handle 값의 assign 임
  • 아래의 그림에서 처럼 p1, p2 가 생성된 상태에서 p2 = p1; 동작은 handle 의 pointer 변경과 같다.


class object handle 의 assignment

5. Class member 들의 access 방법

  • Class 내부의 member 들은 non-static member 들인 경우, 그 access 는 object handle 을 사용하여 access 함
  • Propertie 들과 method 들은 <object handle name>.<member> 형태로 access 함 
Packet p = new();

int var1;
p.command = INIT;     // property access
var1 = p.buffer_size;

status = p.current_status();  // method access

6. Class 의 Static Members

  • Class 의 object 가 생성되면 해당 object 마다 class 에서 정의한 member들 ( property들 과 method들 )이 생기며, object 마다 가지고 있게 됨
  • 때로는 모든 object 들이 공유하는 오직 하나만 존재하는 member 들이 필요한 경우가 있음
  • 이때 이러한 member 들 앞에 "static" 를 사용하여 정의하면 class 당 1개만 존재하는 member를 정의할 수 있음
  • Static member 들은 object 를 생성하는 절차 없이 사용이 가능함
  • Static method 는 non-static member 들을 access 할 수 없음

Static member 의 access 는 <class name>::<member> 로 access 함

class abc;
  static int prop_a = 3;
  int prop_b = 4;
  int prop_c = 5;

  static function method_a();
    $display("method_a() is accessed");
  endfunction
endclass


module tb_top;
  initial begin
    abc obj_abc = new();
    $display("abc's prop_a = %d", abc::prop_a);  // static peoperty 엑세스
    abc::method_a();  // static method 엑세스
    $finish;
  end
endmodule
  • Static member 들이 있는 class 의 경우 아래와 같이 생성되어 사용된다고 이해할 수 있음
