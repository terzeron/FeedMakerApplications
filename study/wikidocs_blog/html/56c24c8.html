📊 시각화: Histogram filter localization

위 시뮬레이션에서, 우리는 로봇의 yaw orientation과 RFID의 위치는 알고 있지만, x,y 위치는 모른다고 가정합니다.

이 filter는 RFID로부터 speed input과 observations 범위를 사용하여 localization을 수행합니다.

초기 위치 정보는 필요하지 않습니다.

🤖 로봇에서 RFID는?

로봇이 위치를 추정할 때, 공간 내에 고정된 RFID 태그들이 있으면,
로봇이 태그에 가까이 갔을 때 태그 신호의 세기 또는 거리를 측정하여
"내가 대략 이 근처에 있다"는 정보를 얻을 수 있습니다.

이게 바로 Histogram Filter나 Particle Filter에서 센서 측정값으로 활용됩니다.

📌 정리

  • RFID는 무선으로 물체를 인식하는 기술
  • 로봇에서는 환경에 설치된 RFID 태그를 기준으로 위치 추정에 활용
  • GPS를 사용할 수 없는 실내에서 유용

코드: histogram_filter.py

🧠 1. Histogram filter localization?

Histogram Filter Localization은 로봇이 자신의 위치를 추정할 때 사용하는 확률 기반(localization) 기법 중 하나로, 로봇의 상태(위치와 방향)를 이산적인 격자(grid) 형태로 표현하여 각 위치에서 로봇이 있을 확률을 업데이트합니다. 이는 Bayes 필터(Bayes Filter)의 한 형태로, 공간을 연속적으로 다루지 않고 격자 셀(cell) 단위로 나누어 계산하는 것이 핵심입니다.

📌 핵심 개념 요약

항목 설명
기반 이론 Bayes 필터
공간 표현 격자(Grid), 이산화된 상태 공간
특징 각 셀에 대해 로봇이 그 셀에 있을 확률을 저장
장점 비선형 모델, 복잡한 환경에서도 적용 가능
단점 고해상도 격자를 쓸수록 계산량 증가

✅ 장점

  • 불확실한 환경(센서 오류, 복잡한 맵)에서도 강인함
  • 비선형, 비가우시안 분포도 표현 가능
  • 비교적 단순한 개념으로 구현 쉬움

❌ 단점

  • 격자 크기가 작아질수록 계산량 급증 (고해상도일수록 느림)
  • 연속적인 공간 표현에는 부적절
  • 3차원 이상 공간에서는 차원의 저주(curse of dimensionality) 발생

🧠 작동 원리

  1. 초기화 (Initialization)

    • 전체 공간을 격자 셀로 나누고, 각 셀에 로봇이 있을 확률을 균등하거나 초기 센서 정보 기반으로 설정합니다.

  2. 예측 단계 (Prediction / Motion Update)

    • 로봇의 움직임 모델(예: odometry)을 이용해, 현재 위치 확률 분포를 이동시킵니다.
    • 각 셀의 확률을 움직임 불확실성을 고려해 인접 셀로 확산시킴.

  3. 갱신 단계 (Correction / Measurement Update)

    • 센서 측정(예: 거리 센서, LiDAR)을 바탕으로 각 셀에서의 측정값과 실제 센서값을 비교하여, 일치하는 정도에 따라 확률을 조정합니다.

  4. 정규화 (Normalization)

    • 모든 셀의 확률 합이 1이 되도록 정규화합니다.

Filtering algorithm

  • Histogram filter는 연속 공간에서의 discrete Bayes filter입니다.
  • 로봇의 존재 확률을 관리하기 위해 규칙적인 그리드를 사용합니다.
  • 그리드의 확률이 높을수록 로봇이 그곳에 있을 가능성이 높다는 것을 의미합니다.
  • 해당 시뮬레이션에서는 x-y 위치를 추정하고자 하므로 2D 그리드 데이터를 사용합니다.
  • histogram filter가 확률 분포를 추정하는 데는 4단계가 있습니다.

Step1: Filter initialization

  • Histogram filter에는 초기 위치 정보가 필요하지 않습니다.
  • 이 경우, 각 그리드 확률을 같은 값으로 초기화할 수 있습니다.
  • 초기 위치 정보를 활용할 수 있다면, 이를 기반으로 초기 확률을 설정할 수 있습니다.
  • 초기 위치 정보가 gaussian distribution으로 제공되는 경우 Gaussian grid map 가 유용할 수 있습니다.

Step2: Predict probability by motion

  • Histogram filter에서 로봇이 다음 격자로 이동할 때 각 격자의 모든 확률 정보가 이동 방향으로 이동합니다.
  • 이 과정은 로봇이 움직일 때 확률 분포가 변화하는 것을 나타냅니다.
  • 로봇이 움직인 후에, 확률 분포가 움직임으로 인한 estimation error를 반영해야 합니다.

Step3: Update probability by observation

  • 이 단계에서는 모든 확률이 관찰 결과에 따라 업데이트됩니다. 이는 bayesian filter의 업데이트 단계입니다.
  • 확률 업데이트 공식은 사용된 센서 모델에 따라 다릅니다.
  • 이 시뮬레이션은 range observation model을 사용합니다.
  • 다음 수식에 의해 각 그리드의 확률이 업데이트 됩니다.

$$
p_t = p_{t-1} * h(z)
$$

$$
h(z) = \frac{\text{exp}(-(d - z)^2 / 2 )}{\sqrt{2\pi}}
$$

  • $p_t$는 time $t$에 대한 probability
  • $h(z)$는 observation $z$에 대한 observation probability
  • $d$는 RD-ID에서 grid center까지의 known distance

Step4: Estimate position from probability

  • 각 시간 단계에서 현재 확률 분포로부터 최종 로봇 위치를 계산할 수 있습니다.
  • 최종 위치를 계산하는 2가지 방법이 있습니다:
    • maximum probability grid position을 사용하는 방법
    • average of probability weighted grind position을 사용하는 방법

def histogram_filter_localization(grid_map, u, z, yaw):
    grid_map = motion_update(grid_map, u, yaw)

    grid_map = observation_update(grid_map, z, RANGE_STD)

    return grid_map

def main():
    print(__file__ + " start!!")

    # RF_ID positions [x, y]
    RF_ID = np.array([[10.0, 0.0],
                      [10.0, 10.0],
                      [0.0, 15.0],
                      [-5.0, 20.0]])

    time = 0.0

    xTrue = np.zeros((4, 1))
    grid_map = init_grid_map(XY_RESOLUTION, MIN_X, MIN_Y, MAX_X, MAX_Y)
    mx, my = calc_grid_index(grid_map)  # for grid map visualization

    while SIM_TIME >= time:
        time += DT
        print(f"{time=:.1f}")

        u = calc_control_input()

        yaw = xTrue[2, 0]  # Orientation is known
        xTrue, z, ud = observation(xTrue, u, RF_ID)

        grid_map = histogram_filter_localization(grid_map, u, z, yaw)

        if show_animation:
            plt.cla()
            # for stopping simulation with the esc key.
            plt.gcf().canvas.mpl_connect(
                'key_release_event',
                lambda event: [exit(0) if event.key == 'escape' else None])
            draw_heat_map(grid_map.data, mx, my)
            plt.plot(xTrue[0, :], xTrue[1, :], "xr")
            plt.plot(RF_ID[:, 0], RF_ID[:, 1], ".k")
            for i in range(z.shape[0]):
                plt.plot([xTrue[0, 0], z[i, 1]],
                         [xTrue[1, 0], z[i, 2]],
                         "-k")
            plt.title("Time[s]:" + str(time)[0: 4])
            plt.pause(0.1)

    print("Done")

1. 그리드 맵 초기화 (init_grid_map)

공간을 0.5m 해상도의 격자로 나눠 각 셀에 확률을 균등하게 초기화합니다.
초기 확률 분포는 정규화됩니다

def init_grid_map(xy_resolution, min_x, min_y, max_x, max_y):
    grid_map = GridMap()

    grid_map.xy_resolution = xy_resolution
    grid_map.min_x = min_x
    grid_map.min_y = min_y
    grid_map.max_x = max_x
    grid_map.max_y = max_y
    grid_map.x_w = int(round((grid_map.max_x - grid_map.min_x)
                             / grid_map.xy_resolution))
    grid_map.y_w = int(round((grid_map.max_y - grid_map.min_y)
                             / grid_map.xy_resolution))

    # 공간을 `0.5m` 해상도의 격자로 나눠 각 셀에 확률을 균등하게 초기화합니다.
    grid_map.data = [[1.0 for _ in range(grid_map.y_w)]
                     for _ in range(grid_map.x_w)]
      # 초기 확률 분포는 정규화됩니다
    grid_map = normalize_probability(grid_map)

    return grid_map

2. 이동 모델 (motion_update)


def motion_update(grid_map, u, yaw):
    grid_map.dx += DT * math.cos(yaw) * u[0]
    grid_map.dy += DT * math.sin(yaw) * u[0]

    x_shift = grid_map.dx // grid_map.xy_resolution
    y_shift = grid_map.dy // grid_map.xy_resolution

    if abs(x_shift) >= 1.0 or abs(y_shift) >= 1.0:  # map should be shifted
        grid_map = map_shift(grid_map, int(x_shift[0]), int(y_shift[0]))
        grid_map.dx -= x_shift * grid_map.xy_resolution
        grid_map.dy -= y_shift * grid_map.xy_resolution

    # Add motion noise
    grid_map.data = gaussian_filter(grid_map.data, sigma=MOTION_STD)

    return grid_map

로봇이 이동하면, 확률 분포도 함께 움직여야 합니다. 여기서는 이동 거리에 따라 그리드 맵을 shift하며, 가우시안 블러로 이동 오차를 반영합니다.

3. 센서 업데이트 (observation_update)


def calc_gaussian_observation_pdf(grid_map, z, iz, ix, iy, std):
    # predicted range
    x = ix * grid_map.xy_resolution + grid_map.min_x
    y = iy * grid_map.xy_resolution + grid_map.min_y
    d = math.hypot(x - z[iz, 1], y - z[iz, 2])

    # likelihood
    pdf = norm.pdf(d - z[iz, 0], 0.0, std)

    return pdf


def observation_update(grid_map, z, std):
    for iz in range(z.shape[0]):
        for ix in range(grid_map.x_w):
            for iy in range(grid_map.y_w):
                grid_map.data[ix][iy] *= calc_gaussian_observation_pdf(
                    grid_map, z, iz, ix, iy, std)

    grid_map = normalize_probability(grid_map)

    return grid_map

센서로부터 거리 측정 값을 받고, 각 셀에서 해당 측정값이 얼마나 그럴듯한지(likelihood) 를 계산하여 확률을 보정합니다.

pdf = norm.pdf(측정거리 - 예상거리, 0.0, std)

센서 업데이트 후에도 다시 확률을 정규화합니다.

matplotlib을 통해 현재 grid map 상에서의 위치 확률 분포를 히트맵으로 시각화합니다.

📌 정리

  • Histogram Filter Localization은 격자 기반의 위치 추정 알고리즘으로, 확률 분포를 직접 시각화하고 이해하기 쉬운 장점이 있습니다.
  • Kalman Filter 계열이 가우시안 분포 가정을 한다면, Histogram Filter는 임의의 분포 형태를 가질 수 있어 비선형/복잡한 환경에 적합합니다.
  • 다만 계산 비용이 커지는 단점이 있으므로, 로봇이 작고 폐쇄된 공간을 다닐 때 유용하게 사용됩니다.
