WSL2 - PostgreSQL 설치하기

@uncleyoo

Postgresql Wsl2

이번 포스트에서는 PostgreSQL 데이터베이스를 활용해서 개발환경을 구축하도록 하겠습니다.

1) 준비물

  • Docker Desktop (Official Page)
  • DBeaver (Official Page)

2) Docker Desktop 에서 postgres를 검색합니다.

'Pull' 버튼을 눌러 postgres 이미지를 저장합니다. 저장된 이미지는 'Images' 메뉴에서 확인이 가능합니다.

2-1) Container와 분리된 독립적인 저장소 추가

Windows의 경로를 이용하게 되면 Docker는 Path를 정확히 판단하지 못하고 오류를 발생하게 됩니다. 이를 해결하기 위해서 Docker에서 관리하는 Volume을 활용합니다. 좌측 메뉴에서 Volumes를 선택하고 'pgdata'라는 이름으로 볼륨을 생성합니다.

2-2) Container 실행

name: postgres
port: 5432
Volume: pgdata | /var/lib/postgresql/data
Env: POSTGRES_PASSWORD | {password}

Docker를 실행 할 때에는 스크린샷과 동일하게 파라미터를 입력해주세요. 모든 입력이 완료되면 'Run' 버튼을 클릭합니다. Image가 정상적으로 실행되면 Docker Desktop의 좌측 메뉴 Containers를 통해 현재 구동중인 항목을 확인 할 수 있습니다.

Terminal에서는 아래와 같은 명령어로 실행이 가능합니다.

docker run --name postgres -e POSTGRES_PASSWORD={password} -d -p 5432:5432  -v ~/docker_volumes/pgdata:/var/lib/postgresql/data -d postgres

2-3) Container와 Volume 연결 확인

녹색불이 들어온걸 확인하면 됩니다.

3) DBeaver를 설치하고 DB 연결을 테스트 합니다.

Community 버전으로 설치를 수행하며 환경에 맞게 설치를 진행합니다. DBeaver는 처음 시작시 설정에 일부 시간이 소요됩니다. 모든 작업이 끝나면 좌측 상단의 '+' 버튼을 눌러 새로운 데이터베이스 연결을 시작합니다.

3-1) 'Postgres'를 선택합니다.

3-2) 비밀번호에 'postgres'를 입력합니다.

3-3) '다음'버튼을 누르면 Driver를 설치합니다.

3-4) 연결 확인

간단하게 Postgre 데이터베이스에 연결까지 확인 했습니다.
